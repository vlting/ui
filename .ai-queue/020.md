# Create Standalone Field Component

## Objective

Create a composable form field wrapper (`Field`) that provides label, description, and error message patterns without requiring a `<Form>` parent. Our `Form` component has field context, but there's no standalone `Field` component matching shadcn/ui's pattern.

## Context

shadcn/ui provides `Field`, `FieldLabel`, `FieldDescription`, `FieldError` as standalone composable primitives. Our Form uses a FormField context internally, but consumers who don't need a full `<form>` element (e.g., single filter inputs, settings toggles) can't use our error/label pattern without wrapping everything in `<Form>`.

## Instructions

### 1. Create Component

Create `packages/components/Field/Field.tsx` with:

**Sub-components:**
- `Field.Root` — Context provider, accepts `error?: boolean`, `disabled?: boolean`, generates IDs
- `Field.Label` — Renders `<label>` with `htmlFor` linked to field input (uses `useId()`)
- `Field.Control` — Slot for the actual input (passes `id`, `aria-describedby`, `aria-invalid`)
- `Field.Description` — Helper text, linked via `aria-describedby`
- `Field.Error` — Error message, shown when `error=true`, has `role="alert"`

**Usage pattern:**
```tsx
<Field.Root error>
  <Field.Label>Email</Field.Label>
  <Field.Control>
    <Input />
  </Field.Control>
  <Field.Description>We'll never share your email.</Field.Description>
  <Field.Error>Please enter a valid email.</Field.Error>
</Field.Root>
```

### 2. Create Spec

Write `packages/components/Field/Field.spec.md` following the unified spec template.

### 3. Create Test

Write `packages/components/Field/Field.test.tsx` covering:
- Renders label, description, error message
- Links label to control via htmlFor/id
- Error message has `role="alert"`
- Description linked via `aria-describedby`
- Disabled state propagates

### 4. Export

Add to `src/index.ts`:
```typescript
export { Field } from './packages/components/Field/Field'
```

### 5. Verify

- Run tests: `npx jest packages/components/Field/`
- Run type check: `npx tsc --noEmit`
