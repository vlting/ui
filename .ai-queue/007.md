# Add new components: Popover, Sheet, Toast, Menu

## Context

Tamagui provides several component packages that our library doesn't yet wrap. These are commonly needed in real applications and are available as fully-styled, accessible components in `node_modules/@tamagui/`. Adding them expands our library's capabilities and keeps us aligned with the Tamagui ecosystem.

## Implementation

### Popover (`packages/components/Popover/Popover.tsx` — NEW)

```tsx
import { Popover as TamaguiPopover } from '@tamagui/popover'
```

1. Create `packages/components/Popover/` directory with `Popover.tsx`.
2. Tamagui's Popover uses floating-ui for positioning, has focus management, dismiss on outside click, Escape key, scroll handling.
3. Compound: `Popover.Trigger`, `Popover.Content`, `Popover.Anchor`, `Popover.Arrow`, `Popover.Close`, `Popover.Adapt`, `Popover.ScrollView`, `Popover.FocusScope`.
4. `styled()` wrap `Popover.Content` for our visual styling (background, border, shadow, border-radius using theme tokens).
5. Export a re-styled compound:
   ```tsx
   export const Popover = {
     Root: TamaguiPopover,  // or styled wrapper
     Trigger: TamaguiPopover.Trigger,
     Content: StyledPopoverContent,
     Arrow: TamaguiPopover.Arrow,
     Close: TamaguiPopover.Close,
   }
   ```
6. Key props: `open`, `defaultOpen`, `onOpenChange`, `placement`, `offset`, `hoverable`.

### Sheet (`packages/components/Sheet/Sheet.tsx` — NEW)

```tsx
import { Sheet as TamaguiSheet, createSheet } from '@tamagui/sheet'
```

1. Create `packages/components/Sheet/` directory with `Sheet.tsx`.
2. Tamagui's Sheet is a bottom sheet with snap points, drag-to-dismiss, overlay.
3. Compound: `Sheet.Frame`, `Sheet.Handle`, `Sheet.Overlay`, `Sheet.ScrollView`.
4. Use `createSheet({ Frame: StyledFrame, Handle: StyledHandle, Overlay: StyledOverlay })` factory for customized visuals.
5. Key props: `open`, `onOpenChange`, `snapPoints`, `position`, `defaultPosition`, `modal`, `dismissOnOverlayPress`, `dismissOnSnapToBottom`.
6. Style with our theme tokens for background, handle color, overlay opacity.

### Toast (`packages/components/Toast/Toast.tsx` — NEW)

```tsx
import {
  Toast as TamaguiToast,
  ToastProvider,
  ToastViewport,
  useToastController,
  useToastState,
} from '@tamagui/toast'
```

1. Create `packages/components/Toast/` directory with `Toast.tsx`.
2. Tamagui's Toast has: imperative API (`useToastController()`), auto-dismiss with configurable `duration`, swipe-to-dismiss, pause on hover.
3. Compound: `Toast.Title`, `Toast.Description`, `Toast.Action`, `Toast.Close`.
4. `styled()` wrap the Toast frame for our visual variants:
   - `variant`: 'default' | 'success' | 'error' | 'warning' (map to theme colors)
   - Style with `$background`, `$borderColor`, theme-appropriate `$color`.
5. Export: `Toast` compound + `ToastProvider` + `ToastViewport` + hooks.
6. The provider and viewport should be added to our `Provider` component in a future task (or document how to add them).

### Menu (`packages/components/Menu/Menu.tsx` — NEW)

```tsx
import { Menu as TamaguiMenu } from '@tamagui/menu'
```

1. Create `packages/components/Menu/` directory with `Menu.tsx`.
2. Tamagui's Menu wraps native context menus on iOS/Android, falls back to popover-based menu on web.
3. Comprehensive compound: `Menu.Trigger`, `Menu.Content`, `Menu.Item`, `Menu.ItemTitle`, `Menu.ItemSubtitle`, `Menu.ItemIcon`, `Menu.Group`, `Menu.Label`, `Menu.Separator`, `Menu.CheckboxItem`, `Menu.RadioGroup`, `Menu.RadioItem`, `Menu.Sub`, `Menu.SubTrigger`, `Menu.SubContent`, `Menu.Arrow`.
4. `styled()` wrap Content, Item, and Separator for our visual styling.
5. Key props: platform-adaptive behavior, keyboard navigation on web, native menu on mobile.
