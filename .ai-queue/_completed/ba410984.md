# Task: Fix ARIA/a11y + focus management for overlay & menu components

## Scope
Fix WCAG A violations and focus management in these component directories under `packages/components/`:
- Dialog, AlertDialog, Sheet, Drawer, Popover, HoverCard, ContextMenu, DropdownMenu, Menu, Menubar, NavigationMenu, Tooltip, Toast, Command, Combobox, Select, Collapsible

## Instructions

1. **Read each component's source** (the main `.tsx` file in each directory).

2. **Fix focus management (critical for overlays):**
   - Dialog/AlertDialog/Sheet/Drawer: focus must move to the dialog on open, return to trigger on close
   - Ensure `aria-modal="true"` on modal overlays
   - Ensure `role="dialog"` or `role="alertdialog"` as appropriate
   - AlertDialog: must have `aria-describedby` pointing to the description
   - Sheet/Drawer: ensure proper `aria-label` or `aria-labelledby`

3. **Fix menu component ARIA:**
   - DropdownMenu/ContextMenu: `role="menu"` on container, `role="menuitem"` on items
   - Menubar: `role="menubar"` with `role="menuitem"` children
   - NavigationMenu: `role="navigation"` with `aria-label`
   - Combobox: `role="combobox"` with `aria-expanded`, `aria-controls`, `aria-activedescendant`
   - Select: `role="listbox"` on options container, `role="option"` on items
   - Command: proper `role="listbox"` or `role="menu"` pattern

4. **Fix tooltip/popover ARIA:**
   - Tooltip: `role="tooltip"` on content, `aria-describedby` on trigger
   - Popover: `aria-haspopup`, `aria-expanded` on trigger
   - HoverCard: `aria-haspopup` on trigger

5. **Fix toast ARIA:**
   - Toast container: `role="status"` or `aria-live="polite"`
   - Urgent toasts: `role="alert"` or `aria-live="assertive"`

6. **Collapsible:**
   - Trigger: `aria-expanded`, `aria-controls`
   - Content: `id` matching `aria-controls`

7. **Do NOT:**
   - Add color-contrast fixes
   - Modify test files
   - Touch files outside the listed component directories

## Acceptance
- All overlay components manage focus correctly (trap on open, restore on close)
- All menu components have proper ARIA roles and keyboard navigation support
- All popover/tooltip components have proper ARIA relationships
- Toast uses live regions appropriately
