## Instructions

The chart pages at `/docs/charts/[type]` currently only show code blocks. The chart components exist in `packages/components/Chart/` and are fully implemented. The `react-native-svg-web` package is already in the root `devDependencies`. Add live chart rendering.

### Step 1: Configure react-native-svg-web alias in Next.js

Check if `examples/docs/next.config.mjs` or `examples/docs/next.config.ts` exists and read its contents. Add a webpack alias for `react-native-svg`:

```js
webpack: (config) => {
  config.resolve.alias = {
    ...config.resolve.alias,
    'react-native-svg': require.resolve('react-native-svg-web'),
  }
  return config
}
```

If the config already has a `webpack` function, merge the alias into it rather than overwriting.

Also check if `react-native` needs an alias too — some Tamagui packages import from `react-native`. The docs app may already have this configured since it renders Tamagui components. If not, add:
```js
'react-native': 'react-native-web',
```

### Step 2: Add sample data to chart-registry.ts

The chart registry already has code examples as strings. For live rendering, we need actual data objects that can be passed as props. Check if the registry already exports sample data objects (the research found `sampleData`, `pieData`, `radarData` etc.).

If these exist, great — use them. If they're embedded in code strings, extract them as exported constants:

```ts
export const chartSampleData = {

    { month: 'Feb', value: 305 },
    // ...
  ],
  bar: [...],
  line: [...],
  pie: [...],
  radar: [...],
  radial: [...],
}
```

### Step 3: Add live chart previews to chart pages

In `[type]/page.tsx`, add a live preview section for each chart type. Use dynamic imports with `ssr: false`:

```tsx
'use client'
import dynamic from 'next/dynamic'

const ChartComponents = {
  area: dynamic(() => import('../../../../packages/components/Chart/AreaChart').then(m => ({ default: m.AreaChart })), { ssr: false }),
  bar: dynamic(() => import('../../../../packages/components/Chart/BarChart').then(m => ({ default: m.BarChart })), { ssr: false }),
  line: dynamic(() => import('../../../../packages/components/Chart/LineChart').then(m => ({ default: m.LineChart })), { ssr: false }),
  pie: dynamic(() => import('../../../../packages/components/Chart/PieChart').then(m => ({ default: m.PieChart })), { ssr: false }),
  radar: dynamic(() => import('../../../../packages/components/Chart/RadarChart').then(m => ({ default: m.RadarChart })), { ssr: false }),
  radial: dynamic(() => import('../../../../packages/components/Chart/RadialChart').then(m => ({ default: m.RadialChart })), { ssr: false }),
}
```

Render one representative live preview per chart type at the top of the page (before the variants code blocks). Use the sample data from Step 2. Wrap in a `ComponentPreview`-style container.

### Important: The page is currently a server component

The `[type]/page.tsx` uses `generateStaticParams()` and is a server component. To add dynamic client-side chart rendering, you'll need to either:
1. Create a separate client component `ChartPreview.tsx` that handles the dynamic imports and rendering, then import it into the server page
2. Or convert the page to a client component (less ideal)

Option 1 is preferred — create a `ChartPreview` client component in `examples/docs/components/` or inline in the charts directory.

**Wait** — creating a new file in `examples/docs/components/` would overlap with other segments' scope. Instead, create the client component in the charts directory itself: `examples/docs/app/docs/charts/chart-preview.tsx`
