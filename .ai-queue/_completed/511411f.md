## Instructions

### Login03 — Muted background login

**Layout:**
- Full-page layout with muted background color (`$color1` or a surface alt theme)
- Centered AuthFormCard on top of the muted background
- Same form content as Login01 (email, password, forgot password, submit, social, footer)
- The difference from Login01 is purely the full-page wrapper with background color

**Props:** Same as Login01 (`Login03Props extends LoginBlockProps` with optional `title`, `description`)

**Implementation:**
- Outer container: full viewport width/height, `backgroundColor: '$color1'`, centered content
- Inner content is the same form pattern as Login01
- Consider extracting the form body to a shared internal helper if it becomes repetitive (but keep it simple — duplication between 5 files is OK for blocks)

### Login04 — Form with adjacent image (inside card)

**Layout:**
- Single Card that contains both the form AND an image side-by-side
- On desktop: left = form fields, right = image (inside the card boundary)
- On mobile: image hidden, just the form
- Key difference from Login02: Login02 has image OUTSIDE the card as a separate full-height column. Login04 has the image INSIDE the card.

**Props:**
```typescript
export interface Login04Props extends LoginBlockProps {
  title?: string
  description?: string
  /** Image rendered adjacent to the form inside the card */
  image?: React.ReactNode
}
```

**Implementation:**
- Card with `flexDirection: 'row'` on desktop, `flexDirection: 'column'` on mobile
- Left side (form): padding, standard form layout
- Right side (image): no padding, image fills the space with `overflow: 'hidden'` and `borderTopRightRadius`/`borderBottomRightRadius` matching the card radius
- Use Tamagui media queries for responsive behavior

### Login05 — Email-only login (magic link)

**Layout:**
- Centered card, simpler than Login01
- Only email field (no password)
- Submit button text: "Sign in with email" or "Send magic link"
- No forgot password link (not needed for email-only)
- Social providers still supported
- Footer link: "Don't have an account? Sign up"

**Props:**
```typescript
export interface Login05Props extends LoginBlockProps {
  title?: string
  description?: string
  /** Submit button text — defaults to "Sign in with email" */
  submitText?: string
}
```

**Implementation:**
- Same pattern as Login01 but stripped down
- Only one input field (email)
- `onSubmit` callback receives `{ email }` (no password)
- Simpler, cleaner layout

### Spec files

- **Login04:** Image inside the card variant. Use when you want a compact hero-style login.
- **Login05:** Email-only variant. Use for magic link or passwordless authentication flows.
