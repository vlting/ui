# Task: Extend stl color and shadow scales for vlt-ui palettes

## What
Extend stl's color generation to support vlt-ui's 12-step palette structure with accent colors, and map shadow scales.

## Reference
Read these files first:
- `packages/design-tokens/base.ts` — vlt-ui color tokens
- `packages/design-tokens/themes.ts` — vlt-ui theme/palette builder, shadow definitions
- `packages/design-tokens/brands/index.ts` — palette structure in BrandDefinition (palettes, accentPalettes, shadows)
- `packages/stl/src/config/scales/color.ts` — stl color generation
- `packages/stl/src/config/scales/shadow.ts` — stl shadow scale
- `packages/stl/src/shared/models/theme.models.ts` — stl theme types
- `packages/stl/src/shared/utils/colorGen.utils.ts` — stl color generation utils
- `packages/stl/src/shared/models/colorGen.models.ts` — color gen model types

## Steps

### 1. Color scale (`packages/stl/src/config/scales/color.ts`)
vlt-ui uses 12-step palettes (index 0-11, lightest to darkest in light mode). Each palette has light + dark variants. stl already has primary1-12, secondary1-12, tertiary1-12.

- Add accent color scales to match vlt-ui's accent palettes: `blue1-12`, `red1-12`, `green1-12`, `orange1-12`, `purple1-12`, `pink1-12`, `yellow1-12`
- For each accent, include text color variants: `blueText1-12`, etc.
- Add basic color constants: `white`, `black`, `transparent` (vlt-ui has these as standalone tokens)
- Ensure light/dark mode awareness via stl's theme variable system

### 2. Semantic color tokens
vlt-ui maps palette indices to semantic names via templates:
- `background`, `backgroundHover`, `backgroundPress`
- `color`, `colorHover`, `colorPress`
- `borderColor`, `borderColorHover`, `borderColorPress`
- `shadowColor`
- Surface levels: `surface1`, `surface2`, `surface3`, `inverse`

Add these as semantic aliases in stl's color scale that reference palette step indices. Use stl's existing pattern of creating CSS variable references.

### 3. Shadow scale (`packages/stl/src/config/scales/shadow.ts`)
Map vlt-ui shadow keys to stl:
- `sm` → `low` (already exists, verify values match)
- `md` → `medium`
- `lg` → `high`
- `xl` → `highSoft`
- `2xl` → `highHeavy`

Add aliases so both key sets work:
```ts
// vlt-ui aliases
shadowSm → shadowLow
shadowMd → shadowMedium
shadowLg → shadowHigh
shadowXl → shadowHighSoft
shadow2xl → shadowHighHeavy
```

Ensure dark mode variants match vlt-ui's higher opacity values.

### 4. Theme models (`packages/stl/src/shared/models/theme.models.ts`)
Add types for:
- `AccentPalette` — accent color palette definition
- `SemanticColorMap` — mapping of semantic names to palette indices
- `SurfaceTheme` — surface level theme template
