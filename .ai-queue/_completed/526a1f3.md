# A11y Fix: Tabs Trigger — Use Native `<button>` Element

## Priority: HIGH

## Root Cause

The styled Tabs component uses `styled(XStack, { tag: 'button' })` for tab triggers, which renders `<div tag="button">`. The headless Tabs layer (`packages/headless/Tabs/Tabs.tsx`) correctly uses native `<button>` elements, but the styled layer does NOT use the headless trigger — it replaces it with its own styled frame.

## Scope

Files modified:
- `packages/components/Tabs/Tabs.tsx`

## Required Changes

### 1. Use the headless Trigger (recommended) OR wrap in native `<button>`

**Option A (preferred)**: Use the headless `TabTrigger` (which renders a native `<button>`) and apply Tamagui styles visually around/inside it:

```tsx
function Trigger({ value, children, ...props }) {
  const { activeValue, setActiveValue } = useTabsContext()
  const isSelected = activeValue === value

  return (
    <HeadlessTabs.Trigger value={value}>
      {/* The headless Trigger renders a native <button> */}
      <StyledTriggerVisual isActive={isSelected} {...props}>
        {children}
      </StyledTriggerVisual>
    </HeadlessTabs.Trigger>
  )
}
```

Wait — check if the headless Trigger uses `cloneElement` or renders its own `<button>` that wraps children. Read the headless source to understand the render pattern, then apply the styled visual accordingly.

**Option B**: If the headless Trigger render pattern doesn't easily wrap styled content, render a native `<button>` directly:

```tsx
function Trigger({ value, children, size, ...props }) {
  const { activeValue, setActiveValue, tabListId, orientation } = useTabsContext()
  const isSelected = activeValue === value

  return (
    <button
      type="button"
      role="tab"
      aria-selected={isSelected}
      aria-controls={`tabpanel-${value}`}
      id={`tab-${value}`}
      tabIndex={isSelected ? 0 : -1}
      onClick={() => setActiveValue(value)}
      style={{ all: 'unset', display: 'inline-flex', cursor: 'pointer' }}
    >
      <StyledTriggerVisual isActive={isSelected} size={size} {...props}>
        {children}
      </StyledTriggerVisual>
    </button>
  )
}
```

### 2. Add focusWithinStyle to visual frame

```tsx
const StyledTriggerVisual = styled(XStack, {

```

### 3. Preserve keyboard navigation

The headless Tabs layer has arrow key navigation and roving tabindex. If using Option A (headless trigger), this comes for free. If using Option B, ensure the keyboard handling from the headless layer is maintained or reimplemented.
