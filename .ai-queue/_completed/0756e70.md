# Segment 1: Foundation — packages/blocks/ structure and shared types

## Objective

Create the `packages/blocks/` directory structure with shared types, barrel exports, and shared auth form composition utilities. This establishes the patterns all subsequent block segments will follow.

## Context

- **Tech Spec:** `.ai-epics/docs/block-system/tech-spec.md`
- **PRD:** `.ai-sagas/docs/shadcn-parity/prd.md` (FR-4: Block System)
- **Feature flag:** `block_system`
- **AI Constitution:** `AI_CONSTITUTION.md` — blocks are presentation-only, no business logic
- **Design Constitution:** `DESIGN_CONSTITUTION.md` — all styling via design tokens
- **Quality Baseline:** `packages/QUALITY_BASELINE.md`

## Key Patterns

- Use `type AnyFC = ComponentType<Record<string, unknown>>` and cast Tamagui components for JSX (Tamagui v2 RC GetFinalProps bug)
- Use `styledHtml('form', {...})` for `<form>` elements (tag prop doesn't work in v2 RC)
- All spacing via design tokens (`$0.5` through `$10`)
- All colors via design tokens (`$color`, `$borderColor`, `$background`, etc.)
- Import components from relative internal paths (e.g., `../../components`, `../../primitives`)
- Focus styles: `focusVisibleStyle: { outlineWidth: 2, outlineOffset: 2, outlineColor: '$outlineColor', outlineStyle: 'solid' }`

## Instructions

### 1. Create `packages/blocks/_shared/types.ts`

Define shared block types:

```typescript
import type { ReactNode } from 'react'

/** Base props all blocks accept */
export interface BlockProps {
  children?: ReactNode
}

/** Social login provider definition */
export interface SocialProvider {
  /** Display name (e.g., "Google", "GitHub") */
  name: string
  /** Icon element to render */
  icon: ReactNode
  /** Handler when this provider is selected */
  onPress: () => void
}

/** Common props shared by all auth blocks (login + signup) */
export interface AuthBlockProps extends BlockProps {
  /** Form submit handler */
  onSubmit?: (data: Record<string, string>) => void
  /** Optional social login providers */
  socialProviders?: SocialProvider[]
  /** Optional logo element rendered above the form */
  logo?: ReactNode
  /** Loading state for the submit button */
  loading?: boolean
  /** Error message to display at the top of the form */
  error?: string
}

/** Login-specific props extending AuthBlockProps */
export interface LoginBlockProps extends AuthBlockProps {
  /** "Forgot password" link href or handler */
  forgotPasswordHref?: string
  onForgotPassword?: () => void
  /** "Sign up" redirect link href or handler */
  signupHref?: string
  onSignup?: () => void
}

/** Signup-specific props extending AuthBlockProps */
export interface SignupBlockProps extends AuthBlockProps {
  /** "Sign in" redirect link href or handler */
  loginHref?: string
  onLogin?: () => void
  /** Terms of service acceptance (if the block has a checkbox) */
  termsHref?: string
  privacyHref?: string
}
```

### 5. Create `packages/blocks/login/_shared.tsx`

Shared auth form composition utilities. These are internal building blocks used by Login01-05 and Signup01-05 (signup blocks will import from `../login/_shared`).

Build these composable pieces using existing @vlting/ui components:

**AuthFormCard** — A Card wrapper with consistent layout for auth forms:
- Wraps `Card` from `../../components/Card`
- Sets consistent padding (`$6` horizontal, `$6` vertical)
- Sets max-width (400px for centered forms via `style={{ maxWidth: 400 }}`)
- Sets `width: '100%'`

**AuthFormHeader** — Logo + title + description area:
- Accepts `logo?: ReactNode`, `title: string`, `description?: string`
- Renders logo (if provided) centered above title
- Title uses `Text` with `fontSize: '$6'`, `fontWeight: '$4'`, `fontFamily: '$heading'`
- Description uses `Text` with `fontSize: '$3'`, `color: '$colorSubtitle'`, `fontFamily: '$body'`
- Centered text alignment
- Vertical gap between elements (`$2`)

**AuthSocialButtons** — Row of social login provider buttons:
- Accepts `providers: SocialProvider[]`
- Renders Button per provider with `variant="outline"`, showing icon + name
- If only 1-2 providers: full-width buttons stacked vertically
- If 3+ providers: row of icon-only buttons

**AuthDivider** — "or continue with" styled separator:
- Horizontal line with centered text
- Uses `Separator` (from primitives) or a styled `View` with a border
- Text: `Text` with `fontSize: '$2'`, `color: '$colorSubtitle'`
- Pattern: `<HStack alignItems="center" gap="$2"><Separator flex={1} /><Text>or continue with</Text><Separator flex={1} /></HStack>`

**AuthFooterLink** — Link text at bottom of form (e.g., "Don't have an account? Sign up"):
- Accepts `text: string`, `linkText: string`, `href?: string`, `onPress?: () => void`
- Centered text using `$colorSubtitle`, with linkText in `$color10` (interactive color)
- The link portion uses a `<button>` via `styledHtml('button')` or a native `<a>` — not a div with onClick

All shared components must:
- Use design tokens exclusively (no hardcoded colors or spacing)
- Cast Tamagui components as `AnyFC` for JSX compatibility
- Be exported as named exports from `_shared.tsx`
