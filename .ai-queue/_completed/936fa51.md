# Eliminate unnecessary wrapper elements inside interactive elements in BrandLayout

## Problem

In `examples/kitchen-sink/src/layouts/BrandLayout.tsx`, native `<button>` and `<Link>` (renders `<a>`) elements wrap Tamagui layout components (`XStack`, `View`) as children solely for spacing/padding. This creates unnecessary `<div>` nodes in the DOM:

```html
<a href="..." style="text-decoration: none; color: inherit; display: block;">
  <div class="...flex row padding...">  <!-- UNNECESSARY -->
    <span class="...text styles...">Button</span>
  </div>
</a>
```

The spacing/layout props on the `XStack`/`View` child should be applied directly to the `<a>` or `<button>` element via inline styles or by extending the existing CSS reset objects.

## Scope

- `examples/kitchen-sink/src/layouts/BrandLayout.tsx`

## Instances to Fix

There are **6 instances** of this pattern in BrandLayout.tsx:

### 1. Nav items (lines ~221-245)
**Current:**
```tsx
<Link to={...} style={LINK_RESET}>
  <XStack paddingVertical={6} paddingLeft={16} paddingRight={8} borderRightWidth={2} borderRightColor={...} backgroundColor={...} style={{ transition: 'all 0.1s' }}>
    <Text ...>{item.label}</Text>
  </XStack>
</Link>
```
**Fix:** Move the XStack layout/spacing props into the `<Link>` `style` prop. The `<Text>` becomes a direct child of `<Link>`. The `borderRightWidth`, `borderRightColor`, `backgroundColor` conditional styles also move to the Link's inline style. Remove the XStack entirely.

## Implementation Notes

### Token Resolution
Since we're moving from Tamagui props to inline CSS styles, Tamagui token values (`$0.25`, `$0.5`, `$1.5`, `$2`, `$3`) need to be resolved to their CSS variable equivalents. Tamagui exposes tokens as CSS custom properties:
- Space tokens: `var(--t-space-X)` (check the actual variable names by inspecting the rendered output or looking at the Tamagui config)
- Color tokens: `var(--color)`, `var(--colorSubtitle)`, `var(--borderColor)`, `var(--color3)`, `var(--color10)`, `var(--background)` (these are already used elsewhere in the file as CSS variables)
- Border radius: `var(--t-radius-X)` or the equivalent

**Alternative approach that avoids token resolution:** Instead of inline styles, use Tamagui's `styled()` or `styledHtml()` to create styled versions of the interactive elements:
```tsx
import { styledHtml } from 'tamagui'

const NavLink = styledHtml('a', {
  display: 'flex',
  flexDirection: 'row',
  paddingVertical: 6,
  paddingLeft: 16,
  paddingRight: 8,
  textDecoration: 'none',
  color: 'inherit',
  // ... etc
})
```
This approach keeps Tamagui token resolution working and is consistent with how the component library does it (NavigationMenu, Sidebar, Menubar all use `styledHtml()`). **This is the preferred approach.**

However, `<Link>` is a React Router component, not a native `<a>`. For React Router Links, you can either:
1. Pass `style` with resolved CSS variables, OR
2. Use the `<Link>` component's `style` prop with CSS variables already used elsewhere in the file (`var(--borderColor)`, `var(--color)`, etc.)

For native `<button>` elements, use `styledHtml('button', {...})` to create styled buttons with Tamagui tokens.
