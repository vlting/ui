# Segment 3: ToggleGroup cast fix + component-examples text node audit

## Objective
Fix the ToggleGroup undefined component error and audit all component examples for text node violations (raw text strings passed as children to View-based Tamagui components).

## Scope

## Instructions

### Fix 1: ToggleGroup cast

The ToggleGroup is cast at line 327 as:
```ts
const ToggleGroup = _ToggleGroup as unknown as {
  Root: AnyFC
  Item: AnyFC
}
```

But `ToggleGroup` uses `withStaticProperties(ToggleGroupRoot, { Item })` — the component itself IS the root. There is NO `.Root` sub-component. `ToggleGroup.Root` evaluates to `undefined`, causing: "Element type is invalid: expected a string or class/function but got: undefined."

**Fix:** Change the cast to:
```ts
const ToggleGroup = _ToggleGroup as AnyFC & { Item: AnyFC }
```

Then update the example at line ~1232 from:
```tsx
<ToggleGroup.Root type="single">
```
to:
```tsx
<ToggleGroup type="single">
```

And the closing tag from `</ToggleGroup.Root>` to `</ToggleGroup>`.

### Fix 2: Text nodes in View-based components

Several examples pass raw text strings as children to View-based Tamagui components, which will cause "Unexpected text node" errors at runtime. Import `Text` from `tamagui` and cast as `AnyFC`:
```ts
import { Text as _Text } from 'tamagui'
const TamaguiText = _Text as AnyFC
```

(Check if Text is already imported — if so, reuse that import.)

**Key locations to fix:**

1. **Accordion.Content** children (around lines 570-584): Three instances of raw text strings like:
   ```tsx
   <Accordion.Content>
     Yes. It adheres to the WAI-ARIA design pattern.
   </Accordion.Content>
   ```
   Wrap each in `<TamaguiText>...</TamaguiText>`.

2. **Audit ALL other compound component children** throughout the entire file. Look for any place where a bare string (not JSX) is passed as children to a component that is likely View-based. Common suspects:
   - `Accordion.Content`
   - `Collapsible.Content`
   - `HoverCard.Content`
   - `Tooltip` content
   - Any `.Content` or `.Description` sub-component

   For each, check if the component is View-based or Text-based. If View-based, wrap the text in `<TamaguiText>`.

3. **Do NOT wrap text that's already inside a `<Text>`, `<span>`, `<p>`, or HTML element** — those already handle text nodes correctly.

4. **Do NOT wrap children of components that are known to handle text internally** (e.g., `Button` with its own Text sub-component, `Badge`, `Label`).
