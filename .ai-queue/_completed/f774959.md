### File 1: `packages/icons/__tests__/DynamicIcon.test.tsx`

Tests for the `DynamicIcon` component:

```tsx
import { render, waitFor } from '@testing-library/react'

```

1. **renders an icon by name** — render `<DynamicIcon name="arrow-right" />`, use `waitFor` to verify an SVG appears (lazy loading is async).

2. **accepts variant prop** — render `<DynamicIcon name="home" variant="fill" />`, wait for SVG to appear.

3. **accepts size and color props** — render `<DynamicIcon name="arrow-right" size={32} color="red" />`, wait and verify the SVG has correct width/height/fill attributes.

4. **handles unknown icon names gracefully** — render `<DynamicIcon name="nonexistent-icon-xyz" />`, wait briefly, verify NO error is thrown and the component renders (may render null/empty).

5. **caches lazy components** — this is an implementation detail test. Import the icon module, render `<DynamicIcon name="arrow-right" />` twice in sequence, verify the same lazy component reference is used (by checking that `React.lazy` isn't called twice for the same name). Alternatively, just verify that rendering the same name twice doesn't cause issues.

**IMPORTANT:** DynamicIcon uses `React.lazy` with dynamic `import()`. In Jest, dynamic imports resolve synchronously when the module is already loaded. You may need `waitFor` or `act` to handle the lazy loading. If `React.lazy` doesn't work well in jsdom, mock the dynamic import:

```tsx
// If needed, mock the dynamic import

  RiArrowRightLine: ({ size, color }: any) => <svg width={size} height={color} />,
}))
```
