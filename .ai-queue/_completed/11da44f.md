#### Universal Changes
These apply to ALL components:

**Import paths:**
- Before: `import { Button } from "@/components/ui/button"`
- After: `import { Button } from "@vlting/ui"`

**Styling:**
- Before: `className="bg-blue-500 p-4 rounded-lg"`
- After: `backgroundColor="$blue10" padding="$4" borderRadius="$4"` (Tamagui token props)
- Explain that Tailwind classes don't work — use Tamagui style props with design tokens

**Event handlers:**
- `onClick` → `onPress`
- `onChange` → `onValueChange` (for select, switch, etc.)

**Size naming:**
- `"default"` → `"md"`, `"sm"` stays `"sm"`, `"lg"` stays `"lg"`

**Compound components:**
- Dot-notation pattern: `<Dialog.Root>`, `<Dialog.Trigger>`, `<Dialog.Content>`
- Some components use `withStaticProperties` (callable, no `.Root`)
- Others are plain objects (need `.Root`)

#### Prop Renames Table
- Create a reference table with columns: shadcn Prop → @vlting/ui Prop → Components Affected
- Key renames: `onClick`→`onPress`, `size="default"`→`size="md"`, `className`→style props, `onChange`→`onValueChange`, `open`/`onOpenChange` (same), `disabled` (same)

#### Component-Specific Changes
For each component that has significant API differences, show before/after code examples:

- **Form / Field**: shadcn uses react-hook-form; @vlting/ui has built-in Field with Label/Control/Description/Error
- **Toast**: Hook-based `toast()` function instead of render-based
- **Tooltip**: Single `<Tooltip>` component wrapping trigger, not `<TooltipProvider>` + `<TooltipTrigger>` + `<TooltipContent>`
- **Select**: Simplified API with `<Select.Item>` instead of separate value/label components
- **DataTable**: Wrapper around @tanstack/react-table with built-in column helpers
- **Combobox**: Single component instead of manual Popover + Command composition

#### Cross-Platform Notes
