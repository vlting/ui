# Stage 3, Segment 2: Fix Wrong Focus Style — Menu & Toggle

## Context

These components use `focusStyle` or `focusWithinStyle` instead of `focusVisibleStyle`. The difference: `focusStyle` fires on ALL focus (including mouse clicks), while `focusVisibleStyle` only fires on keyboard focus. Per WCAG 2.4.7 and our FRONTEND_QUALITY.contract.md, we want keyboard-only focus indicators.

### Standard focusVisibleStyle Pattern

```tsx
focusVisibleStyle: {
  outlineWidth: 2,
  outlineOffset: 2,
  outlineColor: '$outlineColor',
  outlineStyle: 'solid',
}
```

For items inside containers, use `outlineOffset: -2`.

### Changes Required

#### Menu.tsx
- Item, CheckboxItem, RadioItem, SubTrigger all use `focusStyle` — change to `focusVisibleStyle`
- Keep the same styling values, just change the property name from `focusStyle` to `focusVisibleStyle`
- If any of these components lack focus styling entirely, add the standard pattern

#### Toggle.tsx
- The standalone `ToggleFrame` (styledHtml button) uses `focusWithinStyle` — change to `focusVisibleStyle`
- `ToggleGroup.Item` may lack focus styling — add `focusVisibleStyle` if missing
- Keep the same outline styling, just change the property name

### Important Constraints

- Do NOT fix token discipline issues (hardcoded pixels) — that's Stage 4
- Do NOT change HTML element types — that was Stage 2
- Only change to `focusVisibleStyle` — do not remove any other styling
