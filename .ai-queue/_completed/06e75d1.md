# Create Empty State Component

## Objective

Create an `Empty` component for displaying empty state patterns (no data, no results, no items). shadcn/ui provides this as `EmptyMedia`, `EmptyTitle`, `EmptyDescription`. This is a common UI pattern missing from our library.

## Context

Empty states appear everywhere: empty tables, no search results, empty inboxes, first-time-use screens. Without a dedicated component, consumers build ad-hoc empty states with inconsistent spacing, typography, and alignment.

## Instructions

### 1. Create Component

Create `packages/components/Empty/Empty.tsx` with:

**Sub-components:**
- `Empty.Root` — Centered flex container with consistent padding
- `Empty.Media` — Slot for icon or illustration (centered, muted color)
- `Empty.Title` — Heading text (semantic `<h3>` or configurable level)
- `Empty.Description` — Body text (muted, max-width constrained for readability)
- `Empty.Action` — Slot for CTA button (primary action)

**Props:**
- `Root`: `children`, standard layout props
- `Title`: `children`, extends Text props
- `Description`: `children`, extends Text props
- `Media`: `children` (accepts Icon, Image, or custom ReactNode)
- `Action`: `children` (accepts Button or custom ReactNode)

**Design:**
- All spacing from tokens
- Muted text colors from theme (use `$color8` or similar mid-palette)
- Center-aligned, max-width 400px for description text readability
- Works inside Card, Dialog, Table, or standalone

### 2. Create Spec

Write `packages/components/Empty/Empty.spec.md` following the unified spec template.

### 3. Create Test

Write `packages/components/Empty/Empty.test.tsx` covering:
- Renders title and description text
- Renders media slot
- Renders action slot
- All sub-components render without crashing

### 4. Export

Add to `src/index.ts`:
```typescript
export { Empty } from './packages/components/Empty/Empty'
```

### 5. Verify

- Run tests: `npx jest packages/components/Empty/`
- Run type check: `npx tsc --noEmit`
