### 2. Create Icon Component Map

The icon browser needs to render actual SVG icons from `@vlting/ui`. The challenge: there are 3,229 icons and they need to load efficiently.

**Approach — Static import map:**

Since the docs site is a Next.js app that bundles at build time, create a mapping from icon component names to actual components. Check what `@vlting/ui` exports:

```bash
# Check the icon exports from @vlting/ui
grep -r "export.*Ri" packages/components/icons/ | head -20
# or check the barrel export
grep "Ri" src/index.ts | head -20
```

**Option A — Import all icons from the barrel export:**
```tsx
import * as Icons from '@vlting/ui'

// Then in IconCell:
const IconComponent = (Icons as Record<string, React.ComponentType<any>>)[componentName]
if (IconComponent) {
  return <IconComponent size={24} />
}
```

This imports everything but Next.js tree-shaking should handle it since it's used dynamically. However, this may significantly increase bundle size.

**Option B — Dynamic import with React.lazy (preferred for performance):**
```tsx
import dynamic from 'next/dynamic'
import { useMemo } from 'react'

// In IconCell, use dynamic import:
const DynamicIcon = useMemo(() => {
  return dynamic(
    () => import('@vlting/ui').then(mod => {
      const Icon = (mod as any)[componentName]
      return Icon ? { default: Icon } : { default: () => null }
    }),
    { ssr: false, loading: () => <span className="w-6 h-6 bg-gray-100 dark:bg-gray-800 rounded" /> }
  )
}, [componentName])
```

**Option C — Pre-built icon map (most practical):**

Create the icon map at module scope so it's resolved once:
```tsx
import * as AllIcons from '@vlting/ui'

const iconMap: Record<string, React.ComponentType<{ size?: number; color?: string }>> = {}
for (const [name, component] of Object.entries(AllIcons)) {
  if (name.startsWith('Ri') && typeof component === 'function') {
    iconMap[name] = component as React.ComponentType<{ size?: number; color?: string }>
  }
}
```

Then in `IconCell`:
```tsx
const Icon = iconMap[componentName]
if (Icon) {
  return <Icon size={24} />
}
return <span>?</span>  // fallback for missing icons
```

**Choose whichever approach works given the actual export structure of @vlting/ui's icons.** First check how icons are exported — they may be in a separate path like `@vlting/ui/icons` or they may be re-exported through the main barrel.

### 3. Update IconCell Rendering

Replace the Unicode square rendering with actual icon components:

```tsx
function IconCell({ icon, variant }: { icon: IconEntry; variant: 'line' | 'fill' }) {
  const componentName = icon.components[variant]
  const Icon = componentName ? iconMap[componentName] : null

  return (
    <button
      onClick={() => copyImport(componentName)}
      className="flex flex-col items-center gap-1 p-3 rounded-lg hover:bg-gray-100 dark:hover:bg-gray-800 transition-colors cursor-pointer"
      title={`Click to copy: import { ${componentName} } from '@vlting/ui'`}
    >
      <div className="w-8 h-8 flex items-center justify-center">
        {Icon ? (
          <Icon size={24} />
        ) : (
          <span className="text-gray-400 text-xs">N/A</span>
        )}
      </div>
      <span className="text-xs text-gray-600 dark:text-gray-400 truncate w-full text-center">
        {icon.name}
      </span>
    </button>
  )
}
```

### 4. Performance Considerations

- Keep the existing virtualized rendering — only render icons visible in the viewport
- If importing all icons creates a huge bundle, consider:
  - Using `next/dynamic` for each icon cell
  - Lazy loading icon chunks
  - Only importing the subset currently visible
- Test that the icon browser page loads in reasonable time (<3s)
- If virtualization is not yet implemented (check the code), implement it using a simple window-based approach: track scroll position, only render icons within `scrollTop ± viewport * 2` range
