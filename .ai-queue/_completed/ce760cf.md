# Segment 2: Code Generator Script

Create `scripts/generate-icons.mjs` — the Node.js script that reads all Remix Icon SVGs, extracts path data, and generates per-icon `.tsx` components, barrel exports, TypeScript types, and a machine-readable manifest.

## Scope

- `scripts/generate-icons.mjs` (create)
- `package.json` (modify — add `generate:icons` script)

## Instructions

### 1. Create `scripts/generate-icons.mjs`

The script must use **only** Node.js stdlib (`fs`, `path`, `url`) — zero external dependencies.

**Input:** `node_modules/remixicon/icons/{Category}/*.svg`
- 20 categories (folder names): Arrows, Buildings, Business, Communication, Design, Development, Device, Document, Editor, Finance, Food, Game & Sports, Health & Medical, Logos, Map, Media, Others, System, User & Faces, Weather
- 3,229 SVG files total, ALL single-path
- SVG format: `<svg viewBox="0 0 24 24" fill="currentColor" xmlns="..."><path d="M..."/></svg>`
- Naming: `{icon-name}-line.svg` (outline), `{icon-name}-fill.svg` (filled)

**Output directory:** `packages/icons/generated/` (clean before generating — `rm -rf` the directory, then recreate)

**Generated files:**

#### Per-icon `.tsx` files (`packages/icons/generated/Ri{PascalCase}.tsx`)

Convert filename to PascalCase component name with `Ri` prefix:
- `arrow-right-line.svg` → `RiArrowRightLine`
- `arrow-right-fill.svg` → `RiArrowRightFill`
- `game-& sports` category icon names work the same way — just PascalCase the icon filename

Each file:
```tsx
import { createIcon } from '../createIcon'
export const RiArrowRightLine = createIcon('M16.172 11l-5.364...', 'RiArrowRightLine')
```

To extract the path data, use regex: `d="([^"]+)"` on the SVG content.

#### Barrel export (`packages/icons/index.ts`)

```ts
// This file is auto-generated by scripts/generate-icons.mjs
// Do not edit manually.

export { RiArrowDownBoxFill } from './generated/RiArrowDownBoxFill'
export { RiArrowDownBoxLine } from './generated/RiArrowDownBoxLine'
// ... all ~3229 icons, sorted alphabetically
export type { IconFC } from './createIcon'
```

#### Types file (`packages/icons/types.ts`)

```ts

// Do not edit manually.

export type { IconFC } from './createIcon'

export type IconName =
  | 'arrow-down-box-fill'
  | 'arrow-down-box-line'
  // ... all ~3229 icon names (kebab-case filenames without .svg), sorted alphabetically

export type IconCategory =
  | 'arrows'
  | 'buildings'
  // ... all 20 categories (lowercase, derived from folder names), sorted alphabetically
```

For the `IconCategory` type, transform folder names to lowercase. For "Game & Sports" → `'game-and-sports'`, "Health & Medical" → `'health-and-medical'`, "User & Faces" → `'user-and-faces'`. Replace ` & ` with `-and-`, then lowercase and replace spaces with hyphens.

#### Manifest (`packages/icons/manifest.json`)

```json
{
  "version": "4.9.1",
  "count": 3229,
  "categories": ["arrows", "buildings", "business", ...],
  "icons": [
    {
      "name": "arrow-down-box",
      "category": "arrows",
      "variants": ["fill", "line"],
      "components": {
        "fill": "RiArrowDownBoxFill",
        "line": "RiArrowDownBoxLine"
      }
    }
  ]
}
```

Group icons by base name: strip the `-line` or `-fill` suffix from filenames to get the base name. Icons with the same base name are grouped together. Sort icons alphabetically by base name, sort categories alphabetically.

**Script behavior:**
1. Clean output: `rm -rf packages/icons/generated/` then `mkdir -p packages/icons/generated/`
2. Read all SVG files from all category directories
3. For each SVG: extract path data, generate component name, write `.tsx` file
4. Collect all icons into a manifest structure
5. Write `index.ts`, `types.ts`, `manifest.json`
6. Print summary: `Generated N icon components across N categories`

**PascalCase conversion function:**
```js
function toPascalCase(str) {
  return str
    .split(/[-_\s]+/)
    .map(word => word.charAt(0).toUpperCase() + word.slice(1).toLowerCase())
    .join('')
}
```
Then prepend `Ri` and append the variant with first letter uppercased (e.g., `Line`, `Fill`).

Actually, simpler: just PascalCase the full filename (without `.svg`), then prepend `Ri`:
- `arrow-right-line` → `ArrowRightLine` → `RiArrowRightLine`
