# Fix: ARIA Attributes & Semantic HTML — ButtonGroup, Icon, Divider, Drawer, Textarea

Add missing ARIA attributes and semantic HTML to 5 components to meet WCAG 2.2 AA requirements.

## Scope
- `packages/components/ButtonGroup/ButtonGroup.tsx`
- `packages/primitives/Icon.tsx`
- `packages/primitives/Divider.tsx`
- `packages/components/Drawer/Drawer.tsx`
- `packages/components/Textarea/Textarea.tsx`

## Instructions

Read `AI_CONSTITUTION.md` and `DESIGN_CONSTITUTION.md` before starting.

### Component Changes

#### 1. ButtonGroup (`packages/components/ButtonGroup/ButtonGroup.tsx`)
- Add `role="group"` to the container element
- Add an optional `aria-label` prop to the component's props interface
- Pass `aria-label` through to the container. If not provided, do NOT add a hardcoded default (the consumer should provide it).

#### 2. Icon (`packages/primitives/Icon.tsx`)
- Add `aria-hidden="true"` as a default prop on the icon render
- Allow consumers to override it (e.g., by passing `aria-hidden={false}` and an `aria-label`)
- Icons are decorative by default; consumers opt-in to semantic usage

#### 3. Divider (`packages/primitives/Divider.tsx`)
- Add `role="separator"` to the styled component
- Add `aria-orientation` that matches the component's `orientation` prop/variant:
  - `aria-orientation="horizontal"` when horizontal (default)
  - `aria-orientation="vertical"` when vertical
- This can be done in the JSX by passing the prop based on orientation, or in the Tamagui styled config if supported

#### 4. Drawer Title (`packages/components/Drawer/Drawer.tsx`)
- The `Title` sub-component currently wraps children in a `ViewJsx` (styled div)
- Replace with a heading element. Use `styledHtml('h2', { ... })` with the same styling, OR wrap with a native `<h2>` element
- Drawer titles should be `<h2>` (drawers are typically secondary content panels)
- Keep the same visual styling (fontFamily, fontSize, fontWeight, color)

#### 5. Textarea (`packages/components/Textarea/Textarea.tsx`)
- Add `focusVisibleStyle` to the styled textarea definition
- Use the standard focus pattern from other components:
  ```
  focusVisibleStyle: {
    outlineWidth: 2,
    outlineOffset: 2,
    outlineColor: '$outlineColor',
    outlineStyle: 'solid',
  }
  ```

### Rules
1. Do NOT change any existing styling or behavior beyond adding the specified attributes/elements.
2. All changes must maintain backwards compatibility — existing consumers should not break.
3. New props (like `aria-label` on ButtonGroup) must be optional.
4. Preserve the `asChild` or composition patterns already used in each component.
