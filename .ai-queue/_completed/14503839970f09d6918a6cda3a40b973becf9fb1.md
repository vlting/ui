# Integration вҖ” Registry, Exports, Cleanup, and Docs

## Instructions

This is the integration segment. It runs AFTER all block creation segments are complete. It wires up the new blocks, removes the old numbered blocks, and updates the docs system to support variant-based blocks.

### Step 1: Update Shared Types

Update `packages/blocks/_shared/types.ts`:
- Keep existing `BlockProps`, `SocialProvider` types
- Add new block-specific types: `MetricCard`, `PricingPlan`, `PreferenceItem`, `TimelineEvent`, `NotificationItem`, `CommentItem`
- Add variant union types for each block
- Keep `AuthBlockProps`, `LoginBlockProps`, `SignupBlockProps` for backwards compatibility or remove if no longer referenced

- Export shared types (NavItem, NavGroup, FileTreeItem) from `_shared.tsx`
- Remove old Sidebar01-16 exports

- Remove Dashboard01 exports

Create `packages/blocks/auth/index.ts`:

### Step 4: Update Root Exports

Rewrite `packages/blocks/index.ts`:
- Export all 10 new blocks and their prop types
- Export shared types from `_shared/types.ts`
- Export sidebar types (NavItem, NavGroup, FileTreeItem)
- Remove all old Login01-05, Signup01-05, Sidebar01-16, Dashboard01, originals exports

### Step 5: Rewrite Block Registry

Rewrite `apps/docs/lib/block-registry.ts`:
- Change categories to match new blocks: `auth`, `sidebar`, `dashboard`, `data-table`, `settings`, `pricing`, `hero`, `feed`, `app-shell`, `empty-state`
- Each block gets ONE entry (not one per variant) with a `variants` array field
- Add `defaultVariant` field to each entry
- Example registry entry:
  ```typescript
  {
    name: 'Auth Block',
    slug: 'auth',
    category: 'auth',
    description: 'Unified authentication forms for login and signup flows.',
    variants: ['login-standard', 'login-otp', 'login-magic', 'login-social', 'signup-standard', 'signup-social'],
    defaultVariant: 'login-standard',
    code: `<AuthBlock variant="login-standard" />`,
  }
  ```

- When variant changes, re-render the block with the new variant prop
- Update default props: sidebar blocks still need `groups={defaultSidebarGroups}`
- Dashboard/AppShell blocks may need `sidebarGroups` defaults too
