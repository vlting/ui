# Client-Side Search with Flexsearch

Add a client-side search feature to the docs site that searches across components, blocks, charts, and icons by name.

## Context
- Docs site: `examples/docs/` with Next.js 15 App Router
- Existing registries: `lib/registry.ts` (57 components), `lib/block-registry.ts` (30 blocks), `lib/chart-registry.ts` (6 charts), `lib/icon-data.ts` (1689 icons)
- Existing navigation: `lib/navigation.ts` (all routes)
- Site header: `components/site-header.tsx` (sticky top bar with nav links)
- Need to install flexsearch as a dependency

## Instructions

### 1. Install flexsearch
```bash
cd examples/docs && yarn add flexsearch
```
Also add `@types/flexsearch` as a devDependency if needed for types. If types aren't available on npm, create a minimal type declaration.

### 2. Create search index builder (`examples/docs/lib/search-index.ts`)
- Import data from all registries (registry, block-registry, chart-registry, icon-data)
- Build a search index at module scope containing all searchable items:
  ```ts
  interface SearchItem {
    title: string     // display name (e.g., "Button", "Login 01", "Area Chart", "RiSearchLine")
    href: string      // link to the page
    category: 'component' | 'block' | 'chart' | 'icon'
    description?: string
  }
  ```
- Export:
  - `searchItems: SearchItem[]` — the full flat list
  - `search(query: string): SearchItem[]` — filters items by query (case-insensitive substring match on title)
  - Keep it simple: substring match is fine for ~1800 items. If flexsearch integration is too complex for client-side, fall back to simple string filtering. The key requirement is that it's fast and works client-side.

### 3. Create search dialog component (`examples/docs/components/search-dialog.tsx`)
- **'use client'** component
- A modal/dialog overlay triggered by:
  - A search button in the site header (magnifying glass icon + "Search..." text + Ctrl+K shortcut hint)
  - `Ctrl+K` / `Cmd+K` keyboard shortcut
- Dialog contents:
  - Search input at top with auto-focus
  - Results list below grouped by category (Components, Blocks, Charts, Icons)
  - Each result shows: title, category badge, description snippet
  - Click a result → navigate to its page (use Next.js router.push) and close the dialog
  - Keyboard navigation: arrow keys to move between results, Enter to select
  - Show max ~20 results initially, with a count of total matches
  - Empty state when no query: show recent/popular items or a prompt "Type to search..."
  - Empty state when no results: "No results for '{query}'"
- Styling: Use inline styles or Tailwind classes consistent with the existing site
- The dialog should have a semi-transparent backdrop
- Close on Escape, backdrop click, or navigation

### 4. Integrate into site header (`examples/docs/components/site-header.tsx`)
- Add a search trigger button between the nav links and the brand switcher/theme toggle
- The button should show a search icon and "Search..." text with `Ctrl+K` hint
- On mobile (where the nav is hidden), show just the search icon
- Clicking opens the SearchDialog
