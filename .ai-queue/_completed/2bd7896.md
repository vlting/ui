# Polish ToggleGroup visual quality and complete docs content

## Instructions

### Context
The ToggleGroup component currently has basic functional styling but lacks the visual polish expected of a production design system. The docs page also has minimal content.

### Visual Improvements for Toggle/ToggleGroup

Read `DESIGN_CONSTITUTION.md` before making changes. Reference shadcn's Toggle/ToggleGroup for quality targets.

In `packages/components/Toggle/Toggle.tsx`:

1. **Improve border styling**: Use 2px border with `$borderColor` for better visual weight. Add a subtle background on the default state (e.g., `backgroundColor: '$color2'`).

2. **Improve pressed/active state**: Instead of just `$color10` background, use a layered approach — lighter background (e.g., `$color4`) with darker text to create depth.

3. **Add hover state**: Add `hoverStyle` with subtle background shift (e.g., `backgroundColor: '$color3'`).

4. **Improve ToggleGroup spacing**: Ensure toggle items in a group have consistent spacing with no gap between them (connected look, like shadcn's grouped toggle). Items at the edges should have rounded corners only on the outer sides.

5. **Improve focus ring**: Use a softer focus ring with `outlineOffset: 2` and semi-transparent outline color.

### Docs Content

In `apps/docs/lib/registry.ts`, update the `toggle-group` entry to include:
- `whenToUse`: "Use to let users switch between mutually exclusive options, like view modes (grid/list) or text formatting (bold/italic/underline)."
- `whenNotToUse`: "Don't use for navigation or actions with side effects. Use Tabs for content switching or RadioGroup for form selections."
- `a11yNotes`: "Uses role='group' with aria-label. Each toggle uses aria-pressed. Supports keyboard navigation with arrow keys."

In `apps/docs/components/component-examples.tsx`, improve the toggle-group example to show:
- A proper toggle group with icon-like labels (B, I, U for bold/italic/underline)
- Show both single and multiple selection modes if the component supports it
