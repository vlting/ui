## Audit Findings

### VisuallyHidden
- Missing `clip: 'rect(0, 0, 0, 0)'`, `clipPath: 'inset(50%)'`, and `whiteSpace: 'nowrap'`. Content may still be visible.

### Separator
- Missing `aria-orientation` for vertical separators.

### Skeleton
- No `aria-hidden="true"` — loading placeholders are announced by screen readers.

### Progress
- No `aria-label` prop — screen readers can't identify what the progress represents.

### Alert
- `role="alert"` (assertive) for all variants — too aggressive for informational alerts.

### Breadcrumb
- No `focusStyle` on links — keyboard users can't see focused link.

### Avatar
- `accessibilityRole="image"` (RN) may not produce `role="img"` on web.

### Tabs (styled)
- Styled trigger `_StyledTrigger` has `focusStyle`/`hoverStyle` but they may not apply to the actual rendered headless `<button>`.

## Required Changes

### 1. VisuallyHidden — complete sr-only technique
Apply via `style` prop: `clip: 'rect(0, 0, 0, 0)'`, `clipPath: 'inset(50%)'`, `whiteSpace: 'nowrap'`, `border: 0`, `padding: 0`, `margin: -1`.

### 2. Separator — add aria-orientation
Pass `aria-orientation={orientation}` to the rendered element.

### 3. Skeleton — add aria-hidden
Add `aria-hidden="true"` to the skeleton frame.

### 4. Progress — add aria-label prop
Accept `aria-label` prop and pass through. Optionally add `aria-valuetext={`${value}% complete`}`.

### 5. Alert — differentiate live region behavior
Use `role="status"` (polite) for default variant, `role="alert"` (assertive) for destructive.

### 6. Breadcrumb — add focus styles
Add `focusStyle` with outline ring to `BreadcrumbLink`.

### 7. Avatar — add web role
Add `role="img"` alongside `accessibilityRole="image"`. Ensure `alt` on image.

### 8. Tabs — fix styled trigger styles
Ensure `_StyledTrigger` wraps or replaces the headless button so `focusStyle`/`hoverStyle` apply. Recommended: make `_StyledTrigger` the rendered element with `tag: 'button'` and move ARIA attributes from headless to styled.

## Verification

- Build: `yarn build`
- VisuallyHidden: Invisible visually but readable by screen readers
- Skeleton: Not announced by screen readers
- Progress: Label announced
- Breadcrumb: Links show focus ring on Tab
- Tabs: Trigger shows hover and focus styles
