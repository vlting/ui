# Block Documentation Route + Registry

Build the block documentation system with a registry, dynamic route, and preview component.

## Context
- 30 blocks exist across 5 categories: login (5), signup (5), sidebar (16), dashboard (1), originals (3)
- Block source is on `epic/block-system` branch, NOT on `epic/docs-site` or `main`
- Therefore blocks CANNOT be imported for live rendering — use code examples + description only
- The component registry pattern from `examples/docs/lib/registry.ts` should be followed
- The dynamic route pattern from `examples/docs/app/docs/components/[name]/page.tsx` should be followed
- Specs can be read from `origin/epic/block-system` branch via `git show` for descriptions

## Instructions

### 1. Create `examples/docs/lib/block-registry.ts`
Define a `BlockEntry` interface and registry array with metadata for all 30 blocks:

```typescript
export interface BlockEntry {
  name: string        // e.g. "Login 01"
  slug: string        // e.g. "login-01"
  category: 'login' | 'signup' | 'sidebar' | 'dashboard' | 'originals'
  description: string // brief description from spec
  code: string        // representative JSX code example
}
```

Block entries (30 total):
- **Login** (login-01 through login-05): Login01=simple centered, Login02=email verification/OTP, Login03=magic link, Login04=biometric/passwordless, Login05=social OAuth only
- **Signup** (signup-01 through signup-05): Signup01=standard centered, Signup02=multi-step, Signup03=profile picture upload, Signup04=invite-based, Signup05=social-first
- **Sidebar** (sidebar-01 through sidebar-16): Sidebar01=simple nav, Sidebar02=collapsible sections, Sidebar03=nested submenus, Sidebar04=floating variant, Sidebar05=tree line, Sidebar06=tabs-based, Sidebar07=icon-only compact, Sidebar08=inset, Sidebar09=collapsible nested, Sidebar10=popover-triggered, Sidebar11=file tree, Sidebar12=calendar, Sidebar13=dialog-triggered, Sidebar14=right-side, Sidebar15=dual sidebars, Sidebar16=sticky header
- **Dashboard** (dashboard-01): Analytics dashboard with metrics + charts
- **Originals** (mobile-tab-layout, master-detail, app-shell-responsive): Cross-platform layout blocks

For code examples, write simple representative JSX showing the import and basic usage pattern. Since blocks can't be imported yet, keep examples as static strings:
```tsx
code: `import { Login01 } from '@vlting/ui/blocks'

<Login01
  onSubmit={(data) => console.log(data)}
  socialProviders={['google', 'github']}
/>`
```

Export helpers: `getBlock(slug)`, `getAllBlocks()`, `getBlocksByCategory(category)`

### 2. Create `examples/docs/app/docs/blocks/[name]/page.tsx`
Dynamic server component route, following the pattern from `[name]/page.tsx` for components:
- `generateStaticParams()` returns all 30 block slugs
- `generateMetadata()` for page titles
- Page sections:
  - **Header**: Block name, category badge, description
  - **Preview placeholder**: A styled container with a message: "Live preview coming soon — block components are being built in a parallel epic."
  - **Code Example**: Render the code string using the `CodeBlock` component from `examples/docs/components/code-block.tsx`
  - **Category info**: Brief note about the category (login, signup, sidebar, etc.)

### 3. Create `examples/docs/components/block-preview.tsx`
A simple preview container for blocks. Since blocks can't be rendered live yet:
- Show a styled placeholder box with the block name and a "preview coming soon" message
- Use Tailwind for layout (consistent with existing docs components)
- Accept `name` and optional `description` props
