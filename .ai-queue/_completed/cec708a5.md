# Task: Fix ARIA/a11y for interactive UI components

## Scope
Fix WCAG A violations in these component directories under `packages/components/`:
- Accordion, Button, ButtonGroup, Breadcrumb, Carousel, Checkbox, Pagination, RadioGroup, Slider, Switch, Tabs, Toggle

## Instructions

1. **Read each component's source** (the main `.tsx` file in each directory).

2. **Fix common WCAG A issues:**
   - Missing `role` attributes (e.g., `role="tablist"`, `role="tab"`, `role="tabpanel"`)
   - Missing `aria-label` or `aria-labelledby` on interactive elements
   - Missing `aria-expanded`, `aria-selected`, `aria-checked`, `aria-pressed` state attributes
   - Missing `aria-controls` / `aria-describedby` relationships
   - Button elements that render as `<div>` or `<span>` — add `role="button"` and `tabIndex={0}`
   - Missing keyboard handlers: `onKeyDown` for Enter/Space on custom buttons, Arrow keys for RadioGroup/Tabs
   - Checkbox/Switch: ensure `role="checkbox"` or `role="switch"` with `aria-checked`
   - Slider: ensure `role="slider"` with `aria-valuemin`, `aria-valuemax`, `aria-valuenow`
   - Carousel: ensure `role="region"` with `aria-roledescription="carousel"`, slides with `aria-label`
   - Pagination: ensure `nav` landmark with `aria-label="pagination"`

3. **Do NOT:**
   - Add color-contrast fixes (separate concern)
   - Modify test files
   - Touch files outside the listed component directories
   - Over-engineer — only add what WCAG A requires

4. **Patterns to follow:**
   - If the component uses Radix UI primitives, they likely already handle ARIA. Check and augment only if missing.
   - If using STL styled components, ARIA attributes pass through — add them to the JSX.
   - For compound components (Root/Trigger/Content pattern), check each sub-component.

## Acceptance
- All listed components have proper ARIA roles and states
- Interactive elements are keyboard-accessible (focusable, activatable)
- No new TypeScript errors introduced
