# Stage 2, Segment 3: Combobox, ContextMenu & DropdownMenu — Native Buttons

## Context

These three components use `ViewJsx` with `onPress` + ARIA roles for interactive elements (triggers and menu items). They need native `<button>` elements for keyboard accessibility.

**Epic branch:** `fix/quality-audit`
**Feature flag:** `quality_audit`
**Parent issue:** #3 (Epic), #5 (Stage 2)

## Scope

- `packages/components/Combobox/Combobox.tsx`
- `packages/components/ContextMenu/ContextMenu.tsx`
- `packages/components/DropdownMenu/DropdownMenu.tsx`

## Instructions

### Read First
- `AI_CONSTITUTION.md` — especially §2.8 (Semantic HTML)
- `packages/FRONTEND_QUALITY.contract.md` — §1 (Rendered Markup), §2 (Accessibility)
- The existing `Toggle` component as a reference for `styledHtml('button')` usage
- The existing DropdownMenu component — it already has the BEST keyboard implementation among these three (has `tabIndex`, `onKeyDown`, `focusVisibleStyle` on some elements). Use its patterns as a reference.

### Changes Required

#### Combobox.tsx

1. **Fix trigger** — Currently `ViewJsx` with `role="combobox"` + `onPress`:
   - The trigger wraps an input, so this may stay as a container — but ensure the trigger container is focusable
   - If the trigger acts as a button (opens the dropdown), replace with `styledHtml('button')`
   - Add `type="button"`, appropriate ARIA attributes
   - Add `focusVisibleStyle`

2. **Fix option items** — Currently `ViewJsx` with `role="option"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Keep `role="option"` (since they're inside a listbox context)
   - Add `focusVisibleStyle`
   - Remove manual `role="option"` only if the native semantics are sufficient

#### ContextMenu.tsx

1. **Fix Item** — Currently `ViewJsx` with `role="menuitem"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Keep `role="menuitem"` (menu context requires it)
   - Add `focusVisibleStyle`

2. **Fix CheckboxItem** — Currently `ViewJsx` with `role="menuitemcheckbox"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Keep `role="menuitemcheckbox"`
   - Add `focusVisibleStyle`

#### DropdownMenu.tsx

1. **Fix Trigger** — Currently `ViewJsx` with implicit button role + `tabIndex={0}` + `onKeyDown`:
   - Replace with `styledHtml('button')` element
   - Add `type="button"`
   - Keep existing `focusVisibleStyle` (already has it!)
   - Keep existing keyboard handlers
   - Remove manual `tabIndex={0}` — native `<button>` is focusable by default

2. **Fix Item** — Currently `ViewJsx` with `role="menuitem"` + `onPress` + `tabIndex={-1}`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Keep `role="menuitem"`, keep `tabIndex={-1}` (menu items use roving tabindex)
   - Keep existing `focusVisibleStyle` and keyboard handlers

3. **Fix CheckboxItem** — Same pattern as Item:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Keep `role="menuitemcheckbox"`, keep `tabIndex={-1}`
   - Keep existing keyboard handlers

### Important Constraints

- Do NOT change visual appearance — only the underlying HTML element
- Do NOT fix token discipline issues (hardcoded pixels) — that's Stage 4
- Do NOT add keyboard navigation (arrow keys) to ContextMenu or Combobox — only ensure the elements themselves are keyboard-operable
- Preserve existing keyboard handlers in DropdownMenu — it already has good patterns
- Use `styledHtml('button')` from `tamagui`
- Use `as any` pattern on styled options if Tamagui v2 RC type bugs appear
- All new `<button>` elements need `type="button"`
- `styledHtml('button')` elements inherently reset browser button styles — you may need to add `background: 'transparent'`, `border: 'none'`, `padding: 0` or similar resets
