### Special Notes

- **Pagination**: Use `<nav aria-label="Pagination">` wrapper. Current page marked with `aria-current="page"`. Links, not buttons (navigates to a new view).
- **Popover**: If contains interactive content → non-modal dialog (`role="dialog"`). If informational → consider tooltip pattern. Focus management depends on modal vs non-modal.
- **Progress**: Use native `<progress>` element or `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`. Must have accessible label.
- **RadioGroup**: `role="radiogroup"` with `role="radio"` items. Arrow keys move selection. Only one radio is tabbable at a time (roving tabindex).
- **Select**: Custom select MUST provide equivalent keyboard behavior to native `<select>`. Consider offering NativeSelect as the accessible-by-default option and this as the styled alternative.
- **Sheet**: Bottom sheet / side sheet — variant of Dialog/Drawer. If modal: trap focus, Escape dismisses. Drag-to-dismiss must not be the ONLY way to close.
- **Sidebar**: Use `<aside>` or `<nav>` landmark. If collapsible, the toggle must be keyboard-accessible and announce expanded/collapsed state.
- **Slider**: MUST support keyboard: Arrow keys for ±1 step, Page Up/Down for larger increments, Home/End for min/max. `role="slider"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-valuetext`.
- **Switch**: Render as `<button role="switch" aria-checked="true|false">` or use native `<input type="checkbox" role="switch">`. Space toggles. Must not look like a checkbox.
- **Table**: Use native `<table>`, `<thead>`, `<tbody>`, `<th scope="col|row">`, `<td>`. Caption via `<caption>` or `aria-labelledby`. Sortable columns need `aria-sort`.
- **Textarea**: Native `<textarea>` element. Must have associated `<label>`. Support `aria-invalid`, `aria-describedby` for error/helper text.
- **Toast**: Use `role="status"` or `aria-live="polite"`. Auto-dismiss must be pausable on hover/focus (WCAG 2.2.1). Must not steal focus. Action buttons within toast must be keyboard-reachable.
- **Toggle**: Use `aria-pressed="true|false"` on a `<button>`. NOT a checkbox — toggles are buttons that maintain pressed state.
- **Tooltip**: Non-interactive — must never contain focusable elements. Triggered by hover AND focus. Escape dismisses. Use `role="tooltip"` with `aria-describedby` on the trigger.
- **Typography**: Utility component for text styling. Ensure it renders appropriate semantic elements (not always `<span>`).
