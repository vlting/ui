# Task: Add Shiki highlighting to component preview code sections

## Context
The `ComponentPreview` component (`apps/docs/components/component-preview.tsx`) renders code examples in a collapsible section using plain `<pre>{code}</pre>` — no syntax highlighting. The `CodeBlock` component (`apps/docs/components/code-block.tsx`) already has Shiki integration with light/dark theme support. The component preview should use it.

## Files to modify
- `apps/docs/components/component-preview.tsx` — update code rendering
- `apps/docs/components/component-examples.tsx` — if code is passed here, update accordingly

## Implementation

1. Read `apps/docs/components/code-block.tsx` to understand the Shiki CodeBlock API (it's a server component using `codeToHtml`)

2. Since ComponentPreview is a client component ('use client') and CodeBlock is a server component, you CANNOT directly use CodeBlock inside ComponentPreview. Instead:
   - Option A: Pre-render the code HTML server-side and pass it as a prop
   - Option B: Create a client-side Shiki highlighter wrapper using `shiki/bundle/web` or `shikiji`
   - Option C: Move the code rendering to the parent (component-examples.tsx or the page) where it can be server-rendered

3. The simplest approach: In `component-examples.tsx` (which calls ComponentPreview), pass pre-highlighted code HTML instead of raw code strings. Use the same Shiki themes (`github-light-default` / `github-dark`) and the same dual-render approach (light + dark).

4. Ensure the code section has:
   - Proper padding (`p-4` or similar)
   - Right padding sufficient to not overlap the copy button (`pr-12` or similar)
   - Rounded corners matching the container
   - Overflow-x scroll for long lines

- Highlighting works in both light and dark mode
