# Segment 5: Migrate Block + Chart Pages

## Objective
Migrate hardcoded Tailwind color classes to semantic theme tokens in block and chart documentation pages.

## Context
Segment 1 established semantic CSS custom properties. This segment migrates the content pages for blocks and charts.

Current state:
- `app/docs/blocks/[name]/page.tsx` — `bg-blue-100 text-blue-800 dark:bg-blue-900/30 dark:text-blue-300` (badge), `text-gray-600 dark:text-gray-400`
- `app/docs/charts/[type]/page.tsx` — `text-gray-600 dark:text-gray-400`, `text-green-500` (CONTRAST ISSUE), `text-red-500` (CONTRAST ISSUE), `border-blue-200 dark:border-blue-800`, `bg-blue-50 dark:bg-blue-950/30`, `text-blue-800 dark:text-blue-300`

## Instructions

### Migration Pattern

| Old pattern | New pattern |
|------------|-------------|
| `text-gray-600 dark:text-gray-400` | `text-foreground-secondary` |
| `border-gray-200 dark:border-gray-800` | `border-border` |

### Contrast Fixes

1. **charts/[type]/page.tsx**: `text-green-500` (#22c55e) on white only achieves ~2.2:1 — **FAILS WCAG AA**. Replace with `text-success` token (green-600 #16a34a → ~4.5:1 on white, passes).
2. **charts/[type]/page.tsx**: `text-red-500` (#ef4444) on white only achieves ~3.9:1 — **FAILS WCAG AA for normal text**. Replace with `text-destructive` token (red-600 #dc2626 → ~4.6:1, passes).

### Blue Info Boxes (Keep Semantic)

The blue info/note boxes in charts page (`border-blue-200 dark:border-blue-800`, `bg-blue-50 dark:bg-blue-950/30`, `text-blue-800 dark:text-blue-300`) serve a semantic purpose (info callouts). These can stay as blue-themed but should be checked for contrast:
- `text-blue-800` (#1e40af) on `bg-blue-50` (#eff6ff): ~8.3:1 ✓ passes AA
- `text-blue-300` (#93c5fd) on `bg-blue-950/30`: verify contrast

If the dark mode blue combination doesn't meet 4.5:1, adjust to `text-blue-200` on the dark background.

### Category Badges (Keep Semantic)

The badge in blocks page (`bg-blue-100 text-blue-800 dark:bg-blue-900/30 dark:text-blue-300`) is a category indicator. Keep the blue semantic color but verify contrast. Same approach as above.
