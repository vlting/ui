# Segment 1: createIcon Factory

Create the shared icon factory that converts SVG path data into `IconFC`-compatible React components using `react-native-svg`.

## Scope

- `packages/icons/createIcon.tsx` (create)

## Instructions

Create the file `packages/icons/createIcon.tsx` with the following implementation:

```tsx
import { memo, type ComponentType } from 'react'
import Svg, { Path } from 'react-native-svg'

/** Icon component type — compatible with @tamagui/lucide-icons and the Icon primitive */
export type IconFC = ComponentType<{ size?: number | string; color?: string }>

/**
 * Factory that produces an IconFC-compatible component from SVG path data.
 * Used by the code generator to create individual icon components.
 */
export function createIcon(pathData: string, displayName: string): IconFC {
  const Icon = memo(({ size = 24, color = 'currentColor' }: { size?: number | string; color?: string }) => (
    <Svg width={size} height={size} viewBox="0 0 24 24" fill={color}>
      <Path d={pathData} />
    </Svg>
  ))
  Icon.displayName = displayName
  return Icon as IconFC
}
```

Key requirements:
- Import `Svg` and `Path` from `react-native-svg` (works cross-platform: web via `react-native-svg-web`, native via `react-native-svg`)
- Use `memo` to prevent unnecessary re-renders (icons are pure — same props = same output)
- Default size `24` matches Remix Icon's 24×24 viewBox
- Default color `currentColor` inherits from parent text color on web; on RN consumers pass explicit color
- Export the `IconFC` type for use by generated files and consumers
- The returned component must be compatible with the existing `IconFC` type in `packages/primitives/Icon.tsx`
- Do NOT import from `packages/primitives` — re-declare the type locally to avoid circular deps
