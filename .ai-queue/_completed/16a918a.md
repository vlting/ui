# Task: Add build-output CSS source-order verification

## Files to create
- `packages/stl/src/__tests__/dark-mode-css-order.test.ts`

## Context
The dark mode CSS rule `[data-color-mode="dark"]` MUST appear after the `:root` rule in the compiled CSS. Both have specificity `(0,1,0)` so source order is the tiebreaker. If vanilla-extract ever reorders output, dark mode silently breaks.

Since the vanilla-extract build requires a full Vite build pipeline (not available in unit tests), we verify source order by reading the source file and checking that the dark mode `globalStyle` call appears after the `:root` `globalStyle` call.
