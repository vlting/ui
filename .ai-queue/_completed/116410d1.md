# Task: Fix ARIA/a11y for form components, primitives, and data display

## Scope
Fix WCAG A violations in:

**Form components** (`packages/components/`):
- Input, Textarea, InputOTP, InputGroup, Field, Form, DatePicker, Calendar, NativeSelect, Label, Progress

**Primitives** (`packages/primitives/`):
- All primitives: AspectRatio, Badge, Box, Divider, Heading, Icon, Kbd, Label, Portal, Separator, Skeleton, Spacer, Spinner, Stack, Text, VisuallyHidden

**Data display components** (`packages/components/`):
- Avatar, Card, DataTable, Table, Empty, Loader, Resizable, ScrollArea, Separator, Typography

## Instructions

1. **Read each component's source** (the main `.tsx` file).

2. **Fix form component ARIA:**
   - Input/Textarea: ensure `aria-invalid` when error state, `aria-required` when required, `aria-describedby` for help text
   - Field: proper `<label>` → `<input>` association via `htmlFor`/`id`
   - InputOTP: `role="group"` with `aria-label`, individual inputs with `aria-label="digit N"`
   - InputGroup: group with accessible label
   - DatePicker/Calendar: proper grid roles (`role="grid"`, `role="gridcell"`), `aria-selected` for selected dates
   - NativeSelect: ensure `aria-label` when no visible label
   - Progress: `role="progressbar"` with `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-label`
   - Label: ensure it's an actual `<label>` element

3. **Fix primitive ARIA:**
   - Spinner: `role="status"` with `aria-label="Loading"` (or `aria-live="polite"`)
   - Skeleton: `aria-hidden="true"` (decorative) or `aria-busy="true"` on parent
   - Icon: if decorative → `aria-hidden="true"`; if meaningful → `aria-label`
   - Separator/Divider: `role="separator"` (if using `<div>`)
   - Heading: ensure proper `<h1>`-`<h6>` semantic elements (not `<div>` with styles)
   - Badge: if informational, ensure it's announced (not `aria-hidden`)
   - Kbd: use `<kbd>` element
   - VisuallyHidden: verify it uses proper SR-only CSS pattern

4. **Fix data display ARIA:**
   - Table/DataTable: proper `<table>`, `<thead>`, `<tbody>`, `<th scope="col">` semantics
   - Avatar: `alt` text on `<img>`, fallback `aria-label` on initial-only variant
   - Empty: proper heading structure
   - Loader: `role="status"` with `aria-label`
   - ScrollArea: `tabIndex={0}` for keyboard scrolling, `role="region"` with `aria-label`
   - Resizable: proper `role="separator"` on handles with `aria-orientation`

5. **Do NOT:**
   - Add color-contrast fixes
   - Modify test files or Playwright spec
   - Touch overlay/menu/interactive components (covered by other tasks)

## Acceptance
- Form inputs properly expose state (invalid, required, described)
- Label-input associations work for screen readers
- Primitives use semantic HTML or correct ARIA roles
- Data display components are properly structured for AT
