## Instructions

### 1. `InputGroup.tsx`

Create a compound component with these sub-components:

**InputGroup.Root** â€” Wrapper that applies border-radius collapsing. Props:
- `children: ReactNode`
- `orientation?: 'horizontal' | 'vertical'` (default: `'horizontal'`)
- `size?: 'sm' | 'md' | 'lg'` (default: `'md'`)

Implementation: Use the same CSS injection pattern as ButtonGroup for border-radius collapsing. Use a unique class name like `vlting-input-group-h` / `vlting-input-group-v`. The CSS should:
- Strip border-radius on inner edges of adjacent children
- Use `-1px` margin to overlap borders (same as ButtonGroup)
- Use `display: inline-flex` for the container

**InputGroup.Addon** â€” Non-input slot for static text, icons, or labels. Styled with `$color3` background, matching border, and border-radius from the group's size. Props:
- `children: ReactNode`

```typescript
// @ts-expect-error Tamagui v2 RC
const InputGroupAddon = styled(XStack, {
  name: 'InputGroupAddon',
  alignItems: 'center',
  justifyContent: 'center',
  paddingHorizontal: '$2',
  backgroundColor: '$color3',
  borderWidth: 1,
  borderColor: '$borderColor',
  variants: {
    size: {
      sm: { borderRadius: '$3', paddingHorizontal: '$1.5' },
      md: { borderRadius: '$4', paddingHorizontal: '$2' },
      lg: { borderRadius: '$5', paddingHorizontal: '$2.5' },
    },
  } as const,
  defaultVariants: {
    size: 'md',
  },
})
```

**InputGroup.Element** â€” Absolutely-positioned overlay element (e.g., icon inside the input). Props:
- `children: ReactNode`
- `placement?: 'left' | 'right'` (default: `'left'`)

```typescript
// @ts-expect-error Tamagui v2 RC
const InputGroupElement = styled(XStack, {
  name: 'InputGroupElement',
  position: 'absolute',
  top: 0,
  bottom: 0,
  alignItems: 'center',
  justifyContent: 'center',
  paddingHorizontal: '$2',
  zIndex: 1,
  pointerEvents: 'none',
  variants: {
    placement: {
      left: { left: 0 },
      right: { right: 0 },
    },
  } as const,
  defaultVariants: {
    placement: 'left',
  },
})
```

**InputGroup.Input** â€” A thin re-export/wrapper of the existing Input component, ensuring it inherits the group's size context. Use React context to pass `size` from Root to children.

Create a `InputGroupContext` with `React.createContext`:
```typescript
const InputGroupContext = React.createContext<{ size: 'sm' | 'md' | 'lg' }>({ size: 'md' })
```

Root provides the context. Addon, Element, and Input consume it for size.

Assemble with `withStaticProperties`:
```typescript
export const InputGroup = withStaticProperties(InputGroupRoot, {
  Addon: InputGroupAddon,
  Element: InputGroupElement,
  Input: InputGroupInput,
})
```

Export types: `InputGroupProps`, `InputGroupAddonProps`, `InputGroupElementProps`, `InputGroupInputProps`.

### 3. `InputGroup.spec.md`

Follow the spec template. Key sections:
- **Purpose:** Group an input with addons (text labels, icons, buttons) into a visually connected unit
- **When to use:** URL inputs with protocol addon, search with icon, inputs with action buttons
- **When NOT to use:** Simple label + input (use Input), multiple unrelated inputs (use a form layout)
- **Anatomy:** Root (wrapper with radius collapsing), Addon (bordered slot), Element (overlay slot), Input (the input itself)
- **Behavior:** Context provides size to all children. Border-radius collapsing via CSS.
- **Accessibility:** Group uses `role="group"`, Addon content is decorative or labeling (use `aria-label` on Root if needed)
- **Cross-platform:** Works on web. React Native fallback: plain XStack layout without CSS radius collapsing.
