# Fix chart rendering errors and radial chart empty preview

## Instructions

### Problem
Charts on the docs site error on page load and the radial chart preview shows empty content.

### Root Cause Analysis
1. **RadialChart data format mismatch**: The sample data in `chart-preview.tsx` uses `{ x: 'Progress', y: 75, fill: '#3b8fdb' }` but `RadialChart.tsx` expects data keyed by the series key name (e.g., `{ Progress: 75 }`). The component iterates `seriesKeys` from the config and looks up `d[k]` — when data uses `y` instead of the series key name, `Number(undefined)` resolves to 0, making arcs invisible.

2. **Other chart data format issues**: Similar mismatch may exist for other chart types. The sample data format in `chart-preview.tsx` needs to match what each Victory-based chart component expects. Victory charts expect data as `[{ x: 'label', y: value }]` format. Check each chart type's sample data matches the expected format.

3. **Width measurement timing**: The `Chart` container uses ResizeObserver and only renders children when `finalWidth > 0`. In the docs preview container, if the parent element starts at 0 width, charts won't render. Ensure the preview container has a minimum width.

### Steps

1. **Fix sample data format for RadialChart** in `apps/docs/app/docs/charts/chart-preview.tsx`:
   - Change `sampleRadialData` from `[{ x: 'Progress', y: 75, fill: '#3b8fdb' }]` to `[{ Progress: 75 }]` (keyed by series name from the config)

2. **Verify and fix sample data for all other chart types** in the same file:
   - BarChart: data should be `[{ x: 'label', y: value }]` (Victory format)
   - LineChart: data should be `[{ x: number, y: number }]`
   - AreaChart: same as LineChart
   - PieChart: data should be `[{ x: 'label', y: value }]`
   - RadarChart: data should be `[{ x: 'category', y: value }]`

3. **Ensure chart configs match the data format** — the `chartConfig` object's series keys must match what each component reads from the data.

4. **Test each chart type by navigating to its docs page** to ensure previews render without console errors.

5. **If width measurement is an issue**, add explicit width or min-width to the chart preview container in `chart-preview.tsx`.
