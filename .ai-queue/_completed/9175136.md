## Part 2: Information Architecture Overhaul

### New navigation structure

Remove all navigation links from the top nav bar. The top bar should ONLY contain:
- Logo/brand name (left)
- Brand selector buttons + theme toggle (right)

All navigation moves to the sidebar with **collapsible sub-nav groups**:

```
Overview (home)
──────────
Primitives
  > All Primitives
──────────
Components
  > Buttons & Actions     [collapsible]
  > Data Display           [collapsible]
  > Forms & Inputs         [collapsible]
  > Menus & Navigation     [collapsible]
  > Overlays               [collapsible]
──────────
Composed
  > All Composed
──────────
Hooks
  > All Hooks
```

Each collapsible group expands to show individual page links. Clicking the group header toggles expand/collapse.

### Collapsible sub-nav implementation

In `BrandLayout.tsx`:
- Replace the current flat `sidebarGroups` array with a nested structure supporting collapsible groups
- Track which groups are expanded in React state
- Use a click handler on group labels to toggle expansion
- Use a smooth height transition for the expand/collapse animation
- Auto-expand the group that contains the currently active route

### Page content: alphabetical ordering

For ALL pages that display multiple component demos in sections, ensure the sections are rendered in **alphabetical order by section title**. For example, on the Menus & Navigation page, sections should appear as:
- Command
- ContextMenu
- DropdownMenu
- Menubar
- NavigationMenu
- Resizable
- ScrollArea
- Sidebar
- Tabs

Apply this to every multi-section page: ComponentsPage, InputsPage, MenusPage, OverlaysPage, TypographyPage, LayoutPage, PrimitivesPage, ComposedPage, HooksPage.

### Routes
Update `App.tsx` routes to match the new IA. If you split pages differently (e.g., "Buttons & Actions" gets its own route), create new page files as needed. Minimize page file count where possible — it's OK to keep a single page per sidebar category (e.g., one page for "Menus & Navigation" containing all menu/nav demos).
