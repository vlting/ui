## Test Coverage

Same categories as task 015. Additionally, specific test guidance for complex P–Z components:

### Slider
- Keyboard: Arrow keys ±1, Page Up/Down ±10, Home/End min/max
- ARIA: `role="slider"`, `aria-valuenow`, `aria-valuemin`, `aria-valuemax`, `aria-valuetext`
- Range slider: two thumbs, independent keyboard control

### Switch
- Renders as button with `role="switch"` or checkbox with `role="switch"`
- `aria-checked` toggles on Space
- Visual state matches `aria-checked`

### Table
- Renders native `<table>`, `<thead>`, `<th>`, `<tbody>`, `<td>`
- Column headers have `scope="col"`, row headers have `scope="row"`
- Sortable: `aria-sort` on active column header
- Caption or `aria-labelledby` present

### Toast
- `role="status"` or `aria-live="polite"`
- Auto-dismiss pauses on hover and focus
- Action buttons keyboard-reachable
- Does NOT steal focus on appearance

### Tooltip
- Appears on focus AND hover of trigger
- `role="tooltip"` with trigger's `aria-describedby`
- Escape dismisses
- No focusable content inside tooltip

### RadioGroup
- `role="radiogroup"` container
- Arrow keys move selection between radios
- Only selected radio is tabbable (roving tabindex)
- Space selects focused radio
