# Example App: Nav Restructure — Sub-headings Instead of Collapsible

## Context

The kitchen-sink example app's left sidebar navigation currently uses a collapsible accordion pattern for the "Components" section. The user does NOT want collapsible sections. Instead, each nav section should show sub-headings listing the individual component names from that page (matching the `<Section title="...">` headings on each page).

## Requirements

### 1. Remove the collapsible pattern

- Remove the `CollapsibleGroup` component entirely
- Remove `expandedGroups` state, `toggleGroup`, `activeGroupLabel`, and related `useEffect`/`useMemo`
- All nav sections should always be visible (no expand/collapse)

### 2. Add component sub-headings under each nav section

The sidebar data structure needs to include sub-items (component names) for each page. These should render as indented links that anchor/scroll to the corresponding section on the page. Use anchor links (e.g., `#button`, `#buttongroup`) — the page components in a separate segment will add corresponding `id` attributes to section headings.

Here are the section titles per page (from actual `<Section title="...">` usage):

**Primitives** (`/primitives`):
AspectRatio, Badge, Box, Divider, Heading, Label, Separator, Skeleton, Spacer, Stack/VStack/HStack, Text, VisuallyHidden

**Buttons & Actions** (`/components/buttons`):
Button, ButtonGroup, Pagination, Toggle

**Data Display** (`/components/data`):
Accordion, Alert, Avatar, Breadcrumb, Card, Carousel, Collapsible, Kbd, Loader, Progress, Table, Typography

**Forms & Inputs** (`/components/forms`):
Calendar, Checkbox, Combobox, DatePicker, DateRangePicker, Form, Input, InputOTP, NativeSelect, RadioGroup, Select, Slider, Switch, Textarea

**Menus & Navigation** (`/components/menus`):
Command, ContextMenu, DropdownMenu, Menubar, NavigationMenu, Resizable, ScrollArea, Sidebar, Tabs

**Overlays** (`/components/overlays`):
AlertDialog, Dialog, Drawers & Panels, HoverCard, Tooltip

**Composed** (`/composed`):
Accordion, AlertDialog, Breadcrumb, Collapsible, Form, Table

**Hooks** (`/hooks`):
useControllableState, useFocusTrap, useKeyboardNavigation

### 3. USE SEMANTIC HTML

This is critically important. The sidebar should use:
- `<nav>` element for the navigation container (with `aria-label="Sidebar navigation"`)
- Semantic heading elements (`<h2>`, `<h3>`) for group labels — NOT `<span>` or `<button>` styled as headings
- `<ul>` / `<li>` for nav item lists
- `<a>` (via React Router `<Link>`) for navigation links

The group label (e.g., "Components", "Primitives") should be a semantic `<h2>`. The page link (e.g., "All Primitives", "Buttons & Actions") should be a `<Link>` (which renders `<a>`). The sub-items (e.g., "Button", "ButtonGroup") should be anchor links within `<li>` elements.

### 4. Visual design

- Group headings: uppercase, small, muted — same visual style as current `CollapsibleGroup` labels but without the button/chevron
- Page links: same style as current (14px, with active state border-right indicator)
- Sub-item links: slightly smaller (12-13px), more indented (left padding ~36px), muted color, lighter weight. On hover, show the page's active color. When the hash matches, show as active.
- Keep the existing mobile sidebar behavior (hamburger toggle, overlay)
