# Add missing simple components for shadcn/ui parity

## Goal

Add components that shadcn/ui has that our library lacks. This segment covers simpler/primitive components that don't require complex state management or external dependencies.

## Components to Add

### 1. Spinner (`packages/primitives/Spinner.tsx` — NEW)

Simple loading spinner animation.
- Props: `size` ('sm' | 'md' | 'lg'), `color` (theme token)
- Uses CSS animation or Tamagui animation for spinning
- Accessible: `role="status"`, `aria-label="Loading"`
- Reference: shadcn uses a simple SVG spinner with rotate animation

### 2. Kbd (`packages/primitives/Kbd.tsx` — NEW)

Keyboard shortcut display component.
- Props: `children` (string), `size` ('sm' | 'md')
- Renders `<kbd>` HTML element with styled border, background, font
- Style: monospace font, subtle border, slightly raised appearance (like a physical key)
- Use `styledHtml('kbd', {...})` from `@tamagui/web` or `styled.kbd({...})` to render semantic `<kbd>` element

### 3. ScrollArea (`packages/components/ScrollArea/ScrollArea.tsx` — NEW dir)

Styled scrollable container with custom scrollbar.
- If `@tamagui/scroll-view` is available, wrap it. Otherwise, use native scrollable div with custom CSS scrollbar styling.
- Props: `orientation` ('vertical' | 'horizontal' | 'both'), `type` ('auto' | 'always' | 'scroll' | 'hover')
- Compound: `ScrollArea.Root`, `ScrollArea.Viewport`, `ScrollArea.Scrollbar`, `ScrollArea.Thumb`, `ScrollArea.Corner`
- Accessible: proper role and keyboard scrolling support
- Style: thin custom scrollbar (4px width), rounded thumb, semi-transparent

### 4. HoverCard (`packages/components/HoverCard/HoverCard.tsx` — NEW dir)

Card that appears on hover (like GitHub user previews).
- Build on top of `@tamagui/popover` (already installed) with hover trigger instead of click.
- Compound: `HoverCard.Root`, `HoverCard.Trigger`, `HoverCard.Content`
- Props: `openDelay` (default 700ms), `closeDelay` (default 300ms)
- Content styled with: card background, border, shadow, border-radius

### 5. NativeSelect (`packages/components/NativeSelect/NativeSelect.tsx` — NEW dir)

Wrapper around native `<select>` element with themed styling.
- Uses `styledHtml('select', {...})` or `styled.select({...})` to style the native element directly
- Props: `value`, `defaultValue`, `onValueChange`, `placeholder`, `disabled`, `size` ('sm' | 'md' | 'lg')
- Renders actual `<select>` with `<option>` children — fully native, accessible by default
- Style: matches our Input component's border/focus/hover states
- Compound: `NativeSelect.Root` (the styled select), `NativeSelect.Option` (native option)

### 6. InputOTP (`packages/components/InputOTP/InputOTP.tsx` — NEW dir)

One-time password input with individual digit slots.
- Props: `maxLength` (number of slots, default 6), `value`, `onComplete`, `disabled`, `pattern` (regex for allowed chars)
- Renders a row of individual input slots
- Each slot: single character, auto-advance to next on input, backspace goes to previous
- Compound: `InputOTP.Root`, `InputOTP.Group`, `InputOTP.Slot`, `InputOTP.Separator`
- Accessible: one hidden input with `inputmode="numeric"` + visual slots
- Style: bordered boxes with focus ring on active slot

### 7. ButtonGroup (`packages/components/ButtonGroup/ButtonGroup.tsx` — NEW dir)

Groups buttons together with connected styling.
- Props: `orientation` ('horizontal' | 'vertical'), `size` ('sm' | 'md' | 'lg'), `variant`
- Removes border-radius from inner edges, collapses borders between buttons
- Uses CSS `> :not(:first-child):not(:last-child)` pattern for middle items
- Compound: `ButtonGroup.Root`, children are regular `Button` components

### 8. Drawer (`packages/components/Drawer/Drawer.tsx` — NEW dir)

Bottom drawer / slide-up panel (like Sheet but with different UX).
- Build on `@tamagui/sheet` (already installed) or `@tamagui/dialog` with bottom positioning
- Compound: `Drawer.Root`, `Drawer.Trigger`, `Drawer.Content`, `Drawer.Header`, `Drawer.Footer`, `Drawer.Title`, `Drawer.Description`, `Drawer.Close`
- Props: `open`, `onOpenChange`, `direction` ('bottom' | 'top' | 'left' | 'right')
- Style: rounded top corners, handle bar, overlay

### 9. Typography (`packages/components/Typography/Typography.tsx` — NEW dir)

Semantic typography components for consistent text styling.
- Exports: `H1`, `H2`, `H3`, `H4`, `P`, `Lead`, `Large`, `Small`, `Muted`, `Blockquote`, `InlineCode`, `List`, `ListItem`
- Each uses the appropriate HTML element (`<h1>`, `<p>`, `<blockquote>`, etc.) via `styledHtml()` or `styled.h1()` etc.
- Styled with our theme tokens (font family, weight, size, color, spacing)
- Matches shadcn/ui's typography scale

## Key Patterns

For ALL new components:
1. Use Tamagui's `styled()` system with theme tokens (`$background`, `$borderColor`, `$color`, etc.)
2. For semantic HTML elements, use `styledHtml('element', {...})` from `@tamagui/web` or the `styled.element({...})` Proxy
3. Include proper ARIA attributes
4. Export types for all props
5. Support `size` variants where applicable
