## Instructions

### 1. Create `packages/components/Chart/LineChart.tsx`

Define and export `LineChartVariant` type inline:
```typescript
export type LineChartVariant = 'default' | 'dots' | 'multiple' | 'labels' | 'stepped' | 'custom-dots' | 'interactive' | 'linear' | 'legend' | 'icons'
```

Define and export `LineChartProps`:
```typescript
export interface LineChartProps {
  data: ChartDataPoint[]
  variant?: LineChartVariant
  config: ChartConfig
  xAxisKey?: string
  showXAxis?: boolean
  showYAxis?: boolean
  showGrid?: boolean
  showDots?: boolean
  showLabels?: boolean
  showTooltip?: boolean
  interactive?: boolean
  showLegend?: boolean
  tooltipVariant?: TooltipVariant
  curveType?: 'natural' | 'linear' | 'step'
  strokeWidth?: number
  dotSize?: number
  customDot?: React.ComponentType<any>
}
```

Import Victory components:
```typescript
import { VictoryChart, VictoryLine, VictoryScatter, VictoryAxis, VictoryVoronoiContainer, VictoryLabel } from 'victory-native'
```

Implementation approach:
- Use `useChartContext()` to get `resolvedColors`, `victoryTheme`, `dimensions`
- Derive variant-specific behavior:
  - `variant === 'default'`: Basic line with `interpolation="natural"` (smooth curve)
  - `variant === 'dots'` OR `showDots === true`: Render `VictoryScatter` overlaid on each `VictoryLine` with matching data and colors. Dot size from `dotSize` prop (default 4).
  - `variant === 'multiple'`: Multiple lines from config — one `VictoryLine` per series key (this is also the default when config has multiple keys)
  - `variant === 'labels'` OR `showLabels === true`: Add `labels={({ datum }: any) => datum.y}` and `labelComponent={<VictoryLabel />}` to VictoryLine/VictoryScatter
  - `variant === 'stepped'`: Set `interpolation="stepAfter"`
  - `variant === 'custom-dots'`: Use `customDot` as `dataComponent` prop on VictoryScatter
  - `variant === 'interactive'` OR `interactive === true`: Use `VictoryVoronoiContainer` as `containerComponent`
  - `variant === 'linear'`: Set `interpolation="linear"` (straight lines between points)
  - `variant === 'legend'` OR `showLegend === true`: Render `<ChartLegend>` below chart
  - `variant === 'icons'`: Pass icons from config to tooltip/legend
- For multi-series: iterate over `Object.keys(config)` to render one `VictoryLine` per series
- Apply `resolvedColors` from context to each series' `style.data.stroke`
- Default `strokeWidth` is 2
- Default `curveType` is `'natural'` unless variant overrides it
- When `showDots` is true, render a `VictoryScatter` for each series, overlaid after the corresponding `VictoryLine`, using matching data and `style.data.fill` from resolvedColors

**CRITICAL patterns:**
- `theme={victoryTheme as any}` — Victory's types are strict
- For VictoryLine, the color goes to `style.data.stroke` (not `fill`)
- For VictoryScatter (dots), the color goes to `style.data.fill`
- `interpolation` prop values: `"natural"` (smooth), `"linear"` (straight), `"stepAfter"` (stepped)
- When both line and dots are rendered, dots should be rendered AFTER lines so they appear on top
