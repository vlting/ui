### Standard focusVisibleStyle Pattern

```tsx
focusVisibleStyle: {
  outlineWidth: 2,
  outlineOffset: 2,
  outlineColor: '$outlineColor',
  outlineStyle: 'solid',
}
```

For items inside containers (menu items, list items), use `outlineOffset: -2` instead of `2` to keep the outline inside the container.

### Changes Required

#### Card.tsx
- The interactive variant currently uses `focusStyle` — change to `focusVisibleStyle`
- `focusVisibleStyle` only shows the outline on keyboard focus, not mouse clicks (correct behavior)
- Keep the same outline styling, just change the property name

#### Command.tsx
- The `Item` component has `onPress`, `role="option"`, `cursor="pointer"` but NO focus styling
- Add `focusVisibleStyle` with the standard pattern (use `outlineOffset: -2` since items are inside a list)

#### InputOTP.tsx
- The `Slot` component is interactive but lacks focus styling
- Add `focusVisibleStyle` with the standard pattern
- Keep any existing active/selected state styling (boxShadow etc.) — `focusVisibleStyle` is additive

#### Breadcrumb.tsx

- If the Link uses `styledHtml('a')` or a Tamagui component, add `focusVisibleStyle` directly
- If it uses raw HTML, keep the CSS approach but ensure it targets `:focus-visible` (not `:focus`)

#### NativeSelect.tsx
- Uses `styledHtml('select')` — add `focusVisibleStyle` to the styled component definition
- The standard outline pattern works on native `<select>` elements
