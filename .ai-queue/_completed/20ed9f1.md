### 2. Select & Input — Horizontal padding proportions

**Problem:** Horizontal padding on Select and Input is way too large. It should "look" roughly the same as vertical padding (accounting for line-height making vertical appear taller). Rule of thumb: if vertical padding is 8px, horizontal can be ~10px.

**File:** `packages/components/Select/Select.tsx`

**Current Select `SIZE_PADDING_MAP`:**
```ts
{
  sm: { h: '$2', v: '$1', fontSize: '$2' },    // h=$2 ≈ 16px, v=$1 ≈ 10px
  md: { h: '$3', v: '$2', fontSize: '$3' },    // h=$3 ≈ 20px, v=$2 ≈ 16px
  lg: { h: '$4', v: '$2.5', fontSize: '$4' },  // h=$4 ≈ 28px, v=$2.5 ≈ 18px
}
```

These horizontal paddings are way too large relative to vertical. Fix to tight, proportional values:

```ts
const SIZE_PADDING_MAP = {
  sm: { h: '$1.5', v: '$1', fontSize: '$2' },   // h≈12px, v≈10px — close visual parity
  md: { h: '$2', v: '$1.5', fontSize: '$3' },   // h≈16px, v≈12px — close visual parity
  lg: { h: '$2.5', v: '$2', fontSize: '$4' },   // h≈18px, v≈16px — close visual parity
}
```

**File:** `packages/components/Input/Input.tsx`

Input currently uses just `size={sizeToken}` and relies on Tamagui's default Input padding. Check what Tamagui Input renders by default. If its horizontal padding is also too large, override it.

The Input component doesn't set explicit padding — it passes `size` to Tamagui's `<Input>` which has its own internal sizing. To control padding explicitly, add padding props to `StyledInputJsx`:

Add a size context or explicit padding overrides. The simplest approach: extend `StyledInput` with size-dependent padding variants similar to Select. Or pass explicit `paddingHorizontal` / `paddingVertical` based on size.

The Input's padding should match Select's padding so they look consistent side-by-side.

### 3. Select & Input — Size-dependent border radius

**Problem:** Border radius should be slightly smaller on smaller sized fields.

**Files:** `packages/components/Select/Select.tsx`, `packages/components/Input/Input.tsx`

**Current:** Both use a fixed `borderRadius: '$4'` (or Tamagui Input's default).

**Fix — Select:** Add size-dependent radius to the trigger:
```ts
const SIZE_RADIUS_MAP = { sm: '$3', md: '$4', lg: '$5' }
```
Apply: `borderRadius={SIZE_RADIUS_MAP[size]}` on the SelectTrigger.

**Fix — Input:** Add a radius variant to `StyledInput`:
```ts
const StyledInput = styled(TamaguiInput, {
  variants: {
    error: { true: { borderColor: '$red10' } },
    fieldSize: {
      sm: { borderRadius: '$3' },
      md: { borderRadius: '$4' },
      lg: { borderRadius: '$5' },
    },
  } as const,
})
```
Pass `fieldSize={size}` to the input (use a different name than `size` to avoid conflicting with Tamagui's built-in `size` prop).

### 4. Switch — Knob still sticking out

**Problem:** The Switch thumb (knob) is STILL sticking out at the bottom. The current code has `padding={2}` on the track but the knob is still too big.

**File:** `packages/components/Switch/Switch.tsx`

**Current state:**
```tsx
<SwitchFrame
  size={SIZE_MAP[size]}  // e.g., '$4' for md
  padding={2}
  ...
>
  <SwitchThumb
    backgroundColor="$background"
    borderRadius={999}
  />
</SwitchFrame>
```

The thumb has no explicit size — it inherits from Tamagui's Switch.Thumb which tries to fill the track height. With `padding={2}` the thumb should be 4px smaller, but it's clearly not enough or the Tamagui default thumb sizing ignores the padding.

**Fix:** Explicitly set the thumb size to be smaller than the track. Define thumb sizes relative to track sizes:

```ts
// Track heights from Tamagui tokens (approximate pixel values):
// $3 ≈ 28px, $4 ≈ 44px, $5 ≈ 52px
// Thumb should be ~60-70% of track height and square

const THUMB_SIZE_MAP = {
  sm: 16,  // track ~28px → thumb 16px leaves ~6px gap each side
  md: 20,  // track ~44px → thumb 20px leaves ~12px gap each side
  lg: 24,  // track ~52px → thumb 24px leaves ~14px gap each side
}
```

Apply explicit `width` and `height` to the SwitchThumb:
```tsx
<SwitchThumb
  animation="fast"
  backgroundColor="$background"
  borderRadius={999}
  width={THUMB_SIZE_MAP[size]}
  height={THUMB_SIZE_MAP[size]}
/>
```

If these explicit sizes don't override Tamagui's internal thumb sizing, try using `unstyled` on the Thumb to remove defaults, then set all dimensions explicitly.

Also consider reducing the track width-to-height ratio. The track is currently sized via Tamagui tokens which may create a square-ish track. Ideally the track should be wider than tall (roughly 1.75:1 ratio). If needed, add explicit `width` and `height` to the track too, removing the `size` prop.
