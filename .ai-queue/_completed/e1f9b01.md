# Install Accessibility & UI Quality Tooling

## Objective

Install and configure MCP servers, Claude plugins, and npm packages that enable AI agents to verify UI quality (spacing, layouts, contrast, UX principles) and accessibility (WCAG compliance) during the coding loop.

## Scope

- MCP configuration via `claude mcp add` commands (global Claude config)
- `package.json` (new dev dependencies)
- ESLint configuration file(s) (add jsx-a11y plugin)
- `.claude/settings.json` (MCP server entries, if `claude mcp add` doesn't handle it)

## Background Research (Already Completed)

Based on analysis of the article ["Accessibility Tooling for Agentic Coding Loops"](https://dev.to/cameron-accesslint/accessibility-tooling-for-agentic-coding-loops-5b1h) and broader tool research, here is what to install:

## Tier 1: Install Immediately (Free, Highest Value)

### 1. AccessLint MCP Server

Purpose-built accessibility toolkit for AI coding agents. 92 rules covering 23 WCAG 2.1 success criteria (Level A and AA). Benchmarks show 99.5% fix rate (vs 93.5% without), 23% fewer tokens, faster execution.

**Tools it provides:**
- `audit_html` — Scan HTML (string, file, or URL) for accessibility violations
- `diff_html` — Compare before/after audits to detect regressions
- `list_rules` — Query available accessibility rules
- `calculate_contrast_ratio` — WCAG contrast ratio between two colors
- `analyze_color_pair` — Pass/fail for normal text, large text, UI components
- `suggest_accessible_color` — Generate WCAG-compliant color alternatives

**Install:**
```bash
claude mcp add accesslint -- npx -y @accesslint/mcp
```

**Verification:** After installation, confirm the tools are available by asking Claude Code to call `list_rules`.

### 2. WCAG Guidelines MCP Server

Complete WCAG 2.2 reference — 87 success criteria, 400+ techniques, 101 glossary terms. Not an auditor; a knowledge base so the agent can look up WCAG requirements on demand.

**Tools it provides (20 total):**
- `list-principles`, `list-guidelines`, `list-success-criteria`, `get-criterion`
- `list-techniques`, `get-technique`, `get-techniques-for-criterion`
- `get-failures-for-criterion`, `search-wcag`, `get-full-criterion-context`
- And more

**Install:**
```bash
claude mcp add wcag -- npx -y wcag-guidelines-mcp
```

### 3. ESLint MCP Server

Official ESLint MCP server — runs linting rules (including jsx-a11y if configured) directly from the agent.

**Install:**
```bash
claude mcp add eslint -- npx @eslint/mcp@latest
```

### 4. eslint-plugin-jsx-a11y (npm package)

Static AST checker that catches accessibility issues in JSX at build time. Rules include: `alt-text`, `aria-props`, `aria-role`, `click-events-have-key-events`, `no-autofocus`, `tabindex-no-positive`, etc.

**Install:**
```bash
# From the repo root
npm install --save-dev eslint-plugin-jsx-a11y
```

**Configure:** Add to the project's ESLint config. If using flat config (ESLint 9+):
```js
import jsxA11y from 'eslint-plugin-jsx-a11y';
export default [jsxA11y.flatConfigs.recommended];
```

If using legacy config:
```json
{
  "extends": ["plugin:jsx-a11y/recommended"]
}
```

Check the existing ESLint config format before applying. Adapt to whatever config style the project uses.

## Tier 2: Install for Live Testing (Free, requires dev server)

### 5. Playwright MCP Server (by Microsoft)

Official Playwright MCP — navigate and interact with pages through structured accessibility snapshots (YAML-style accessibility tree), not screenshots. Powerful for verifying layout, interactive behavior, and page structure.

**Install:**
```bash
claude mcp add playwright -- npx @playwright/mcp@latest
```

### 6. a11y-mcp (axe-core based)

MCP server that performs accessibility audits on live webpages using axe-core. Supports WCAG tag filtering (`wcag2a`, `wcag2aa`, `wcag21a`, `best-practice`).

**Install:**
```bash
claude mcp add a11y -- npx a11y-mcp
```

### 7. Lighthouse MCP

Wraps Google Lighthouse — full audits for accessibility, performance, best practices, SEO. Supports mobile/desktop device emulation.

**Install:**
```bash
claude mcp add lighthouse -- npx lighthouse-mcp
```

**Requirements:** Chrome/Chromium must be installed on the machine.

## Installation Procedure

Execute the installations in this order:

1. **Tier 1 MCP servers** (AccessLint, WCAG, ESLint) — run the 3 `claude mcp add` commands
2. **eslint-plugin-jsx-a11y** — `npm install --save-dev` + ESLint config update
3. **Tier 2 MCP servers** (Playwright, a11y-mcp, Lighthouse) — run the 3 `claude mcp add` commands
4. **Verify** — for each installed MCP, confirm it appears in the MCP server list and its tools are callable
