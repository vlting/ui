# Segment 2: VisuallyHidden text node fix

## Objective
Fix the "Unexpected text node: Loading. A text node cannot be a child of a `<View>`" error in the VisuallyHidden component. Since VisuallyHidden is built on a `View` (via `styled(View, ...)`), raw text string children crash at runtime.

## Scope
- `packages/primitives/VisuallyHidden.tsx`

## Instructions

1. **Import `Text` from tamagui** (it's already importing `View` and `styled` from `tamagui`):
   ```ts
   import { View, Text, styled } from 'tamagui'
   ```

2. **Cast Text for JSX usage** (standard Tamagui v2 RC pattern):
   ```ts
   type AnyFC = ComponentType<Record<string, unknown>>
   const TextJsx = Text as AnyFC
   ```
   You'll need to import `ComponentType` from `react` (add to the existing `import type React from 'react'` line).

3. **Wrap children in a Text component** inside the VisuallyHidden function. Change the component body from:
   ```tsx
   <VisuallyHiddenFrame {...props} style={srOnlyStyle}>
     {children}
   </VisuallyHiddenFrame>
   ```
   to:
   ```tsx
   <VisuallyHiddenFrame {...props} style={srOnlyStyle}>
     <TextJsx>{children}</TextJsx>
   </VisuallyHiddenFrame>
   ```

   This ensures that raw text strings like `"Loading"` are properly wrapped in a Text node before being placed inside the View-based frame.

4. **Do NOT modify any callers** (e.g., Button.tsx). The fix should be entirely within VisuallyHidden.tsx so all consumers automatically benefit.

## Verification
- No TypeScript errors in `VisuallyHidden.tsx`
- The pattern matches the project's existing Tamagui v2 RC workaround style (AnyFC cast)
- Text children like `"Loading"` will now render correctly inside the View frame
