# Hero + Feed + Empty State Blocks

## Instructions

Create three new blocks for common UI patterns.

### HeroBlock

**Variants:**

| Variant | Description |
|---------|------------|
| `centered` | Centered text with CTA buttons, optional background gradient |
| `split` | Two-column: text on left, image/illustration placeholder on right |
| `image-bg` | Full-width with background image overlay and centered text |

**Props:**
```typescript
interface HeroBlockProps {
  variant: 'centered' | 'split' | 'image-bg'
  title: string
  description?: string
  primaryAction?: { label: string; onPress?: () => void; href?: string }
  secondaryAction?: { label: string; onPress?: () => void; href?: string }
  // Split variant
  media?: ReactNode  // image, illustration, or any content for right column
  mediaPosition?: 'left' | 'right'
  // Image-bg variant
  backgroundImage?: string  // URL
  overlay?: boolean  // dark overlay for text readability
  // Common
  badge?: string  // e.g., "New", "Coming Soon" — displayed above title
  align?: 'left' | 'center'
}
```

### FeedBlock

**Variants:**

| Variant | Description |
|---------|------------|
| `timeline` | Vertical timeline with dated entries and connecting line |
| `notifications` | Notification feed with read/unread states and icons |
| `comments` | Comment thread with avatars, replies, and timestamps |

**Props:**
```typescript
interface FeedBlockProps {
  variant: 'timeline' | 'notifications' | 'comments'
  title?: string
  // Timeline variant
  events?: TimelineEvent[]
  // Notifications variant
  notifications?: NotificationItem[]
  onMarkRead?: (id: string) => void
  onMarkAllRead?: () => void
  // Comments variant
  comments?: CommentItem[]
  onReply?: (parentId: string, text: string) => void
  onLike?: (id: string) => void
}

interface TimelineEvent {
  id: string
  title: string
  description?: string
  date: string
  icon?: ReactNode
  status?: 'completed' | 'current' | 'upcoming'
}

interface NotificationItem {
  id: string
  title: string
  description?: string
  timestamp: string
  read: boolean
  icon?: ReactNode
  onPress?: () => void
}

interface CommentItem {
  id: string
  author: { name: string; avatar?: string }
  text: string
  timestamp: string
  likes?: number
  liked?: boolean
  replies?: CommentItem[]
}
```

### EmptyStateBlock

**Variants:**

| Variant | Description |
|---------|------------|
| `no-data` | "No items yet" with illustration and action button |
| `error` | Error state with retry action |
| `coming-soon` | Feature coming soon with optional email signup |

**Props:**
```typescript
interface EmptyStateBlockProps {
  variant: 'no-data' | 'error' | 'coming-soon'
  title?: string
  description?: string
  icon?: ReactNode  // large illustration/icon
  action?: { label: string; onPress?: () => void }
  // Error variant
  errorCode?: string | number
  onRetry?: () => void
  // Coming soon variant
  onNotify?: (email: string) => void
}
```

### Implementation

1. Create `packages/blocks/hero/HeroBlock.tsx` + `index.ts`
   - Use H1, P, Button, Badge, Card, XStack, YStack from @vlting/ui
   - Centered: YStack with centered text alignment, gradient bg optional
   - Split: XStack with text column + media column, responsive stacking
   - Image-bg: full-width container with position absolute overlay

2. Create `packages/blocks/feed/FeedBlock.tsx` + `index.ts`
   - Use Card, Avatar, Badge, Button, Separator, Item from @vlting/ui
   - Timeline: vertical line with positioned dots/icons at each event
   - Notifications: Item-based list with read/unread styling
   - Comments: threaded layout with Avatar, indented replies

3. Create `packages/blocks/empty-state/EmptyStateBlock.tsx` + `index.ts`
   - Use YStack, H3, P, Button, Input, Field from @vlting/ui
   - All variants: centered layout with icon, title, description, action
   - Error: adds error code display and retry button
   - Coming soon: adds email input for notification signup

### Patterns
- AnyFC casting for all Tamagui/UI components
- styledHtml for form elements (coming-soon email form)
- Responsive layouts using Tamagui media queries
- All data via props, no internal business logic
