# Refactor kitchen-sink example: replace inline CSS with Tamagui patterns

## Objective

Eliminate all inline CSS `style={{}}` usage, raw HTML layout elements (`<div>`, `<span>`, `<button>`), and anti-patterns (buttons containing divs/spans for spacing) in the kitchen-sink example app. Replace with Tamagui `styled()` components, layout primitives (`XStack`, `YStack`, `View`), and token-based values.

## Principles

1. **All styling must use Tamagui** — `styled()` components with variants for dynamic styles. Only use inline `style` for dynamically *calculated* values (e.g., a height computed from JS at runtime).
2. **No raw HTML for layout** — replace `<div>` with `YStack`/`XStack`/`View`, `<span>` with `Text`/`SizableText`, `<header>`/`<aside>`/`<main>` with Tamagui `View` (with `tag` prop if semantic HTML is needed for accessibility: `<View tag="header">`).
3. **Buttons must not contain DIVs or SPANs for spacing** — use the button's own `padding`, `margin`, `gap` props. If a button needs an icon + text, use `XStack` inside or Tamagui's `Button.Icon` + `Button.Text` pattern.
4. **Use design tokens** — replace hardcoded pixel values with `$space` tokens (e.g., `$2`, `$3`), hardcoded colors with `$color` tokens, hardcoded font sizes with `$fontSize` tokens.
5. **Prefer `styled()` with variants** over conditional inline styles — e.g., an `isActive` variant on a nav link component instead of `style={{ color: isActive ? '#2554e8' : undefined }}`.

## File-by-File Instructions

### BrandLayout.tsx (LARGEST — 18+ inline style instances, entire layout is vanilla HTML)

This is a complete rewrite of the styling approach. The structure/logic stays the same, but every HTML element with `style={{}}` gets replaced:

**Layout containers:**
- `<div style={{ minHeight: '100vh', display: 'flex', flexDirection: 'column', ... }}>` → `<YStack minHeight="100vh" backgroundColor="$background" ...>`
- `<div style={{ display: 'flex', flex: 1 }}>` → `<XStack flex={1}>`
- `<div style={{ display: 'flex', alignItems: 'center', gap: 24 }}>` → `<XStack alignItems="center" gap="$4">`
- `<div style={{ display: 'flex', alignItems: 'center', gap: 8 }}>` → `<XStack alignItems="center" gap="$2">`

**Semantic HTML sections:**
- `<header style={{ position: 'sticky', ... }}>` → Create a `styled(XStack, { tag: 'header', ... })` component or `<XStack tag="header" position="sticky" ...>`
- `<aside style={{ width: 240, ... }}>` → `<YStack tag="aside" width={240} ...>`
- `<main style={{ flex: 1, minWidth: 0 }}>` → `<YStack tag="main" flex={1} minWidth={0}>`

**Buttons — eliminate raw HTML buttons:**
- `<button style={{ display: 'flex', alignItems: 'center', ... }}>` → Create `styled()` components or use Tamagui `Button` with proper variants
- Brand selector buttons → `styled()` button component with `active` variant for the selected state
- Theme toggle button → `styled()` button or Tamagui `Button` with `variant="ghost"` or similar
- Mobile hamburger → `styled()` button or Tamagui icon button

**Spans:**
- `<span style={{ fontWeight: 700, fontSize: 15, ... }}>` → `<Text fontWeight="700" fontSize="$3" ...>`
- `<span style={{ fontSize: 10, transition: ..., transform: ... }}>▶</span>` (expand/collapse icon) → This is a dynamically calculated rotation, so use `<Text fontSize={10}>` with inline style ONLY for the `transform` (calculated from `expanded` state). Or better: use a Tamagui-compatible icon component with `rotate` animation.

**Sidebar navigation links:**
- `<Link style={{ display: 'block', padding: '6px 16px 6px 24px', fontSize: 14, ... }}>` → Create a `styled()` component for nav links with an `active` variant that handles `backgroundColor`, `fontWeight`, `color`, and `borderRight` changes.

**Mobile overlay:**
- `<div style={{ position: 'fixed', inset: 0, ... backgroundColor: 'rgba(0,0,0,0.4)' }}>` → `<View position="fixed" inset={0} backgroundColor="rgba(0,0,0,0.4)" zIndex={30} />`

**Animated expand/collapse (sidebar groups):**
- `<div style={{ overflow: 'hidden', maxHeight: expanded ? ... : 0, transition: ... }}>` → The conditional `maxHeight` is a dynamically calculated value, so inline `style` for `maxHeight` is acceptable. But wrap it in a Tamagui `View` with `overflow="hidden"` as a Tamagui prop, and only keep the calculated `maxHeight` + transition as inline style.
