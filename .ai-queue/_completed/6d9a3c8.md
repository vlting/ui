# Segment 4: Component preview popup overflow fix

## Objective
Fix the DatePicker and Dropdown popups being cutoff by the component preview container. The outer `<div>` has `overflow-hidden` which clips any popover/dropdown content that extends beyond the container bounds.

## Scope
- `examples/docs/components/component-preview.tsx`

## Instructions

1. **Remove `overflow-hidden` from the outer container.** On line 15, change:
   ```tsx
   <div className="my-6 border border-border rounded-lg overflow-hidden">
   ```
   to:
   ```tsx
   <div className="my-6 border border-border rounded-lg">
   ```

2. **Add `overflow-hidden` to child sections that need it.** The title bar and code sections should still clip their content properly. Add `overflow-hidden` to:
   - The title bar div (line 17): already has `border-b`, just ensure it has `overflow-hidden` if needed for rounding
   - The code preview section (line 48): the `overflow-x-auto` on the pre tag is sufficient

3. **Ensure border-radius still works visually.** Without `overflow-hidden` on the parent, the border-radius on the outer div will still render the rounded border, but child content could visually overflow the rounded corners. To prevent this:
   - Add `overflow-hidden` to the title bar div (line 17) to clip its background within the rounded top corners:
     ```tsx
     <div className="px-4 py-2 border-b border-border bg-surface-muted overflow-hidden first:rounded-t-lg">
     ```
   - Or use `rounded-t-lg` on the title bar and `rounded-b-lg` on the last visible section to match the parent's rounding.

4. **The preview area (line 21-22) MUST NOT have overflow-hidden.** This is the area where DatePicker popups, Dropdown menus, and other popover content needs to extend beyond the container. Ensure it stays `overflow-visible` (the default).

- The component preview container renders with rounded corners on all edges
- Title bar background clips properly within the rounded top
- DatePicker and Dropdown popups can extend beyond the preview container
