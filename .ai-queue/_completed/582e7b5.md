# Rewrite Button + Toggle + Input + Textarea + Label + Form using Tamagui official components

> **This task SUPERSEDES the styledHtml approach from tasks 001-002.** Instead of using `styledHtml()` to render semantic HTML, we now use Tamagui's official component packages (`@tamagui/button`, `@tamagui/input`, `@tamagui/label`, `@tamagui/form`, `@tamagui/toggle-group`), which already render correct semantic HTML elements, handle ARIA attributes, and support keyboard navigation out of the box.

## Why this approach

Tamagui provides 22+ official component packages (all already installed in `node_modules/@tamagui/`). These components:
- **Render correct semantic HTML**: `@tamagui/button` renders `<button>`, `@tamagui/input` renders `<input>`, `@tamagui/label` renders `<label>`, `@tamagui/form` renders `<form>`
- **Use theme tokens**: They reference `$background`, `$borderColor`, `$color`, etc. — which our brand system already provides via `createBrandConfig()`
- **Support `styled()` wrapping**: Every component can be extended with `styled(Button, { ...customStyles })`
- **Have factory patterns**: Some offer `createCheckbox()`, `createTabs()`, etc. for deep customization
- **Support `unstyled` mode**: Pass `unstyled={true}` for headless use

## Implementation

### Button (`packages/components/Button/Button.tsx`)

Replace the entire custom implementation with Tamagui's Button.

```tsx
import { Button as TamaguiButton, type ButtonProps as TamaguiButtonProps } from '@tamagui/button'
import { styled, Theme, withStaticProperties } from 'tamagui'
```

1. Create `ButtonFrame = styled(TamaguiButton, { ...our custom variants })`.
   - Tamagui's Button already renders `<button type="button">` with correct semantics.
   - Tamagui's Button already has: `size` token support, `variant: 'outlined'`, `chromeless`, `circular`, `elevation`, `disabled`, hover/press/focus styles.
   - Add our custom variants on top: `variant` ('solid'/'outline'/'ghost') and `tone` ('neutral'/'primary'/'success'/'warning'/'danger').
   - For `variant: 'solid'` → use Tamagui's default (background filled).
   - For `variant: 'outline'` → map to Tamagui's `variant: 'outlined'`.
   - For `variant: 'ghost'` → map to `chromeless: true`.
   - For `tone` → wrap with `<Theme name={toneMap[tone]}>` (same approach as current code).

2. Preserve our compound API: `Button.Text`, `Button.Icon`. Tamagui's Button already provides `Button.Text` and `Button.Icon` — use those, extending with `styled()` if needed.

3. Preserve our `ButtonProps` interface (`variant`, `tone`, `size`, `loading`, `disabled`, `onPress`).

4. **Key difference**: Tamagui's Button `size` uses token values (`$4`, `$3`, etc.), not named sizes ('sm'/'md'/'lg'). Map our named sizes to tokens: `sm → '$3'`, `md → '$4'`, `lg → '$5'`.

## Key Patterns

For ALL replacements, follow this pattern:
1. Import the Tamagui component
2. `styled()` wrap it with our custom variants and theme token overrides
3. Create a wrapper React component if needed to maintain our API (e.g., `tone` prop → `<Theme>` wrapper)
4. Export with the same name and props interface for backwards compatibility
5. Remove our custom headless hooks/contexts that Tamagui handles (e.g., keyboard nav, ARIA, focus management)
