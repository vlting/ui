## Instructions

### Migration Pattern

| Old pattern | New pattern |
|------------|-------------|
| `text-gray-600 dark:text-gray-400` | `text-foreground-secondary` |
| `text-gray-700 dark:text-gray-300` | `text-foreground-secondary` |
| `border-gray-200 dark:border-gray-800` | `border-border` |
| `bg-white dark:bg-gray-950` | `bg-background` |
| `hover:bg-gray-50 dark:hover:bg-gray-900` | `hover:bg-surface-muted` |
| `hover:border-gray-300 dark:hover:border-gray-700` | `hover:border-border` (or a slightly lighter variant) |
| `bg-black/50` (overlay) | Keep as-is — overlays are legitimately fixed-color |
| `bg-gray-900 dark:bg-white` (icon fill) | `bg-foreground` |
| `text-white dark:text-gray-900` | `text-background` |

### Semantic Colors (Keep)

The `text-green-700 dark:text-green-400` ("When to use") and `text-red-700 dark:text-red-400` ("When not to use") headings in component pages serve a **semantic purpose** — green = recommended, red = avoid. These can be migrated to `text-success` and `text-destructive` tokens respectively. Check contrast:
- `text-green-700` (#15803d) on white: ~5.1:1 ✓ passes AA
- `text-red-700` (#b91c1c) on white: ~5.4:1 ✓ passes AA
- `text-green-400` (#4ade80) on dark: check contrast
- `text-red-400` (#f87171) on dark: check contrast

Use the `success` and `destructive` semantic tokens from globals.css. Ensure dark mode variants meet WCAG AA.
