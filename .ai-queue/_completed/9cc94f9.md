3. Add new MCP tools:

### `get_theme_tokens`
- Returns the full list of STL CSS custom properties with their default values
- Reads from `packages/stl/src/config/scales/` to generate the token list

### `audit_component`
- Takes a component name, runs `validate-tokens.ts` on that component's directory

- Optionally run axe-core hints based on known issues from COMPONENT_QUALITY_REPORT.md

### `get_block`
- Takes a block name, returns the full source code of that block
- Reads from `packages/blocks/{blockName}/`
- Returns the main block file + any sub-components
