# Document the "no layout wrappers inside interactive elements" anti-pattern

## Problem

The AI constitution (§2.8) has a general "No styling-only wrappers" rule, but it doesn't specifically call out the common anti-pattern of nesting layout containers (`<div>`, `XStack`, `YStack`, `View`) inside interactive elements (`<a>`, `<button>`) for spacing/padding. This pattern:

1. Inflates the DOM with unnecessary nodes
2. Can break accessibility (screen readers, focus behavior)
3. Is semantically wrong — interactive elements should own their own box model

The rule needs a concrete anti-pattern example so AI agents (and human developers) recognize and avoid it.

## Scope

- `AI_CONSTITUTION.md` (§2.8 DOM Optimization Rules)

## Instructions

1. Read `AI_CONSTITUTION.md` in full
2. In **§2.8 → DOM Optimization Rules**, add a new bullet point and accompanying example that specifically addresses nesting layout wrappers inside interactive elements. The rule should:

   - State that `<a>` and `<button>` elements must own their own spacing/layout styles — never delegate to a child `<div>`, `<View>`, `<XStack>`, `<YStack>`, or `<Stack>`
   - Show a concrete "Bad" vs "Good" example:

   **Bad:**
   ```tsx
   <a href="/page" style={{ display: 'block' }}>
     <XStack padding="$2" gap="$1">
       <Text>Label</Text>
     </XStack>
   </a>
   ```
   Renders: `<a><div style="padding…"><span>Label</span></div></a>` — unnecessary `<div>`

   **Good (React Router / third-party Link):**
   ```tsx
   <Link to="/page" style={{ display: 'flex', padding: 'var(--t-space-2)', gap: 'var(--t-space-1)' }}>
     <Text>Label</Text>
   </Link>
   ```
   Renders: `<a style="padding…"><span>Label</span></a>` — no wrapper

   **Good (native element):**
   ```tsx
   const NavButton = styledHtml('button', {
     display: 'flex',
     padding: '$2',
     gap: '$1',
   })
   // ...
   <NavButton><Text>Label</Text></NavButton>
   ```
   Renders: `<button style="padding…"><span>Label</span></button>` — no wrapper

3. The addition should be concise — a few sentences of rationale plus the examples. Do not restructure or rewrite existing content in §2.8.

4. Optionally, if DESIGN_CONSTITUTION.md §8.2 ("DOM Structure Quality") would benefit from a brief cross-reference to this new rule, add one sentence there. Do not duplicate the full examples.
