# Segment 3: Add fontConfig to all 4 brand definitions

**Epic:** #12 | **Stage:** #13 (Font System Foundation)

## Scope
- `packages/design-tokens/brands/default.ts`
- `packages/design-tokens/brands/fun.ts`
- `packages/design-tokens/brands/posh.ts`
- `packages/design-tokens/brands/shadcn.ts`

## Instructions

Add the `fontConfig` property to each of the 4 brand definitions. This uses the new `BrandFontConfig` interface (added in segment 001) to define the 4 font slots with heading weight alternation.

The `fontConfig` property should be added alongside the existing `fonts` property in each brand. The existing `fonts` (low-level Tamagui font overrides) remain unchanged — `fontConfig` is the high-level API that drives the new font system.

### 1. `packages/design-tokens/brands/default.ts`

Add `fontConfig` to the `defaultBrand` object. Place it BEFORE the existing `fonts` property:

```typescript
fontConfig: {
  heading: { family: 'Inter', weights: { heavy: 600, light: 400 } },
  body: { family: 'Inter', weight: 400 },
  mono: {
    family: 'JetBrains Mono',
    fallback: 'ui-monospace, SFMono-Regular, monospace',
    weight: 400,
  },
  quote: { family: 'Inter', weight: 300, style: 'italic' },
},
```

Also update the import line to include `BrandFontConfig` if needed for type checking. Actually, since the brand object is typed as `BrandDefinition` and `fontConfig` is optional on `BrandDefinition`, no import change is needed — TypeScript will infer the shape.

### 2. `packages/design-tokens/brands/fun.ts`

Add `fontConfig` to the `funBrand` object, before the existing `fonts` property:

```typescript
fontConfig: {
  heading: {
    family: 'DM Serif Display',
    fallback: 'Playfair Display, Georgia, serif',
    weights: { heavy: 400, light: 400 },
  },
  body: {
    family: 'DM Sans',
    fallback: 'Nunito Sans, Inter, system-ui, sans-serif',
    weight: 400,
  },
  mono: {
    family: 'Fira Code',
    fallback: 'ui-monospace, monospace',
    weight: 400,
  },
  quote: {
    family: 'DM Serif Display',
    fallback: 'Georgia, serif',
    weight: 400,
    style: 'italic',
  },
},
```

Note: DM Serif Display only comes in weight 400 (regular), so both heavy and light are 400. The visual distinction for this brand comes from the serif vs sans-serif contrast rather than weight.

  mono: {
    family: 'JetBrains Mono',

    weight: 400,
  },
  quote: { family: 'Inter', weight: 300, style: 'italic' },

### Important notes

- Do NOT remove or modify the existing `fonts` property in any brand — it continues to serve as low-level overrides that are applied after `fontConfig` processing
- The `fontConfig` property is placed BEFORE `fonts` in the object literal for readability (high-level config first, then low-level overrides)
- All font family names must exactly match Google Fonts registry names (case-sensitive) since `getGoogleFontsUrl()` uses them to build the URL

- Weight values are valid CSS font-weight numbers (100-900)
