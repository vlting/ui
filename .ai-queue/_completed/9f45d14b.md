## CSS Variable Convention
Components use `var(--stl-{token}, {fallback})` pattern. Key vars:
- `--stl-background` → background color
- `--stl-foreground` → text color
- `--stl-border` → border color
- `--stl-color1` through `--stl-color12` → palette steps
- `--stl-shadow-sm`, `--stl-shadow-md`, `--stl-shadow-lg` → shadows
- `--stl-radius0` through `--stl-radius12` → border radius
- `--stl-space0` through `--stl-space16` → spacing
- `--stl-maxAlpha8` → dark overlay/shadow color

## Key principles
- **Overlay rgba** — wrap in `var(--stl-overlay, rgba(...))` to make tokenizable
- **Shadow rgba** — wrap in `var(--stl-shadow-{size}, ...)` — keep the rgba as fallback
- **Existing `var(--stl-*)` with fallbacks** — remove the fallback values (the vars are always defined by STL)
- **ScrollArea scrollbar opacity** — leave as-is (browser-specific visual)
- **Do NOT touch .native.tsx files or test files.**
