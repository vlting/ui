### MobileTabLayout — Tab Bar Navigation (Mobile-First)

A mobile-first layout with a tab bar at the bottom for primary navigation. Tabs switch between content panels. On wider screens (gtMd), the tabs shift to a sidebar or top position.

**Props:** `MobileTabLayoutProps` with:
- `tabs: TabItem[]` — tab definitions
- `defaultTab?: string` — default active tab value
- `activeTab?: string` — controlled active tab
- `onTabChange?: (value: string) => void` — tab change callback
- `children?: ReactNode` — additional content

**TabItem type:**
```typescript
interface TabItem {
  value: string
  label: string
  icon?: ReactNode
  content: ReactNode
}
```

**Layout:**
- Container with flex={1} and flexDirection="column"
- Content area: flex={1}, renders the active tab's content via Tabs.Content
- Bottom tab bar: Tabs.List at the bottom with Tabs.Trigger for each tab
- Tab triggers show icon (if provided) above label
- Uses Tabs.Root for state management

### MasterDetail — Split View Responsive Pattern

A responsive split-view layout with a master list on the left and a detail panel on the right. On narrow screens (md and below), only the master or detail view is shown based on selection state.

**Props:** `MasterDetailProps` with:
- `masterContent: ReactNode` — master list/panel content
- `detailContent?: ReactNode` — detail panel content (shown when an item is selected)
- `masterWidth?: number | string` — master panel width (default: 320)
- `showDetail?: boolean` — whether to show detail panel (for responsive control)
- `onBack?: () => void` — back button callback (shown on mobile detail view)
- `masterHeader?: ReactNode`
- `detailHeader?: ReactNode`

**Layout:**
- Desktop (gtMd): Side-by-side row layout. Master panel with fixed width, Separator, Detail panel with flex={1}.
- Mobile (md and below): Show either master or detail based on `showDetail` prop.
  - When `showDetail` is false/undefined: show masterContent full width.
  - When `showDetail` is true and detailContent exists: show detailContent full width with a back button at top.

**Implementation:**
- Use `useMedia()` from tamagui for responsive behavior, or use `$gtMd` media props on View.
- Since `useMedia()` may not exist in v2 RC, use conditional rendering based on a prop (`showDetail`) instead of runtime media queries. The consumer handles the responsive logic.
- Back button uses Button component from `../../components/Button`.
