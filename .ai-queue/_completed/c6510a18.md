# Task: Convert docs shell & layout components from Tailwind to STL

## Context
We're replacing Tailwind utility classes with STL styling in the docs app. STL uses `styled()` from `@vlting/stl-react` which produces pre-generated CSS classes via Vanilla Extract. Components can also reference STL CSS variables directly via `var(--stl-*)`.

## Available STL Tools
- `styled('div', { display: 'flex', gap: '$4', padding: '$2' })` — creates styled component with token refs
- Token syntax: `'$4'` maps to space/size tokens, `'$primary9'` maps to color tokens
- Conditions: `{ dark: { background: '$surface1' } }` for dark mode
- Responsive: `{ md: { display: 'none' } }` for breakpoints
- Pseudo-classes: `{ ':hover': { background: '$surface2' } }`

## Tailwind → STL Token Mapping (common patterns)
- `flex` → `display: 'flex'`
- `items-center` → `alignItems: 'center'`
- `justify-between` → `justifyContent: 'space-between'`
- `gap-N` → `gap: '$N'` (use nearest STL space token)
- `p-N` → `padding: '$N'`
- `px-N` → `px: '$N'`
- `py-N` → `py: '$N'`
- `w-full` → `width: '100%'`
- `h-N` → `height: '$N'`
- `text-sm` → `fontSize: '$p'` or `fontSize: '$14'`
- `font-medium` → `fontWeight: '$500'`
- `border-b` → `borderBottom: '1px solid var(--stl-borderColor)'`
- `bg-background` → `background: '$background'`
- `text-foreground` → `color: '$color'`
- `hidden md:flex` → use responsive conditions
- `sticky top-0` → `position: 'sticky'`, `top: '0'`
- `z-50` → `zIndex: '$6'` (or appropriate STL zIndex token)
- `rounded-md` → `borderRadius: '$4'`
- `dark:` prefix → use `{ dark: { ... } }` condition

## Approach
For each file:
1. Read the file to understand the current Tailwind usage
2. Create STL styled components at the top of the file using `styled()` from `@vlting/stl-react`
3. Replace `className="..."` patterns with the styled components
4. Use STL responsive conditions for breakpoint-based visibility/layout
5. Use `dark:` condition for dark mode styles
6. Use semantic STL color tokens (`$background`, `$color`, `$borderColor`, `$surface1-3`, `$primary9`, etc.)

## Rules
- NO inline `style={{}}` for values that should be tokens
- NO creating a utility CSS file — use STL `styled()` exclusively
- ALL spacing values must use STL space tokens
- ALL colors must use STL color tokens
- If a Tailwind value has no exact STL token match, use the nearest token or a CSS variable

## Acceptance criteria
- Zero Tailwind utility classes in the 4 files

- Layout responds correctly at different breakpoints
