# Refactor BrandLayout to use Tamagui `asChild` pattern instead of inline styles / styledHtml

## Problem

The current BrandLayout.tsx uses two approaches to avoid layout wrappers inside interactive elements:
1. **`styledHtml('button', {...})`** for native buttons (HamburgerButton, PillButton)
2. **Inline CSS `style` with CSS custom variables** for React Router `<Link>` elements

Both approaches work but lose Tamagui's token resolution and theming system — inline styles use raw `var(--color3)` strings, and `styledHtml` requires `as any` casts. A better pattern exists: use Tamagui layout components (`XStack`, `View`) with the `asChild` prop, which tells Tamagui to clone the child element and apply styles directly to it instead of rendering a wrapper `<div>`.

### The `asChild` Pattern

```tsx
<XStack asChild paddingVertical={6} paddingLeft={16} paddingRight={8}>
  <Link to="/page" style={{ textDecoration: 'none', color: 'inherit' }}>
    <Text>Label</Text>
  </Link>
</XStack>
```

Tamagui clones the `<Link>` and merges the XStack's resolved styles onto it. The rendered output is `<a style="padding…"><span>Label</span></a>` — no extra `<div>`, and Tamagui tokens resolve correctly.

This pattern is already used throughout the `@vlting/ui` component library (Dialog.Trigger, Tooltip.Trigger, Popover.Trigger, AlertDialog.Trigger, Button).

## Instances to Refactor

### 1. Nav items (Link elements)
**Current** (inline CSS with CSS vars):
```tsx
<Link to={...} style={{ ...LINK_RESET, padding: '6px 8px 6px 16px', borderRight: `2px solid ${isActive ? 'var(--color)' : 'transparent'}`, backgroundColor: isActive ? 'var(--color3)' : 'transparent', transition: 'all 0.1s' }}>
  <Text ...>{item.label}</Text>
</Link>
```

**New** (`asChild`):
```tsx
<XStack
  asChild
  paddingVertical={6}
  paddingLeft={16}
  paddingRight={8}
  borderRightWidth={2}
  borderRightColor={isActive ? '$color' : 'transparent'}
  backgroundColor={isActive ? '$color3' : 'transparent'}
  style={{ transition: 'all 0.1s' }}
>
  <Link to={...} onClick={onNavClick} style={LINK_RESET}>
    <Text ...>{item.label}</Text>
  </Link>
</XStack>
```

### 2. Sub-items (Link elements)
**Current:**
```tsx
<Link to={...} style={{ ...LINK_RESET, padding: '2px 16px 2px 36px', transition: 'all 0.1s' }}>
```

**New:**
```tsx
<XStack asChild paddingVertical="$0.25" paddingLeft={36} paddingRight="$2" style={{ transition: 'all 0.1s' }}>
  <Link to={...} style={LINK_RESET}>
    <Text ...>{sub.label}</Text>
  </Link>
</XStack>
```

### 3. Mobile hamburger button
**Current** (`styledHtml` HamburgerButton):
```tsx
<HamburgerButton type="button" onClick={...} className="mobile-hamburger" aria-label="...">
  <Text ...>☰</Text>
</HamburgerButton>
```

**New:**
```tsx
<View asChild paddingVertical="$0.5" paddingHorizontal="$0.75" display="none">
  <button type="button" onClick={...} style={BUTTON_RESET} className="mobile-hamburger" aria-label="...">
    <Text ...>☰</Text>
  </button>
</View>
```

### 4. Brand selector buttons
**Current** (`styledHtml` PillButton):
```tsx
<PillButton key={key} type="button" onClick={...} borderColor={...} backgroundColor={...}>
  <Text ...>{b.label}</Text>
</PillButton>
```

**New:**
```tsx
<XStack
  asChild
  paddingVertical="$0.5"
  paddingHorizontal="$1.5"
  borderRadius="$2"
  borderWidth={1}
  borderColor={isCurrent ? '$color10' : '$borderColor'}
  backgroundColor={isCurrent ? '$color3' : 'transparent'}
  style={{ transition: 'all 0.15s' }}
>
  <button key={key} type="button" onClick={...} style={BUTTON_RESET}>
    <Text ...>{b.label}</Text>
  </button>
</XStack>
```

### 5. Theme toggle button
Same as brand selector but without conditional borderColor/backgroundColor — uses `borderColor="$borderColor"` and no backgroundColor override.
