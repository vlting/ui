3. **Token replacement patterns:**
   - `#ffffff` / `#fff` → `var(--stl-surface1, #fff)` or `var(--stl-background, #fff)`
   - `#111111` / `#000` → `var(--stl-color, #111)` or `var(--stl-foreground, #000)`
   - `#f3f4f6` / `#e5e7eb` → `var(--stl-surface2, #f3f4f6)` or `var(--stl-surface3, #e5e7eb)`
   - `#6a6a6a` / `#9ca3af` → `var(--stl-color8, #9ca3af)`
   - `rgba(0,0,0,0.5)` → `var(--stl-overlay, rgba(0,0,0,0.5))`
   - Hardcoded px padding/margin → use `var(--stl-space-N)` tokens or keep as CSS fallback `var(--stl-space4, 16px)`
   - `border-radius: Npx` → `var(--stl-radiusN, Npx)`

4. **Rules:**
   - Keep the fallback value in `var()` — `var(--token, fallback)`
   - Don't change the visual appearance, just tokenize
   - Skip 1px borders and 0px values (allowed by exclude rules)
   - Skip viewBox, strokeWidth attributes
   - For Charts: the hex colors are often data-driven — wrap in var() with the current value as fallback

5. After changes, run `npx tsx scripts/validate-tokens.ts` to verify reduction
