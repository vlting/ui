### 3. Create Item.spec.md

Create `packages/components/Item/Item.spec.md`. Follow the template at `packages/_templates/component.spec.template.md`.

Key sections:

**1. Purpose:**
- Generic list item layout for menus, settings panels, contact lists, etc.
- When to use: List items needing icon/avatar + text + optional trailing action
- When NOT to use: Simple text lists (use Text), complex card layouts (use Card), navigation items (use NavigationMenu)

**2. UX Intent:**
- Interaction goal: Scannable list with consistent structure
- Mental model: Each item is a row with leading visual, main content, and trailing action
- UX laws: Fitts's Law (touch targets), Hick's Law (reducing choices per item)

**3. Anatomy:**
- `Item` (Root) — XStack container, full width
- `Item.Leading` — Left slot: avatar, icon, checkbox (flexShrink: 0)
- `Item.Content` — Main area: title + description (flex: 1)
- `Item.Title` — Primary text (medium weight)
- `Item.Description` — Secondary text (muted color)
- `Item.Trailing` — Right slot: action button, badge, chevron (flexShrink: 0)
- All sub-components except Root are optional

**4. Behavior:**
- States: Default, hover (interactive), pressed (interactive), focused (interactive)
- Keyboard: Tab to focus (when interactive), Enter/Space to activate
- Motion: None (hover/press via Tamagui style props)

**5. Accessibility:**
- Touch target: minHeight 44px (md), 36px (sm), 52px (lg) — WCAG 2.5.8
- Focus indicator: 2px solid outline on interactive variant — WCAG 2.4.7
- No implicit ARIA role — consumers add `role="listitem"` or wrap in `role="list"` as needed
- Color contrast: Title uses `$color` (primary), Description uses muted color — both must meet 4.5:1

**6. Styling:**
- All values from design tokens
- Size variants: sm, md (default), lg
- Interactive variant: hover/press/focus styles
- Dark mode: automatic via token system
- Responsive: full width, adapts to container

**7. Composition:**
- Item.Leading can contain: Avatar, Icon, Checkbox, RadioGroup.Item
- Item.Content always contains Title; Description is optional
- Item.Trailing can contain: Button, Badge, Icon, Switch
- Anti-patterns: Don't nest Item inside Item; Don't put complex forms inside Item

**8. Breaking Change Criteria:**
- Removing any sub-component
- Changing size variant token values
- Changing the interactive variant behavior

**9. Test Requirements:**
- Renders all sub-components
- Size variants apply correct spacing
- Interactive variant has hover/focus styles
- Renders without optional sub-components
- Cross-platform rendering

```typescript
import { render, screen } from '../../../src/__test-utils__/render'
import { Item } from './Item'

describe('Item', () => {
  it('renders with all sub-components', () => {
    render(
      <Item>
        <Item.Leading>
          <span>icon</span>
        </Item.Leading>
        <Item.Content>
          <Item.Title>Title text</Item.Title>
          <Item.Description>Description text</Item.Description>
        </Item.Content>
        <Item.Trailing>
          <span>action</span>
        </Item.Trailing>
      </Item>,
    )
    expect(screen.getByText('Title text')).toBeTruthy()
    expect(screen.getByText('Description text')).toBeTruthy()
    expect(screen.getByText('icon')).toBeTruthy()
    expect(screen.getByText('action')).toBeTruthy()
  })

  it('renders without optional sub-components', () => {
    render(
      <Item>
        <Item.Content>
          <Item.Title>Just a title</Item.Title>
        </Item.Content>
      </Item>,
    )
    expect(screen.getByText('Just a title')).toBeTruthy()
  })

  it('renders with interactive variant', () => {
    const { container } = render(
      <Item interactive>
        <Item.Content>
          <Item.Title>Interactive item</Item.Title>
        </Item.Content>
      </Item>,
    )
    expect(screen.getByText('Interactive item')).toBeTruthy()
    // Interactive variant should set cursor: pointer
    // Note: Tamagui v2 RC may not render all style props in JSDOM
  })

  it('renders with size variants', () => {
    const { rerender } = render(
      <Item size="sm">
        <Item.Content>
          <Item.Title>Small</Item.Title>
        </Item.Content>
      </Item>,
    )
    expect(screen.getByText('Small')).toBeTruthy()

    rerender(
      <Item size="lg">
        <Item.Content>
          <Item.Title>Large</Item.Title>
        </Item.Content>
      </Item>,
    )
    expect(screen.getByText('Large')).toBeTruthy()
  })
})
```
