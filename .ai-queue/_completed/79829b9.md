# Sidebar Block — Consolidated Navigation Sidebars

## Instructions

Create a single `SidebarBlock` component that replaces all 16 sidebar blocks (Sidebar01-16) with a variant-driven design.

### Variants

| Variant | Description | Based On |
|---------|------------|----------|
| `grouped` | Simple grouped navigation | Sidebar01 |
| `collapsible` | Collapsible sections with expand/collapse | Sidebar02 |
| `nested` | Nested submenus with indentation | Sidebar03 |
| `floating` | Floating/elevated sidebar with shadows | Sidebar04 |
| `icon-only` | Compact icon-only mode (labels on hover) | Sidebar07 |
| `file-tree` | File/folder tree navigation | Sidebar11 |
| `calendar` | Sidebar with calendar widget | Sidebar12 |

### Props Interface

```typescript
interface SidebarBlockProps {
  variant: 'grouped' | 'collapsible' | 'nested' | 'floating' | 'icon-only' | 'file-tree' | 'calendar'
  groups: NavGroup[]
  header?: ReactNode
  footer?: ReactNode
  side?: 'left' | 'right'
  defaultOpen?: boolean
  width?: number
  collapsedWidth?: number
  // File tree specific
  fileTree?: FileTreeItem[]
  // Calendar specific
  events?: CalendarEvent[]
  selectedDate?: Date
  onDateChange?: (date: Date) => void
}
```

### Implementation

1. Create `packages/blocks/sidebar/SidebarBlock.tsx`:
   - Single exported `SidebarBlock` component with `variant` prop
   - Variant rendering via switch pattern
   - Reuse existing sidebar shared components from `packages/blocks/sidebar/_shared.tsx` (SidebarNavItem, SidebarNavGroup, etc.)
   - Use `@vlting/ui` Sidebar component and sub-components
   - Follow AnyFC casting pattern
