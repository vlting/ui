### Changes Required

#### Menubar.tsx

1. **Fix Trigger** тАФ Currently `ViewJsx` with `role="menuitem"` + `onPress`:
   - Replace with `styledHtml('button')` element
   - Add `type="button"`
   - Keep `role="menuitem"` (menubar context)
   - Add `focusVisibleStyle` (standard pattern: `outlineWidth: 2, outlineOffset: 2, outlineColor: '$outlineColor', outlineStyle: 'solid'`)

2. **Fix Item** тАФ Currently `ViewJsx` with `role="menuitem"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Keep `role="menuitem"`
   - Add `focusVisibleStyle`

#### NavigationMenu.tsx

1. **Fix Trigger** тАФ Currently `ViewJsx` with `role="button"` + `onPress` + `tabIndex={0}` + `onKeyDown`:
   - Replace with `styledHtml('button')` element
   - Add `type="button"`
   - Remove manual `role="button"` тАФ native button provides this
   - Remove manual `tabIndex={0}` тАФ native button is focusable by default
   - Keep existing `onKeyDown` handler
   - Keep existing `focusVisibleStyle` (already has it!)

2. **Fix Link** тАФ Currently `ViewJsx` with `role="link"` + `onPress`:
   - Replace with `styledHtml('a')` element тАФ links should be `<a>`, not `<button>`
   - Remove manual `role="link"` тАФ native `<a>` provides this
   - Keep `tabIndex={-1}` if present (for roving tabindex)
   - Keep existing `onKeyDown` handler
   - Keep existing `focusVisibleStyle`
   - Add `href` prop pass-through if not already present

#### Sidebar.tsx

1. **Fix Trigger** тАФ Currently `ViewJsx` with `role="button"` + `onPress`:
   - Replace with `styledHtml('button')` element
   - Add `type="button"`, `aria-label="Toggle sidebar"`
   - Remove manual `role="button"`
   - Add `focusVisibleStyle`

2. **Fix MenuItem** тАФ Currently `ViewJsx` with `role="menuitem"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Keep `role="menuitem"` if inside a menu context, or remove if just a nav item
   - Add `focusVisibleStyle`

### Important Constraints

- Do NOT change visual appearance тАФ only the underlying HTML element
- Do NOT fix token discipline issues (hardcoded pixels) тАФ that's Stage 4
- Do NOT fix SSR-unsafe module-level counters (`menuIdCounter`, `navItemId`) тАФ that's out of Stage 2 scope
- Do NOT add reduced motion support тАФ that's Stage 5

- All new `<button>` elements need `type="button"`
- `styledHtml()` elements inherently reset browser styles тАФ add `background: 'transparent'`, `border: 'none'`, `padding: 0` or similar resets as needed
