## Instructions

### Issue 1 — shadcn brand hover states appear pink/red instead of neutral

When using the shadcn brand, interactive hover states (buttons, menu items, etc.) appear pink/red instead of neutral gray. In shadcn/ui, hover states use neutral grays — red is ONLY for the destructive variant.

**Investigation steps:**
1. Read `packages/design-tokens/brands/shadcn.ts` — look at how `accentPalettes` is configured
2. Read `packages/design-tokens/themes.ts` — understand how the theme builder uses accent palettes from brand configs. Look for how `$accentBackground`, `$accentColor`, or similar tokens are populated
3. Read `packages/design-tokens/brands/defaultBrand.ts` and `packages/design-tokens/brands/fun.ts` for comparison — how do they handle accent palettes?

**Fix approach:** The goal is that normal interactive hover states use neutral grays for the shadcn brand. Only the destructive variant should use red. Options:
1. If shadcn's accent palette is configured as `red` and feeds into hover tokens, change the accent to a neutral palette or remove it
2. If there's a specific mapping in the theme builder that routes accent palettes to hover tokens, ensure shadcn's neutral palette is used for those tokens instead
3. Review whether other brands (default, fun, posh) have similar issues and fix them too if applicable

### Issue 2 — Destructive button uses dark red bg with white text

The Button.tsx destructive variant currently uses:
- `backgroundColor: '$color10'` — dark red in the red theme context
- `color: '$color1'` — near-white in the red theme context
- `hoverStyle: { backgroundColor: '$color11' }` — even darker red

The shadcn/ui convention is the opposite: **light red background with dark red text**.

**Fix:** In `Button.tsx`, find the `VARIANTS` object and change the destructive variant styles:

```tsx
destructive: {
  backgroundColor: '$color4',   // light red bg (was $color10)
  color: '$color11',            // dark red text (was $color1)
  hoverStyle: {
    backgroundColor: '$color5', // slightly darker on hover (was $color11)
  },
}
```

The `VARIANT_THEME_OVERRIDE: { destructive: 'red' }` mechanism wraps the button in `<Theme name="red">`, so these token numbers pull from each brand's red/accent palette:
- `$color4` = light/pastel version (~#fecaca range in shadcn)
- `$color5` = slightly darker hover
- `$color11` = dark version for text contrast

**Important:** Test this visually (or by reading palette values) across ALL brands:
- `packages/design-tokens/brands/defaultBrand.ts`
- `packages/design-tokens/brands/fun.ts`
- `packages/design-tokens/brands/posh.ts`
- `packages/design-tokens/brands/shadcn.ts`

Each brand defines a red accent palette. The token steps (4, 5, 11) should produce a reasonable light-bg/dark-text pattern in each. If not, adjust the numbers. The key principle: background should be a light/pastel version of red, text should be dark red.
