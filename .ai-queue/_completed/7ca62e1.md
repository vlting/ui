# Task: Wire token validation script into CI

## Context
`scripts/validate-tokens.ts` already exists and scans components/primitives for hardcoded px/hex/rgb/hsl values. It needs to be integrated into the CI pipeline and made runnable via npm script.

## Instructions

1. **Add npm script** to root `package.json`:
   ```
   "validate:tokens": "npx tsx scripts/validate-tokens.ts"
   ```

2. **Add to CI** in `.github/workflows/quality.yml`:
   - Add a new job `token-validation` that runs after `lint`
   - Steps: checkout, setup-node, yarn install, `yarn validate:tokens`
   - This job should NOT block the pipeline (use `continue-on-error: true` for now — E3 will fix violations)
   - But it MUST report violations in the job output

3. **Improve the validation script** (`scripts/validate-tokens.ts`):
   - Add `--ci` flag support: when passed, output violations in GitHub Actions annotation format (`::warning file={file},line={line}::{message}`)
   - Add exit code 0 even with violations when `--report-only` flag is passed (for CI)
   - Update npm script: `"validate:tokens": "npx tsx scripts/validate-tokens.ts"` and `"validate:tokens:ci": "npx tsx scripts/validate-tokens.ts --ci --report-only"`

4. **Test locally**: Run `yarn validate:tokens` and verify it produces output listing violations.
