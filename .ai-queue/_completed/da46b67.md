# A11y Fix: Select Trigger — Use Native `<button>` Element

## Priority: HIGH

## Root Cause

`styled(XStack, { tag: 'button' })` renders `<div tag="button">`, not `<button>`. The Select trigger is not keyboard-focusable.

## Scope

Files modified:
- `packages/components/Select/Select.tsx`

## Required Changes

### 1. Wrap trigger in native `<button>`

The `Select` component already has keyboard handling and ARIA attributes. The fix is to render a native `<button>` instead of relying on the broken `tag: 'button'` in styled():

```tsx
// In the Select component render:
<button
  type="button"
  role="combobox"
  aria-expanded={open}
  aria-haspopup="listbox"
  aria-disabled={disabled || undefined}
  onClick={() => !disabled && setOpen(!open)}
  onKeyDown={handleKeyDown}
  style={{ all: 'unset', display: 'inline-flex', width: '100%', cursor: disabled ? 'not-allowed' : 'pointer' }}
>
  <SelectTriggerVisual size={size} error={error} disabled={disabled}>
    {/* trigger content */}
  </SelectTriggerVisual>
</button>
```

Remove `tag: 'button'` from `SelectTrigger` styled() — keep it as a purely visual wrapper.

### 2. Add `focusWithinStyle` to the visual trigger

So that when the native `<button>` receives focus, the visual wrapper shows a focus ring:

```tsx
const SelectTriggerVisual = styled(XStack, {
  // ...existing styles (remove tag: 'button')
  focusWithinStyle: {
    borderColor: '$borderColorFocus',
    outlineWidth: 2,
    outlineOffset: 0,
    outlineColor: '$outlineColor',
    outlineStyle: 'solid',
  },
})
```

### 3. Ensure keyboard navigation works

The Select already has `handleKeyDown` with Enter/Space/Escape/Arrow handling. Verify these still work with the native button wrapper. The native `<button>` will fire `onClick` on Enter/Space by default, so the keyboard handler may need adjustment to avoid double-firing.

### 4. Add label association

Accept an `id` prop or generate one with `useId()`. If used with a label, support `aria-labelledby`.
