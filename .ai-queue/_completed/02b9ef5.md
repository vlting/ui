# Auth Block — Unified Login + Signup

## Instructions

Create a single `AuthBlock` component that replaces the 5 login blocks (Login01-05) and 5 signup blocks (Signup01-05) with a variant-driven design.

### Variants

| Variant | Description | Based On |
|---------|------------|----------|
| `login-standard` | Email + password login form | Login01 |
| `login-otp` | Email verification with OTP input | Login02 |
| `login-magic` | Magic link authentication | Login03 |
| `login-social` | OAuth-only login (no email/password) | Login05 |
| `signup-standard` | Standard registration form | Signup01 |
| `signup-social` | Social-first registration | Signup05 |

### Props Interface

```typescript
interface AuthBlockProps extends BlockProps {
  variant: 'login-standard' | 'login-otp' | 'login-magic' | 'login-social' | 'signup-standard' | 'signup-social'
  title?: string
  description?: string
  logo?: ReactNode
  onSubmit?: (data: Record<string, string>) => void
  socialProviders?: SocialProvider[]
  loading?: boolean
  error?: string
  // Login-specific
  forgotPasswordHref?: string
  onForgotPassword?: () => void
  signupHref?: string
  onSignup?: () => void
  // Signup-specific
  loginHref?: string
  onLogin?: () => void
  termsHref?: string
  privacyHref?: string
}
```

### Implementation

1. Create `packages/blocks/auth/AuthBlock.tsx`:
   - Single exported `AuthBlock` component with `variant` prop
   - Internal variant rendering via switch/map pattern
   - Use the existing `_shared.tsx` helpers from `packages/blocks/login/_shared.tsx` as reference (AuthFormCard, AuthFormHeader, AuthDivider, AuthSocialButtons, AuthFooterLink)
   - Follow AnyFC casting pattern for all Tamagui/UI components
   - Use `styledHtml('form', {...})` for form elements
   - All callbacks optional, no internal business logic

2. Create `packages/blocks/auth/_shared.tsx`:
   - Copy and adapt shared components from `packages/blocks/login/_shared.tsx`
   - AuthFormCard, AuthFormHeader, AuthDivider, AuthSocialButtons, AuthFooterLink

### Patterns to Follow
- Read existing `packages/blocks/login/Login01.tsx` for the standard form pattern
- Read existing `packages/blocks/login/Login02.tsx` for the OTP pattern
- Read existing `packages/blocks/login/Login03.tsx` for the magic link pattern
- Use AnyFC casting: `type AnyFC = ComponentType<Record<string, unknown>>`
- Use `styledHtml('form', {...})` for `<form>` elements
- Use `@vlting/ui` components: Button, Field, Input, Card, etc.
- Semantic HTML, accessible, keyboard navigable
- Error state with `role="alert"`
