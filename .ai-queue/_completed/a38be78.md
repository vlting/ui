# CodeBlock + PropTable + ComponentPreview Components

Build the three core documentation components used across all component pages.

## Context
- Tech Spec: `.ai-epics/docs/docs-site/tech-spec.md`
- Existing docs site scaffold in `examples/docs/`
- Providers component at `examples/docs/components/providers.tsx` provides brand context
- These components will be used by the dynamic `[name]` route and all component pages
- shiki needs to be added as a dependency for syntax highlighting

## Instructions

### 0. Add shiki dependency

Add `shiki` to `examples/docs/package.json` dependencies:
```json
"shiki": "^3.0.0"
```

Run `yarn install --ignore-engines` from the repo root to update the lockfile.

### 1. Create `examples/docs/components/code-block.tsx`

A syntax-highlighted code block with copy-to-clipboard button. This is a **server component** that uses shiki for highlighting at build time.

```typescript
interface CodeBlockProps {
  code: string
  language?: string   // default: 'tsx'
  showLineNumbers?: boolean
  title?: string      // optional filename/title shown above the code
}
```

Implementation notes:
- Use shiki's `codeToHtml()` for server-side syntax highlighting
- Use a theme that works well in both light and dark mode (e.g., `github-light` and `github-dark` with CSS media query or class-based selection matching the `next-themes` `class` attribute strategy)
- Wrap the highlighted HTML in a container with:
  - Optional title bar (filename)
  - Copy button (client component — extract a `CopyButton` client sub-component that uses `navigator.clipboard.writeText`)
  - Rounded corners, border, subtle background
- Style with Tailwind CSS utility classes (this is docs-only styling)
- The code should be displayed in a `<pre>` block with horizontal scroll for long lines

### 2. Create `examples/docs/components/prop-table.tsx`

Auto-generated props table from API mapping JSON data. Server component.

```typescript
interface PropTableProps {
  props: Record<string, {
    type: string
    values?: string[]
    required?: boolean
    default?: string
    description?: string
  }>
  title?: string  // e.g., "Button" or "Dialog.Content"
}
```

Implementation:
- Render an HTML `<table>` with columns: Prop, Type, Default, Description
- For props with `values` array, show them as inline code badges (e.g., `"sm" | "md" | "lg"`)
- Required props should have a visual indicator (e.g., asterisk or "Required" badge)
- Style with Tailwind CSS
- If compound component (multiple sub-component prop sets), render a separate table per sub-component with the sub-component name as heading

### 3. Create `examples/docs/components/component-preview.tsx`

A live component preview container that renders components with current brand theming.

```typescript
interface ComponentPreviewProps {
  children: ReactNode     // The actual component to render
  code?: string           // Code string to show below the preview
  title?: string          // Preview title
  className?: string
}
```

Implementation:
- This is a **client component** (needs to re-render when brand changes)
- Render the children inside a bordered container with padding
- Children render within the existing Tamagui Provider (already in the layout tree), so they automatically get the current brand theming
- Below the preview area, optionally show the code using CodeBlock (import and render it)
- Add a "Show code" / "Hide code" toggle button that expands/collapses the code section
- The preview container should have a subtle dotted/grid background pattern to distinguish it from the page content
- Center the preview content within the container
