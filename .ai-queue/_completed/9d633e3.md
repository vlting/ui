# A11y Fix: Input & Textarea — Use Native `<input>` and `<textarea>` Elements

## Priority: CRITICAL

## Root Cause

**`tag` prop in Tamagui v2 RC's `styled()` does NOT change the rendered HTML element.**

- `styled(View, { tag: 'input' })` renders `<div tag="input">`, not `<input>`. Users CANNOT TYPE into it.
- `styled(Text, { tag: 'textarea' })` renders `<span tag="textarea">`, not `<textarea>`.
- `styled(Text, { tag: 'label' })` renders `<span tag="label">`, not `<label>`.

These components are fundamentally broken for web.

## Scope

Files modified:
- `packages/components/Input/Input.tsx`
- `packages/components/Textarea/Textarea.tsx`

## Required Changes

### 1. Input — Use native `<input>` element

Replace the `styled(View, { tag: 'input' })` `InputField` with a native `<input>` element:

```tsx
// REMOVE this — it renders as <div>, not <input>:
const InputField = styled(View, { tag: 'input', ... })

// REPLACE with a native <input> inside the styled frame:
function Input({ size = 'md', placeholder, value, defaultValue, onChangeText, label, ... }) {
  const inputId = useId()
  const helperId = useId()

  return (
    <YStack>
      {label && (
        <label htmlFor={inputId} style={{ /* label styles */ }}>
          {label}
        </label>
      )}
      <InputFrame size={size} error={error} disabled={disabled}>
        {leadingSlot && <SlotFrame>{leadingSlot}</SlotFrame>}
        <input
          id={inputId}
          type="text"
          placeholder={placeholder}
          value={value}
          defaultValue={defaultValue}
          onChange={onChangeText ? (e) => onChangeText(e.target.value) : undefined}
          disabled={disabled}
          aria-invalid={error || undefined}
          aria-describedby={displayHelper ? helperId : undefined}
          aria-label={!label ? placeholder : undefined}
          style={{
            flex: 1,
            border: 'none',
            outline: 'none',
            background: 'transparent',
            fontFamily: 'inherit',
            fontSize: 'inherit',
            color: 'inherit',
            padding: 0,
            width: '100%',
          }}
        />
        {trailingSlot && <SlotFrame>{trailingSlot}</SlotFrame>}
      </InputFrame>
      {displayHelper && (
        <InputHelper id={helperId} tone={error ? 'error' : 'neutral'}>
          {displayHelper}
        </InputHelper>
      )}
    </YStack>
  )
}
```

**Key points:**
- The `InputFrame` (styled XStack) remains as the visual border/background wrapper — this is fine as a `<div>`
- The actual `<input>` is a native HTML element inside the frame
- The label is a native `<label>` element with `htmlFor` linking to the input `id`
- Use `useId()` from React for unique IDs
- The native `<input>` inherits font properties from the frame via CSS `inherit`

### 5. Textarea — Use native `<textarea>` element

Same pattern as Input. Replace `styled(Text, { tag: 'textarea' })` with a native `<textarea>`:

```tsx
function Textarea({ value, defaultValue, onChangeText, placeholder, label, ... }) {
  const textareaId = useId()
  const helperId = useId()

  return (
    <TextareaFrame>
      {label && (
        <label htmlFor={textareaId}>
          <StyledLabelText>{label}</StyledLabelText>
        </label>
      )}
      <textarea
        id={textareaId}
        value={value}
        defaultValue={defaultValue}
        onChange={onChangeText ? (e) => onChangeText(e.target.value) : undefined}
        placeholder={placeholder}
        disabled={disabled}
        rows={rows}
        maxLength={maxLength}
        aria-invalid={error || undefined}
        aria-describedby={displayHelper ? helperId : undefined}
        style={{
          fontFamily: 'inherit',
          fontSize: 'inherit',
          color: 'inherit',
          background: 'transparent',
          border: '1px solid',
          borderColor: 'inherit',
          borderRadius: 'inherit',
          padding: '8px 12px',
          minHeight: 80,
          outline: 'none',
          resize: 'vertical',
          width: '100%',
        }}
      />
      {displayHelper && (
        <TextareaHelper id={helperId} color={error ? '$red10' : '$colorSubtitle'}>
          {displayHelper}
        </TextareaHelper>
      )}
    </TextareaFrame>
  )
}
```

**Alternative approach for Textarea**: Keep the styled frame for the border/visual and put a bare `<textarea>` inside it (like the Input pattern with the frame as wrapper).
