## Context
useSearch filters a list of items by a query string using a consumer-provided filter function. Currently returns `inputProps` as a static object instead of a `getInputProps()` getter, inconsistent with all other prop-getter hooks.

**Current API (from useSearch.ts):**
- Props: `{ items: T[], filterFn: (item: T, query: string) => boolean }`
- Returns: `{ query, setQuery, filtered, inputProps: { value, onChange } }`
- `filtered` is memoized via `useMemo`
- `inputProps.onChange` expects `ChangeEvent<HTMLInputElement>`
- No ARIA attributes on inputProps

    <div>
      <input {...getInputProps()} data-testid="input" />
