# Per-Category Barrel Exports & .gitattributes

Create per-category barrel export files and mark generated files in `.gitattributes`.

## Scope
- `packages/icons/categories/` (create directory + 20 files)
- `.gitattributes` (create)

## Instructions

### Part 1: Per-Category Barrel Exports

Create `packages/icons/categories/` directory with one barrel export file per category. Each file re-exports all icons belonging to that category.

Read `packages/icons/manifest.json` to determine which icons belong to each category. The 20 categories are:
arrows, buildings, business, communication, design, development, device, document, editor, finance, food, game-and-sports, health-and-medical, logos, map, media, others, system, user-and-faces, weather

For each category, create `packages/icons/categories/{category}.ts`:
```ts
// packages/icons/categories/arrows.ts
export { RiArrowDownCircleFill } from '../generated/RiArrowDownCircleFill'
export { RiArrowDownCircleLine } from '../generated/RiArrowDownCircleLine'
// ... all icons in the "arrows" category
```

**IMPORTANT:** Write a small Node.js script at the top of your work to generate these files from the manifest (don't hand-write 20 files). You can create a temporary script, run it, then delete it. Or just write them directly using the manifest data.

Actually — **update `scripts/generate-icons.mjs`** to also generate the category barrel files. Add a new section to the existing generator that:
1. Reads the manifest (or uses the in-memory data already available during generation)
2. For each category, collects all component names
3. Writes `packages/icons/categories/{category}.ts` with re-exports from `../generated/`
4. Also writes `packages/icons/categories/index.ts` that re-exports all categories:
```ts
export * from './arrows'
export * from './buildings'
// ...
```

### Part 2: .gitattributes

Create `.gitattributes` at the repo root with:
```
packages/icons/generated/** linguist-generated=true
packages/icons/categories/** linguist-generated=true
packages/icons/manifest.json linguist-generated=true
packages/icons/index.ts linguist-generated=true
packages/icons/types.ts linguist-generated=true
```

This tells GitHub to collapse generated files in PR diffs and exclude them from language statistics.

### Part 3: Re-run generator

After updating `scripts/generate-icons.mjs`, run it to generate the category files:
```bash
node scripts/generate-icons.mjs
```

## Verification
- `packages/icons/categories/` contains 20 category files + 1 index.ts
- Each category file re-exports the correct icons (spot-check arrows and system)
- `.gitattributes` exists at repo root with the correct entries
- Generator still produces identical output when run twice (idempotent)
