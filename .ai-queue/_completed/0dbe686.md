# Segment 5: Implement Block Live Previews

## Objective
Replace the generic "Live preview available when block components are shipped" placeholder in block pages with actual live block component rendering.

## Scope
- `examples/docs/components/block-preview.tsx`
- `examples/docs/app/docs/blocks/[name]/page.tsx`

## Instructions

### Step 1: Update block-preview.tsx

Transform `block-preview.tsx` from a static placeholder into a component that renders actual block components from `@vlting/ui`.

The blocks exported from `@vlting/ui` include:
- Login01, Login02, Login03, Login04, Login05
- Signup01, Signup02, Signup03, Signup04, Signup05
- Sidebar01 through Sidebar15, SidebarWFloatingHeader
- Dashboard01
- MobileTabLayout, AppStoreCard, FullScreenHero

**Approach:**
1. Create a mapping of block slugs (from block-registry) to their component imports
2. Use dynamic imports or a static map to render each block
3. Wrap the rendered block in a scaled-down container (blocks are full-page layouts, so scale them to fit the preview)

**Important Tamagui v2 RC consideration:**
Block components are Tamagui components and need the AnyFC cast pattern:
```tsx
import { Login01 as _Login01 } from '@vlting/ui'
type AnyFC = ComponentType<any>
const Login01 = _Login01 as AnyFC
```

**Preview container approach:**
Since blocks are full-page layouts, render them inside a scaled container:
```tsx
<div className="relative overflow-hidden rounded border border-border" style={{ height: 400 }}>
  <div style={{ transform: 'scale(0.5)', transformOrigin: 'top left', width: '200%', height: '200%' }}>
    <BlockComponent />
  </div>
</div>
```

If a block component fails to render (Tamagui dependency issues), fall back to the code preview gracefully. Use try/catch or an ErrorBoundary.

### Step 2: Update blocks/[name]/page.tsx

Update the block detail page to pass the block slug to `BlockPreview` so it can look up the right component:
- Add `slug={block.slug}` prop to `<BlockPreview>`
- Or restructure to render the block component directly in the page

### Fallback Strategy
If importing all 30 blocks creates bundle size issues or Tamagui rendering problems:
1. Start with the simplest blocks (Login01, Signup01, Dashboard01)
2. For blocks that fail to render, show the code example instead of the placeholder
3. Replace the "available when shipped" text with a styled code block showing the block's code

## Verification
- Block pages show either live rendered blocks or code previews (not the generic placeholder)
- At minimum Login01, Signup01, and Dashboard01 render live
- No runtime errors on block pages
- Fallback to code preview works for any blocks that can't render
