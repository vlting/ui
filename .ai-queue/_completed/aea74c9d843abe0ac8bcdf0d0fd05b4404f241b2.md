### AppShellResponsive — Responsive Sidebar with Sheet Drawer on Mobile

A responsive app shell that uses a full Sidebar on desktop and a Sheet (bottom drawer) on mobile for navigation. The consumer controls which mode is active via a prop.

**Props:** `AppShellResponsiveProps` with:
- `groups: NavGroup[]` — navigation groups (same as sidebar blocks)
- `header?: ReactNode` — navigation header
- `footer?: ReactNode` — navigation footer
- `children?: ReactNode` — main content area
- `mode?: 'sidebar' | 'sheet'` — display mode (default: 'sidebar')
  - Consumer switches this based on screen width (e.g., using a media query hook or responsive prop)
- `sheetOpen?: boolean` — controlled Sheet open state (only used in sheet mode)
- `onSheetOpenChange?: (open: boolean) => void` — Sheet state callback
- `sidebarCollapsible?: 'offcanvas' | 'icon' | 'none'`
- `sidebarWidth?: number`

**Layout:**

**Sidebar mode (desktop):**
- Row layout: Sidebar.Root (left) + main content area (flex={1})
- Standard sidebar with groups via SidebarNavGroup helper
- Sidebar.Header, Sidebar.Content, Sidebar.Footer

**Sheet mode (mobile):**
- Main content area takes full width
- Sheet.Root at the bottom with navigation inside Sheet.Frame
- Sheet.Overlay, Sheet.Handle for standard drawer UX
- Inside Sheet.Frame: header, then a scrollable nav section with SidebarNavGroup helpers (not inside a Sidebar.Root), then footer
- The `<nav>` element wraps the groups for landmark semantics

**Implementation notes:**
- Import NavGroup and SidebarNavGroup from `../sidebar/_shared`
- In sheet mode, render navigation groups inside a `<nav>` element (styledHtml('nav')) WITHOUT using Sidebar.Root (Sheet and Sidebar are separate paradigms)
- Use Button from `../../components/Button` for a "Menu" trigger button in sheet mode (only rendered in sheet mode, placed in the main content area or passed as part of children)
