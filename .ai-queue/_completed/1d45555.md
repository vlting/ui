## Test Coverage by Component Type

### Interactive Components (Button, Checkbox, Input, Select, etc.)
- Renders correct semantic element (e.g., `<button>`, `<input>`)
- All variants render (variant, size, tone combinations)
- States: disabled, loading, error, active
- Keyboard interaction: Tab focus, Enter/Space activation, Escape dismissal
- ARIA attributes: roles, states, properties
- Focus management: visible focus, focus trap (for modals)
- Screen reader: accessible name, state announcements

### Container Components (Card, Accordion, Dialog, etc.)
- Renders correct structure
- Open/close behavior
- Keyboard: trigger activation, Escape to close
- Focus trap (for modals)
- ARIA: expanded/collapsed, controls, labelledby

### Form Components (Input, Checkbox, Select, etc.)
- Label association (htmlFor/id)
- Error state: `aria-invalid`, error message with `aria-describedby`
- Helper text: `aria-describedby`
- Required field: `aria-required`
- Controlled and uncontrolled behavior
