### 1. `packages/components/Chart/ChartTooltip.tsx`

A presentational tooltip component that renders a styled box with data series information. This component does NOT manage positioning — Victory's container handles that. This component provides the **content rendering** for tooltips.

**Props:**

```typescript
import type { ComponentType } from 'react'
import type { TooltipVariant, TooltipIndicator } from './types'

interface TooltipPayloadItem {
  name: string
  value: number
  color: string
}

export interface ChartTooltipProps {
  /** Tooltip display variant */
  variant?: TooltipVariant
  /** Whether the tooltip is currently visible */
  active?: boolean
  /** Data items to display */
  payload?: TooltipPayloadItem[]
  /** Category label (e.g., month name, date) */
  label?: string
  /** Custom label formatter */
  labelFormatter?: (label: string) => string
  /** Custom value formatter */
  valueFormatter?: (value: number) => string
  /** Hide the label */
  hideLabel?: boolean
  /** Hide the color indicator */
  hideIndicator?: boolean
  /** Indicator style */
  indicator?: TooltipIndicator
  /** Key to use for item names */
  nameKey?: string
  /** Custom content (used with variant="custom") */
  children?: React.ReactNode
}
```

**Implementation approach:**

- Use `useTheme()` from tamagui to resolve token colors at render time
- Use a native `<div>` for the tooltip container (not Tamagui View — needs precise inline styles for positioning context)
- Tooltip container styles:
  - `background`: `theme.background.val`
  - `border`: `1px solid ${theme.borderColor.val}`
  - `borderRadius`: `7px` (matches `$radius.3`)
  - `padding`: `8px 12px` (matches `$space.0.75` / `$space.1.5`)
  - `boxShadow`: `0 2px 8px rgba(0,0,0,0.08)`
  - `fontSize`: `12px`
  - `color`: `theme.color.val`
  - `minWidth`: `120px`
  - `pointerEvents`: `none` (tooltip shouldn't capture mouse events)

- If `active` is false or `payload` is empty, return `null`

- **Label section:** If `label` is present and `hideLabel` is not true, render the label at the top. Apply `labelFormatter` if provided. Style: `color: theme.color8.val` (subtitle color), `marginBottom: 4px`, `fontWeight: 500`

- **Payload items:** For each item in `payload`, render a row with:
  - Color indicator (unless `hideIndicator`):
    - `dot` (default): 8px circle with `backgroundColor: item.color`
    - `line`: 12px wide, 2px tall line with `backgroundColor: item.color`
    - `dashed`: 12px wide, 2px tall dashed line with `borderBottom: 2px dashed item.color`
  - Name: `item.name` (or `item[nameKey]` if nameKey provided)
  - Value: `item.value` (formatted via `valueFormatter` if provided), right-aligned, `fontVariantNumeric: 'tabular-nums'`
  - Row style: `display: flex`, `alignItems: center`, `gap: 8px`, `padding: 2px 0`

- **Variant behavior:**
  - `default`: Standard label + items with dot indicators
  - `cursor`: Same as default (cursor line is handled by Victory container, not this component)
  - `dot`: Dot indicators (same as default)
  - `line`: Line indicators
  - `label`: Only show label, no payload items
  - `custom`: Render `children` instead of payload items
  - `advanced`: Show all elements with icons from ChartConfig (requires `useChartContext`)
  - `icons`: Show icons from ChartConfig next to item names
  - `formatter`: Always use `valueFormatter` (same as default if formatter provided)

- For `icons` and `advanced` variants, use `useChartContext()` to access `config` for icon components. Render the icon at 14px size next to the name.

### 2. `packages/components/Chart/ChartLegend.tsx`

A legend component that renders color indicators + labels from ChartConfig.

**Props:**

```typescript
import type { ChartConfig, LegendLayout } from './types'

export interface ChartLegendProps {
  /** Legend layout direction */
  layout?: LegendLayout
  /** Chart configuration (provides labels and colors) */
  config: ChartConfig
  /** Vertical alignment hint (for container positioning) */
  verticalAlign?: 'top' | 'bottom'
  /** Key to use for item names (defaults to 'label' from config) */
  nameKey?: string
}
```

**Implementation approach:**

- Use `useTheme()` from tamagui to resolve token colors
- Use `getSeriesColors` from `./utils` to resolve config colors to hex
- Build a flat theme lookup from `useTheme()` (same pattern as Chart.tsx)

- Use semantic HTML: `<div>` wrapper, `<ul>` list, `<li>` items
- Wrapper: `display: flex`, `justifyContent: center`, `padding: 8px 0`
- List: `display: flex`, `listStyle: none`, `padding: 0`, `margin: 0`, `gap: 16px`
  - If `layout === 'vertical'`: `flexDirection: column`, `gap: 8px`
- Each `<li>`: `display: flex`, `alignItems: center`, `gap: 6px`, `fontSize: 12px`
- Color indicator: `<span>` with `display: inline-block`, `width: 8px`, `height: 8px`, `borderRadius: 50%`, `backgroundColor: resolvedColor`, `flexShrink: 0`
- Label text: `color: theme.color.val`

**Accessibility:**
- `<ul>` with `role="list"` for explicit list semantics
- Each `<li>` has text content (not color-only) — the label text is the accessible identifier
- Color indicators are decorative (`aria-hidden="true"`)
