# Icon Browser Page

Build a client-side icon browser page at `/docs/icons` that displays all 3,229 icons from the `@vlting/ui` icon system (Remix Icon v4.9.1) with search, category filter, and virtualized rendering.

## Context
- Icon package: `packages/icons/`
- Manifest: `packages/icons/manifest.json` — contains `{ version, count: 3229, categories: [...20], icons: [...1689 entries with name/category/variants/components] }`
- Each icon entry has `name`, `category`, `variants: ['fill', 'line']`, and `components: { fill: 'RiFooFill', line: 'RiFooLine' }`
- 20 categories: arrows, buildings, business, communication, design, development, device, document, editor, finance, food, game-and-sports, health-and-medical, logos, map, media, others, system, user-and-faces, weather
- Icons are React components accepting `{ size?: number | string; color?: string }`
- Existing docs site pattern: `examples/docs/` with Next.js 15 App Router
- This is a **client component** page (needs search state, click handlers, dynamic rendering)

## Instructions

### 1. Create icon data file (`examples/docs/lib/icon-data.ts`)
- Import the manifest data from `packages/icons/manifest.json` (or inline the data if import won't work in Next.js)
- Export types: `IconEntry { name: string; category: string; variants: string[]; components: Record<string, string> }`
- Export the icon list and category list
- Export helper: `searchIcons(query: string, category?: string): IconEntry[]` — filters by name substring match

### 2. Create icon browser component (`examples/docs/components/icon-browser.tsx`)
- **'use client'** component
- State: search query (string), selected category (string | 'all'), selected variant ('line' | 'fill'), copied icon name (string | null for feedback)
- **Search bar**: Input at top with placeholder "Search 3,229 icons..."
- **Category filter**: Horizontal scrollable row of category buttons/chips. "All" + 20 categories. Active category highlighted.
- **Variant toggle**: Line / Fill toggle buttons
- **Icon grid**: CSS Grid layout (responsive columns, ~80px per cell)
  - Each cell: icon rendered at 24px, icon name below in small text
  - On click: copy import statement to clipboard (`import { RiIconNameLine } from '@vlting/ui'`), show brief "Copied!" feedback
  - Use Tamagui View/Text for layout, themed colors for text/backgrounds
- **Virtualized rendering**: Since there are 3,229 icons, use windowing. Options:
  - CSS `content-visibility: auto` on grid rows (simplest, good browser support)
  - Or a simple intersection observer approach
  - Do NOT add react-window/react-virtuoso as a dependency — keep it lightweight
- **Empty state**: "No icons found" when search/filter yields nothing
- **Count display**: Show "Showing X of 3,229 icons" below filters

### 3. Create icon browser page (`examples/docs/app/docs/icons/page.tsx`)
- Server component wrapper that imports and renders the IconBrowser client component
- Page metadata: title "Icons — @vlting/ui", description
- Header section with title "Icon Browser", description "Browse and search 3,229 Remix Icons. Click any icon to copy its import."
- Import path reference: `import { RiIconName } from '@vlting/ui'`
- DynamicIcon usage example via CodeBlock: `import { DynamicIcon } from '@vlting/ui'` with `<DynamicIcon name="arrow-down" variant="line" size={24} />`

### 4. Icon rendering approach
- Since we can't import all 3,229 icons statically (bundle size), use the `DynamicIcon` component from `packages/icons/Icon.tsx` which uses lazy loading
- Or render SVGs directly if DynamicIcon isn't suitable in the docs app context
- If neither works easily, render icon names only with a placeholder SVG and the copy-to-clipboard functionality — the browser is primarily a reference tool
