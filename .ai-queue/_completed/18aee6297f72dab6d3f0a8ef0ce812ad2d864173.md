# Stage 2, Segment 2: Calendar & Carousel — Native Buttons

## Context

Both components have interactive elements (`ViewJsx` with `onPress` + `role="button"`) that are not keyboard-focusable. These need to be replaced with native `<button>` elements.

**Epic branch:** `fix/quality-audit`
**Feature flag:** `quality_audit`
**Parent issue:** #3 (Epic), #5 (Stage 2)

## Scope

- `packages/components/Calendar/Calendar.tsx`
- `packages/components/Carousel/Carousel.tsx`

## Instructions

### Read First
- `AI_CONSTITUTION.md` — especially §2.8 (Semantic HTML)
- `packages/FRONTEND_QUALITY.contract.md` — §1 (Rendered Markup), §2 (Accessibility)
- The existing `Toggle` component as a reference for `styledHtml('button')` usage

### Changes Required

#### Calendar.tsx

1. **Fix nav arrows (Previous/Next month)** — Currently `ViewJsx` with `role="button"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`, `aria-label="Previous month"` / `aria-label="Next month"`
   - Add `focusVisibleStyle` (standard pattern: `outlineWidth: 2, outlineOffset: 2, outlineColor: '$outlineColor', outlineStyle: 'solid'`)
   - Remove the manual `role="button"` — native `<button>` provides this

2. **Fix day cells** — Currently `ViewJsx` with `role="gridcell"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`
   - Add `aria-label` with the full date string
   - Add `aria-selected="true"` for the selected day (instead of or in addition to `role="gridcell"`)
   - Add `focusVisibleStyle`
   - Remove manual `role="gridcell"` if switching to button (keep `role="gridcell"` if day cells remain in a grid structure)

3. **Keyboard operability** — All buttons must be:
   - Focusable via Tab
   - Activatable via Enter and Space

#### Carousel.tsx

1. **Fix Previous/Next buttons** — Currently `ViewJsx` with `role="button"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`, `aria-label="Previous slide"` / `aria-label="Next slide"`
   - Add `focusVisibleStyle`
   - Remove manual `role="button"`

2. **Fix dot indicators** — Currently `ViewJsx` with `role="button"` + `onPress`:
   - Replace with `styledHtml('button')` elements
   - Add `type="button"`, `aria-label="Go to slide N"`
   - Add `aria-current="true"` for the active dot
   - Add `focusVisibleStyle`
   - Remove manual `role="button"`
   - Note: Do NOT fix the 8x8px size issue here — token discipline is Stage 4

3. **Keyboard operability** — All buttons must be:
   - Focusable via Tab
   - Activatable via Enter and Space

### Important Constraints

- Do NOT change visual appearance — only the underlying HTML element
- Do NOT fix token discipline issues (hardcoded pixels) — that's Stage 4
- Do NOT add keyboard navigation (arrow keys between days/slides) — only ensure Tab/Enter/Space work
- Do NOT add reduced motion support — that's Stage 5
- Use `styledHtml('button')` from `tamagui`
- Use `as any` pattern on styled options if Tamagui v2 RC type bugs appear
- All new `<button>` elements need `type="button"`
