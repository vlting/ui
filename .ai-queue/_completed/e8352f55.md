# Task: Remove brand infrastructure from docs and showcase apps

## Context
The branding system is being replaced by `generateTheme()`. All brand-specific UI (BrandProvider, BrandSwitcher, BrandContext, useBrand, useBrandCSSProperties) must be removed from the apps. The theme demo mechanism should use `generateTheme()` + preset configs instead.

## Requirements

### 1. Rewrite `apps/docs/components/providers.tsx`

**Current state (154 lines):**
- BrandContext with brand state, setBrand, brandOptions
- useBrand() hook
- useBrandCSSProperties() hook (sets ~15+ CSS vars + calls injectBrandVars)
- BrandWrapper wraps StlProvider
- BrandProvider wraps ThemeProvider > BrandContext.Provider
- Providers wraps everything

**New state:**
- Remove BrandContext, useBrand, useBrandCSSProperties, BrandWrapper, BrandProvider entirely
- `Providers` component becomes:
  ```tsx
  import { StlProvider } from '@vlting/stl-react'
  // Import from local relative path since this is a monorepo app
  import { generateTheme, themeToVars, THEME_PRESET_DEFAULT } from '../../../packages/design-tokens'

  // Module-level theme — computed once at import time
  const theme = generateTheme(THEME_PRESET_DEFAULT)
  const lightVars = themeToVars(theme, 'light')
  const darkVars = themeToVars(theme, 'dark')

  export function Providers({ children }: { children: React.ReactNode }) {
    return (
      <StlProvider defaultColorMode="light">
        {children}
      </StlProvider>
    )
  }
  ```
- For the theme demo/switcher feature: create a simple mechanism where a `<style>` tag is written with vars from `themeToVars()`. This can be a small `ThemeDemoSwitcher` component that:
  - Has a `<select>` dropdown with the 4 presets
  - On change, calls `generateTheme(preset)` + `themeToVars(theme, mode)`
  - Writes vars to a `<style id="stl-theme-demo">` tag
  - Stores selection in localStorage
