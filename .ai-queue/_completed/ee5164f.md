## Instructions

Edit `examples/docs/components/component-examples.tsx` to add live renderers for all missing components. This file already imports some components from `@vlting/ui` with AnyFC casts. Add the remaining imports and renderers.

### Current Pattern

The file uses this pattern:
1. Import from `@vlting/ui` with underscore prefix: `import { Alert as _Alert } from '@vlting/ui'`
2. Cast to AnyFC: `const Alert = _Alert as AnyFC & { Root: AnyFC; Title: AnyFC; ... }`
3. Add to `liveExamples` map: `alert: { Basic: () => (<Alert.Root>...</Alert.Root>) }`

### Key Notes
- The renderer function names in `liveExamples` must match the `slug` field in registry.ts (e.g., `'alert-dialog'`, `'radio-group'`, `'toggle-group'`)
- The example names in each renderer must match the example names in registry.ts (e.g., `Basic`, `Variants`)
- Read the registry.ts file (after segment 001 adds examples) to verify the example names
- For components that are tricky to render in isolation (Sheet, Drawer — need open state), use `useState` in the renderer
- The AnyFC cast pattern suppresses Tamagui v2 RC type issues

### Guidelines
- Keep renderers concise — they just need to show the component working
- Use inline styles (`style={{ display: 'flex', gap: 8 }}`) for layout, not Tailwind classes (these render inside the preview iframe)
- Match existing renderer style from button, input, card examples
- For overlay/modal components, include a trigger button that opens them
