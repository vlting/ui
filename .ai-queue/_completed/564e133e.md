# Task: Fix Button focus outline — wrong scale tokens

## Root Cause
Button compound variants use color-scale tokens (`$primary10`, `$primary9`) for the `outline` shorthand,
but `outline` resolves against the **outline scale** which only has combo tokens.
`$primary10` doesn't exist in the outline scale → silently dropped → no visible outline.

## Fix
Replace color tokens with outline-scale combo tokens in all compound variant `:focus` blocks:

| Current (broken) | Fixed | Resolves to |
|---|---|---|
| `outline: '$primary10'` | `outline: '$primaryMax'` | primary10 color |
| `outline: '$primary9'` | `outline: '$primary'` | primary9 color |
| `outline: '$secondary10'` | `outline: '$secondaryMax'` | secondary10 color |
| `outline: '$secondary9'` | `outline: '$secondary'` | secondary9 color |
| `outline: '$neutral10'` | `outline: '$neutralMax'` | neutral10 color |
| `outline: '$neutral9'` | `outline: '$neutral'` | neutral9 color |
| `outline: '$error10'` | `outline: '$errorMax'` | error10 color |
| `outline: '$error9'` | `outline: '$error'` | error9 color |

- `packages/components/Button/Button.tsx`

## Also
Remove the redundant individual outline props from base `:focus` (width/style/offset) — the combo shorthand already sets all of them.
