# Segment 2: Migrate Home Page + Site Header + Theme Toggle + Brand Switcher

## Objective
Migrate hardcoded Tailwind color classes to semantic theme tokens in the main landing page, site header, theme toggle, and brand switcher components.

## Context
Segment 1 established semantic CSS custom properties (`--color-*`) and Tailwind theme extensions. This segment migrates the top-level UI shell to use those tokens.

Current state of these files:
- `app/page.tsx` — Uses `text-gray-600 dark:text-gray-400`, `bg-gray-900`, `border-gray-300`, `bg-gray-50 dark:bg-gray-900`, etc.
- `components/site-header.tsx` — Uses `border-gray-200 dark:border-gray-800`, `text-gray-600 dark:text-gray-400`, `border-gray-300 dark:border-gray-600`, etc.
- `components/theme-toggle.tsx` — Uses `border-gray-300 dark:border-gray-600`, `hover:bg-gray-100 dark:hover:bg-gray-800`
- `components/brand-switcher.tsx` — Uses `border-gray-300 dark:border-gray-600`, `text-gray-700 dark:text-gray-200`, `text-gray-500 dark:text-gray-400`

## Instructions

### Migration Pattern

Replace hardcoded Tailwind color classes with the new semantic token classes. The mapping:

| Old pattern | New pattern |
|------------|-------------|
| `text-gray-900 dark:text-gray-100` | `text-foreground` |
| `text-gray-600 dark:text-gray-400` | `text-foreground-secondary` |
| `text-gray-500 dark:text-gray-400` | `text-muted-foreground` |
| `text-gray-700 dark:text-gray-200` | `text-foreground-secondary` |
| `border-gray-200 dark:border-gray-800` | `border-border` (or just `border` since DEFAULT is set) |
| `border-gray-300 dark:border-gray-600` | `border-border` |
| `bg-gray-50 dark:bg-gray-900` | `bg-surface-muted` |
| `bg-white dark:bg-gray-950` | `bg-background` |
| `hover:bg-gray-100 dark:hover:bg-gray-800` | `hover:bg-accent` |
| `bg-gray-900 text-white` (buttons) | `bg-foreground text-background` (inverts nicely) |

### Contrast Fixes

- Ensure all text meets WCAG AA: `text-muted-foreground` (gray-600 in light, gray-400 in dark) achieves ~5.7:1 on white — passes AA
- For the primary CTA button on the home page (`bg-gray-900 text-white`), using `bg-foreground text-background` maintains maximum contrast
- For secondary buttons, ensure hover states have visible contrast change

### Files to Modify

1. **`examples/docs/app/page.tsx`** — Replace all hardcoded gray/blue Tailwind classes with semantic tokens
2. **`examples/docs/components/site-header.tsx`** — Replace border, text, background colors
3. **`examples/docs/components/theme-toggle.tsx`** — Replace border and hover colors
4. **`examples/docs/components/brand-switcher.tsx`** — Replace border, text, and SVG colors. Keep the `flash` blue accent colors since they indicate active state (but migrate them to `primary` token if possible)

### Important Notes
- Do NOT change the structure or behavior of these components — only color classes
- Keep `dark:` prefixed classes ONLY if the semantic token doesn't already handle both modes (most tokens handle both via CSS variables, so `dark:` variants can often be removed)
- The blue focus ring `focus:ring-blue-500/50` in brand-switcher can stay or migrate to `focus:ring-primary/50`
