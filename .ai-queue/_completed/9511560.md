## Instructions

### 1. Create `packages/blocks/sidebar/_shared.tsx`

Define shared types and utilities for sidebar blocks:

```typescript
// Types
export interface NavItem {
  label: string
  icon?: React.ReactNode
  href?: string
  onPress?: () => void
  active?: boolean
  disabled?: boolean
  children?: NavItem[]   // for nested/submenu navigation
  badge?: string | number
}

export interface NavGroup {
  label?: string
  items: NavItem[]
  collapsible?: boolean
  defaultOpen?: boolean
}

export interface SidebarBlockProps {
  groups: NavGroup[]
  header?: React.ReactNode
  footer?: React.ReactNode
  collapsible?: 'offcanvas' | 'icon' | 'none'
  side?: 'left' | 'right'
  variant?: 'sidebar' | 'floating' | 'inset'
  defaultOpen?: boolean
  open?: boolean
  onOpenChange?: (open: boolean) => void
  width?: number
  collapsedWidth?: number
}
```

Create a `SidebarNavItem` helper component that renders a `Sidebar.MenuItem` with the NavItem data:
- If `item.icon` exists, render it alongside the label text
- If `item.badge` exists, render a Badge in the trailing position
- If `item.href` exists, wrap content in a styledHtml('a') link
- Handle `active`, `disabled`, `onPress` props

Create a `SidebarNavGroup` helper that renders a `Sidebar.Group` with a `Sidebar.GroupLabel` (if label is provided) and `Sidebar.Menu` with SidebarNavItems.

### 2. Create Sidebar01 — Simple Navigation Grouped by Section

A straightforward sidebar with navigation items organized into labeled groups.

Layout:
- `Sidebar.Root` wrapping everything
- Optional `Sidebar.Header` with custom content
- `Sidebar.Content` with `SidebarNavGroup` for each group
- Optional `Sidebar.Footer` with custom content
- Separators between groups

Props: `Sidebar01Props extends SidebarBlockProps` with optional `title?: string`

### 3. Create Sidebar02 — Collapsible Sections

Same as Sidebar01 but each group section can be collapsed/expanded. Uses the `Collapsible` component from `packages/components/Collapsible`.

Layout:
- Each group wraps its content in `Collapsible.Root` / `Collapsible.Trigger` / `Collapsible.Content`
- GroupLabel acts as the collapsible trigger with a chevron icon indicator
- Groups default to open/closed based on `NavGroup.defaultOpen` (default: true)

Props: `Sidebar02Props extends SidebarBlockProps`

### 4. Create spec.md files

Follow the simplified template pattern from the auth block specs. Include:
- Block name and description
- Props table
- Layout description
- Accessibility notes (nav landmark, menu roles, keyboard navigation)

### 5. Create test files

Each test file should verify:
- Renders navigation items from groups data
- Renders group labels when provided
- Renders header/footer when provided
- Active item styling (via `active` prop)
- Disabled item behavior
- Click handlers fire for items with `onPress`
- Navigation landmark present (`<nav>` or `role="complementary"`)

Use `@testing-library/react` web patterns:
- `fireEvent.click` (NOT `fireEvent.press`)
- `screen.getByText` / `screen.getByRole`
- `container.querySelector` for landmark checks
