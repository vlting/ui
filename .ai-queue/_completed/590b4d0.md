# Segment 1: Define Semantic Color Token System

## Objective
Establish a comprehensive set of semantic CSS custom properties in `globals.css` and extend `tailwind.config.ts` to reference them. This creates the token foundation that other segments will migrate to.

## Context
The docs site currently uses:
- Hardcoded Tailwind color classes (`text-gray-600`, `border-gray-200`, etc.) in ~20 files
- Two CSS variables in `globals.css`: `--background` and `--foreground`
- Brand-specific CSS variables set by `useBrandCSSProperties()` in `providers.tsx`: `--brand-bg`, `--brand-text`, `--brand-accent`, etc.
- Some files reference `var(--color-foreground)`, `var(--color-border)`, etc. which are NOT currently defined

The goal is to define a complete semantic token system that:
1. Works correctly in both light and dark mode with WCAG AA contrast
2. Responds to brand switching via the `--brand-*` CSS variables
3. Can be used as Tailwind utility classes

## Instructions

### Step 1: Extend `globals.css`

Add a comprehensive set of semantic CSS custom properties to `examples/docs/app/globals.css`. These should use the `--brand-*` variables (set by `useBrandCSSProperties()`) when available, with sensible fallbacks for initial page load.

Define these semantic variables in `:root` (light mode) and `.dark` (dark mode):

**Surfaces:**
- `--color-background` — page background (fallback: `var(--brand-bg, #ffffff)` light / `var(--brand-bg, #0a0a0a)` dark)
- `--color-surface` — card/panel surfaces (fallback: white light / gray-950 dark)
- `--color-surface-muted` — muted surface (fallback: gray-50 light / gray-900 dark)

**Text:**
- `--color-foreground` — primary text (fallback: `var(--brand-text-strong, #0a0a0a)` light / `var(--brand-text-strong, #ededed)` dark)
- `--color-foreground-secondary` — secondary text, MUST meet 4.5:1 contrast on background (fallback: gray-700 `#374151` light / gray-300 `#d1d5db` dark)
- `--color-muted-foreground` — muted text, MUST meet 4.5:1 contrast on background (fallback: gray-600 `#4b5563` light / gray-400 `#9ca3af` dark) — NOTE: gray-500 (#6b7280) on white only achieves ~4.6:1, borderline. Use gray-600 for safety.

**Borders:**
- `--color-border` — standard border (fallback: `var(--brand-border, #e5e7eb)` light / `var(--brand-border, #374151)` dark)
- `--color-border-muted` — subtle border

**Accent:**
- `--color-accent` — accent background for hover/active states (fallback: `var(--brand-accent-subtle, #f3f4f6)` light / `var(--brand-accent-subtle, #1f2937)` dark)
- `--color-accent-foreground` — text on accent background
- `--color-primary` — primary action color (fallback: `var(--brand-accent, #2563eb)` light / `var(--brand-accent, #60a5fa)` dark)
- `--color-primary-foreground` — text on primary (white light / dark)

**Semantic:**
- `--color-muted` — muted background
- `--color-destructive` — error/destructive red that meets WCAG AA (use `#dc2626` light / `#f87171` dark)
- `--color-success` — success green that meets WCAG AA (use `#16a34a` light / `#4ade80` dark)

Ensure ALL text colors meet WCAG AA contrast ratio:
- Normal text (< 18px): 4.5:1 minimum
- Large text (>= 18px bold or >= 24px): 3:1 minimum

### Step 2: Extend `tailwind.config.ts`

Add these CSS variables as Tailwind theme colors so they can be used as utility classes:

```typescript
theme: {
  extend: {
    colors: {
      background: 'var(--color-background)',
      foreground: 'var(--color-foreground)',
      'foreground-secondary': 'var(--color-foreground-secondary)',
      muted: {
        DEFAULT: 'var(--color-muted)',
        foreground: 'var(--color-muted-foreground)',
      },
      border: 'var(--color-border)',
      'border-muted': 'var(--color-border-muted)',
      accent: {
        DEFAULT: 'var(--color-accent)',
        foreground: 'var(--color-accent-foreground)',
      },
      primary: {
        DEFAULT: 'var(--color-primary)',
        foreground: 'var(--color-primary-foreground)',
      },
      surface: {
        DEFAULT: 'var(--color-surface)',
        muted: 'var(--color-surface-muted)',
      },
      destructive: 'var(--color-destructive)',
      success: 'var(--color-success)',
    },
    borderColor: {
      DEFAULT: 'var(--color-border)',
    },
  },
},
```
