# Rewrite Dialog + AlertDialog + Tooltip + Slider + Progress using Tamagui official components

## Context

Part of the broader effort to leverage Tamagui's official component packages. These components handle focus trapping, portal rendering, floating positioning, ARIA attributes, and keyboard navigation — all built-in.

## Implementation

### Dialog (`packages/components/Dialog/Dialog.tsx`)

Replace with `@tamagui/dialog`.

```tsx
import { Dialog as TamaguiDialog } from '@tamagui/dialog'
```

1. Tamagui's Dialog renders `<dialog>` element via Portal, with `aria-modal`, focus trap, Escape dismiss, scroll lock.
2. Compound: `Dialog.Trigger`, `Dialog.Portal`, `Dialog.Overlay`, `Dialog.Content`, `Dialog.Title`, `Dialog.Description`, `Dialog.Close`, `Dialog.Adapt`, `Dialog.FocusScope`.
3. `styled()` wrap `Dialog.Content` for our size variants (sm/md/lg mapping to max-width).
4. `styled()` wrap `Dialog.Overlay` for our animation (fade).
5. Preserve our compound API: `Dialog.Root`, `Dialog.Trigger`, `Dialog.Overlay`, `Dialog.Content`, `Dialog.Title`, `Dialog.Description`, `Dialog.Close`.
6. **Remove dependency on headless Dialog** — Tamagui provides all the behavior.
7. Remove our custom `useFocusTrap` — Tamagui uses `@tamagui/focus-scope`.

### AlertDialog (`packages/components/AlertDialog/AlertDialog.tsx`)

Replace with `@tamagui/alert-dialog`.

```tsx
import { AlertDialog as TamaguiAlertDialog } from '@tamagui/alert-dialog'
```

1. Tamagui's AlertDialog extends Dialog with `role="alertdialog"`, prevents outside interaction.
2. Adds: `AlertDialog.Action`, `AlertDialog.Cancel`, `AlertDialog.Destructive` button roles.
3. `styled()` wrap Content/Overlay for our visual customization.
4. Preserve our compound API, adding new sub-components from Tamagui.

### Tooltip (`packages/components/Tooltip/Tooltip.tsx`)

Replace with `@tamagui/tooltip`.

```tsx
import { Tooltip as TamaguiTooltip, TooltipSimple } from '@tamagui/tooltip'
```

1. Tamagui's Tooltip uses floating-ui for positioning, has `role="tooltip"`, Escape dismiss.
2. Supports: `open`, `onOpenChange`, `delay`, `restMs`, `groupId`, `side`/`align` via floating.
3. Compound: `Tooltip.Trigger`, `Tooltip.Content`, `Tooltip.Arrow`, `Tooltip.Anchor`.
4. `styled()` wrap `Tooltip.Content` for our visual styling.
5. Our current Tooltip is simple (just `TooltipProps` with `content`, `delay`, `side`, `children`). We can keep our simplified API by creating a wrapper that maps to Tamagui's compound structure, similar to `TooltipSimple`.

### Slider (`packages/components/Slider/Slider.tsx`)

Replace with `@tamagui/slider`.

```tsx
import { Slider as TamaguiSlider } from '@tamagui/slider'
```

1. Tamagui's Slider has: `role="slider"`, full ARIA (`aria-valuenow`, `aria-valuemin`, `aria-valuemax`), pointer drag, keyboard support (arrows, PageUp/Down, Home/End), orientation support.
2. Compound: `Slider.Track`, `Slider.TrackActive` (alias `Range`), `Slider.Thumb`.
3. Note: Tamagui's `value` is `number[]` (multi-thumb). Map our single `value: number` to `value: [number]`.
4. `styled()` wrap Track, TrackActive, Thumb for our visual customization.
5. Preserve our `SliderProps` interface.

### Progress (`packages/components/Progress/Progress.tsx`)

Replace with `@tamagui/progress`.

```tsx
import { Progress as TamaguiProgress } from '@tamagui/progress'
```

1. Tamagui's Progress has: `role="progressbar"`, full ARIA, animated indicator, `size` support.
2. Compound: `Progress.Indicator`.
3. `styled()` wrap for our size variants (sm/md/lg mapping to heights).
4. Preserve our `ProgressProps` interface (`value`, `max`, `size`).
