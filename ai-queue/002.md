# Fix Table Component — Proper HTML Table Rendering

## Problem

The Table component in `packages/components/Table/Table.tsx` uses `styled(View, { tag: 'table' })` etc., but Tamagui v2's `tag` prop does NOT override the default `display: flex` layout. The browser needs `display: table`, `display: table-row-group`, `display: table-row`, and `display: table-cell` CSS for proper table formatting context.

**Current result:** The table renders as a vertical stack of text nodes — no columns, no grid alignment. It looks nothing like a table.

## Theme System Context

The current implementation uses Tamagui design tokens correctly for colors and spacing. The issue is purely the missing `display` CSS properties.

## Required Changes

### 1. Fix `packages/components/Table/Table.tsx`

Add explicit `display` properties to each styled component. In Tamagui v2, you can set `display` as a style property:

```
StyledTable:  display: 'table' (web) — need to handle this for web only
StyledHeader: display: 'table-header-group'
StyledBody:   display: 'table-row-group'

StyledHead:   display: 'table-cell'
StyledCell:   display: 'table-cell'

```

**Important Tamagui v2 note:** Tamagui may not support `display: 'table'` directly since it's a non-standard display value for the Tamagui layout engine. If `display` doesn't accept `'table'` as a value, you'll need to use a different approach:

**Approach A — Use `style` prop for web-specific CSS:**
Tamagui components accept a `style` prop for raw CSS. You can add:
```tsx
const StyledTable = styled(View, {
  tag: 'table',
  width: '100%',
  // @ts-expect-error web-only CSS
  style: { display: 'table', borderCollapse: 'collapse' },
})
```

**Approach B — Use className injection:**
Create a small CSS module or inline `<style>` tag that targets the table elements by their HTML tags, since `tag: 'table'` does render the correct HTML element.

**Approach C — Use raw HTML elements:**
Instead of `styled(View, { tag: 'table' })`, use `styled('table' as any, { ... })` or create thin React wrappers around native HTML `<table>`, `<tr>`, `<td>` elements with Tamagui token-based styling via `style` props.

Try Approach A first. If it doesn't work, try Approach C.
