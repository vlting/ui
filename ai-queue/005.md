## Implementation Guidelines

### Compound component pattern
Complex components should use the compound component pattern with `withStaticProperties`:
```tsx
const AccordionRoot = styled(...)
const AccordionItem = styled(...)
const AccordionTrigger = styled(...)
const AccordionContent = styled(...)

export const Accordion = withStaticProperties(AccordionRoot, {
  Item: AccordionItem,
  Trigger: AccordionTrigger,
  Content: AccordionContent,
})
```

### Where to put files
- `packages/components/<ComponentName>/` directory
- Complex components may have multiple files (e.g., `Accordion.tsx`, `AccordionItem.tsx`, `AccordionContext.tsx`)
- Export from `packages/components/index.ts`
- Re-export from `src/index.ts`

### Styling — same shadcn philosophy as Task 004
- Neutral by default
- Minimal decoration
- Consistent interactive states
- Use Tamagui built-ins where available (Sheet, Toast, Popover)

### Animations
- Accordion expand/collapse: use Tamagui animations (`animation="medium"`)
- Sheet slide: Tamagui Sheet handles this
- Toast enter/exit: Tamagui Toast handles this
- Collapsible: same as Accordion
- Keep animations subtle and fast

### Accessibility
These complex components have significant a11y requirements:
- AlertDialog: focus trap, Escape to close, return focus
- DropdownMenu: arrow key navigation, typeahead
- Accordion: aria-expanded, arrow keys between items
- Toast: role="status", aria-live, auto-dismiss
- Table: proper `<table>`, `<thead>`, `<tbody>`, `<th>`, `<td>` semantics

Where Tamagui/Radix provides built-in a11y, leverage it. Where building custom, ensure keyboard navigation and screen reader support.
