# 003 — Pagination: Hover Contrast on Selected Pages + Semantic Button Tags

## Objective

Two issues with Pagination:
1. When hovering over the selected (active) page, the background changes to `$backgroundHover` (very light) while text stays `$color1` (also light) — no contrast.
2. The page buttons use `styled(XStack)` with `tag="button"`, but this does NOT render actual `<button>` elements in Tamagui v2 RC. They must be real `<button>` tags for accessibility.

## Context

- **Hover contrast**: `PageButton` has `hoverStyle: { backgroundColor: '$backgroundHover' }` (palette[1] = very light). When `active={true}`, the background is `$color10` (dark primary) and text is `$color1` (white). On hover, background flips to light but text stays light = invisible.
- **Button tags**: The `tag` prop in `styled()` does NOT change the rendered HTML element in Tamagui v2 RC. `styled(XStack, { tag: 'button' })` renders `<div tag="button">`, NOT `<button>`. Need native button elements for proper semantics and accessibility.

## Tasks

### Hover Contrast Fix

1. **In `PageButton`** (`packages/components/Pagination/Pagination.tsx`):
   - Override `hoverStyle` for the `active` variant. When `active={true}`, the hover background should darken slightly (e.g., `$color11`) instead of switching to `$backgroundHover`.
   - This ensures the selected page remains visually distinct on hover.

2. **In `PageButtonText`**:
   - Ensure the text color for `active` variant does not change on hover — it should remain `$color1` (light on dark background) when the parent is active.

3. **In `NavButton`**:
   - Same treatment for Previous/Next buttons if they have a selected/active state.

### Semantic Button Tags

4. **Convert `PageButton` to render actual `<button>` elements**:
   - Use native `<button>` HTML elements with Tamagui styled wrappers inside for visual styling.
   - Or use `styledHtml('button', { ... })` which correctly sets the HTML element.
   - Ensure all existing styles, variants (`active`, `size`), and hover/press styles are preserved.
   - Remove the `tag="button"` prop usage that doesn't work.

5. **Convert `NavButton` similarly** — must render as `<button>` tag.

6. **Ensure keyboard navigation works**: Tab between buttons, Enter/Space to activate.

7. **Test** hover contrast across light and dark themes with multiple brands.
