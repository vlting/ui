# A11y: Table Component — Proper HTML Table Rendering

## Priority: HIGH

## Scope

Files modified:
- `packages/components/Table/Table.tsx`

## Problem

The Table component uses `styled(View, { tag: 'table' })` etc., but Tamagui v2's `tag` prop does NOT override the default `display: flex`. Without explicit `display: table` / `display: table-row` / `display: table-cell` CSS, the browser renders everything as stacked flex children — no columns, no grid.

## Required Changes

### 1. Add explicit display CSS

Try adding `display` via the `style` prop:
```tsx
const StyledTable = styled(View, {
  tag: 'table',
  width: '100%',
  // @ts-expect-error web-only CSS
  style: { display: 'table', borderCollapse: 'collapse' },
})
```

Apply `display: 'table-header-group'`, `'table-row-group'`, `'table-footer-group'`, `'table-row'`, `'table-cell'` to Header, Body, Footer, Row, Head/Cell respectively.

If `style` doesn't work with `styled()`, use wrapper functions or native HTML elements.

### 2. Add table polish
- Header background: `backgroundColor: '$color2'` on header row
- Row hover: `hoverStyle: { backgroundColor: '$backgroundHover' }` on Row
- Caption: `captionSide: 'bottom'`

### 3. Add `scope` to table headers
Add `scope: 'col'` to `StyledHead` for screen reader column association. If it doesn't work via styled(), document for consumer usage.
