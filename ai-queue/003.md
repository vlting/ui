# Fix Kitchen-Sink Layout — Stack Form Components Vertically

## Problem

In the kitchen-sink example app, sections for form/input components (Input, Textarea, Select, Checkbox, RadioGroup, Switch, Slider, Progress) use `<XStack gap="$3" flexWrap="wrap">` to lay out DemoCards horizontally. This makes the input components cramped and narrow — they should take the full width of the page.

**Current pattern (bad):**
```tsx
<Section title="Input">
  <XStack gap="$3" flexWrap="wrap">
    <DemoCard label="Default">...</DemoCard>
    <DemoCard label="With label & helper">...</DemoCard>
    <DemoCard label="Error state">...</DemoCard>
    <DemoCard label="Sizes">...</DemoCard>
  </XStack>
</Section>
```

**Desired pattern (good):**
```tsx
<Section title="Input">
  <DemoCard label="Default">...</DemoCard>
  <DemoCard label="With label & helper">...</DemoCard>
  <DemoCard label="Error state">...</DemoCard>
  <DemoCard label="Sizes">...</DemoCard>
</Section>
```

Note: `Section` already wraps children in a `<YStack gap="$3">`, so removing the inner `XStack` wrapper will naturally stack DemoCards vertically with proper spacing.
