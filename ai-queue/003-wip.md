# Create `./examples/kitchen-sink` — Component Showcase App

## Overview

Create a multi-page web example app at `./examples/kitchen-sink/` that demonstrates every component from the `layout`, `primitives`, and `forms` modules. The app should feel like a real product (a fictional SaaS dashboard), not a sterile component list.

---

## Pre-flight

Before starting, read:
- `AI_CONSTITUTION.md` — no business logic, presentation-only components
- `DESIGN_CONSTITUTION.md` — UX laws, token discipline, familiar patterns
- The `src/layout/`, `src/primitives/`, and `src/forms/` index files to understand what's exported

---

## App Setup

### Framework & Tooling

- **Vite + React** (`npm create vite@latest kitchen-sink -- --template react-ts`)
- **Routing**: React Router v7 (`react-router-dom`)
- **Styling**: Tamagui (already in `@vlting/ui`) — configure a `TamaguiProvider` at the app root using `@vlting/ui/config`
- **Location**: `./examples/kitchen-sink/`

### Workspace Integration

Add to the root `package.json` `workspaces` field (or create one if absent):
```json
"workspaces": ["examples/*"]
```

The example app's `package.json` should list `@vlting/ui` as a local dependency:
```json
{
  "name": "@vlting/examples-kitchen-sink",
  "dependencies": {
    "@vlting/ui": "*",
    "react-router-dom": "^7.0.0",
    "tamagui": "2.0.0-rc.14"
  }
}
```

### Provider Setup (`src/main.tsx`)

```tsx
import { TamaguiProvider } from 'tamagui'
import { voltaConfig } from './theme/voltaBrand'

<TamaguiProvider config={voltaConfig} defaultTheme="light">
  <RouterProvider router={router} />
</TamaguiProvider>
```

> The `voltaConfig` is produced by `createBrandConfig(voltaBrand)` — see the Brand & Theme section below.

## App Concept: "Volta" — Fictional SaaS Dashboard

The example app is a fictional project management / team dashboard called **Volta**. Tagline: *"Move at the speed of your team."* It gives the implementing agent creative latitude to build convincing UI with realistic-looking mock data (hardcoded arrays/objects — no API calls).

### Brand Usage in the App

- **Logo/wordmark**: use the lucide `Zap` icon in `$cobalt7` color (light) / `$cobalt7` (dark) — the key cobalt shade
- **Primary `Button`**: `backgroundColor="$cobalt7"`, text white — the main CTA color
- **Active nav items**: background `$cobalt1`, text `$cobalt7`, left-border accent `$cobalt7`
- **Focus rings**: the default `$outline` token will inherit the brand's focus ring settings
- **Sidebar background**: `$color1` (the near-white slate — distinct from pure white)
- **TopNav background**: white in light / `$color2` in dark
- **Card surfaces**: `$color1` on top of `$color2` page background

## Routes & Pages

### Route Structure

```
/                     → DashboardPage
/users                → UsersPage
/users/:id            → UserProfilePage
/onboarding           → OnboardingPage
/settings             → SettingsPage
/compose              → ComposePage
```

All routes except `/onboarding` are wrapped in `AppShellLayout` (see below).

---

### AppShellLayout (`src/layouts/AppShellLayout.tsx`)

The persistent chrome for all main pages:

- **`AppShell`** as the outer container
- **`AppShell.Header`** → `TopNav`:
  - Leading: app logo/wordmark (`Text` + lucide `Zap` icon)
  - Trailing: `CommandPalette` trigger button (kbd shortcut hint `⌘K`), notification bell (`Tooltip`-wrapped), user `Avatar` that opens a `Sheet` on press (with profile/logout options)
- **`AppShell.Sidebar`** → `Sidebar`:
  - Nav links to each route using `Button` (tertiary variant) with lucide icons
  - Active state highlighted with token color
  - Bottom: a `Switch`-wrapped dark/light theme toggle
- **`AppShell.Content`** → `<Outlet />` (React Router)
- **`BottomTabs`**: rendered below content on mobile (use `useMedia` from tamagui to conditionally show)
- **`Drawer`**: on mobile, sidebar becomes a `Drawer` triggered by a hamburger button in `TopNav`
- **`CommandPalette`**: globally available, opens on `⌘K`, lists all pages as searchable items

6. `ResizablePanel` demo section: a side-by-side view with a project list on the left and a detail pane on the right
7. `Accordion` at the bottom: "Announcements" section with 3 expandable items

### OnboardingPage (`src/pages/OnboardingPage.tsx`)

**Components used**: `MultiStepForm`, `FormContainer`, `FieldWrapper`, `Label`, `HelperText`, `InlineError`, `Input`, `Select`, `Checkbox`, `OTPInput`, `PasswordStrengthMeter`, `Button`, `ValidationMessage`

A full-screen onboarding wizard (no AppShell — standalone layout). Steps:

1. **Welcome** — Name + email inputs, terms `Checkbox`
2. **Create Password** — Password `Input` + `PasswordStrengthMeter`, confirm password, `InlineError` on mismatch
3. **Verify Email** — Instructional text + `OTPInput` (6 digits), resend link
4. **Your Role** — `Select` for role, `RadioGroup` for team size
5. **Done** — `ValidationMessage` (success), confetti-style `Text` celebration, "Go to Dashboard" `Button`

Validation: Next button disabled unless current step fields are valid. Show `InlineError` on attempted advance with invalid fields.

## Implementation Notes

- Import all components from the appropriate `@vlting/ui` sub-path (e.g., `@vlting/ui/layout`, `@vlting/ui/primitives`, `@vlting/ui/forms`)
- All state is local React state (`useState`) — no global store
- Mock data only — no fetch, no Supabase, no external APIs
- Dark mode: the sidebar `Switch` toggles `defaultTheme` on the `TamaguiProvider` between `light` and `dark`
- Responsive: use `useMedia` from tamagui to adapt layout (sidebar → Drawer, BottomTabs on mobile)
- TypeScript throughout — no `any`
- Keep each page file focused; extract reusable local sub-components only if the file exceeds ~200 lines
